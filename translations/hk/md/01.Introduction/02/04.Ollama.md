## Ollama 中的 Phi 系列


[Ollama](https://ollama.com) 讓更多人可以透過簡單的指令直接部署開源的 LLM 或 SLM，並且也可以建立 API 以協助本地 Copilot 的應用場景。

## **1. 安裝**

Ollama 支援在 Windows、macOS 與 Linux 上運行。你可以透過此連結安裝 Ollama（[https://ollama.com/download](https://ollama.com/download)）。安裝成功後，你可以直接在終端機視窗使用 Ollama 指令來呼叫 Phi-3。你可以在 [可在 Ollama 庫中取得的可用模型](https://ollama.com/library) 查看所有可用項目。如果你在 Codespace 中打開此倉庫，則會已經安裝好 Ollama。

```bash

ollama run phi4

```

> [!NOTE]
> 模型會在你第一次執行時先下載。當然，你也可以直接指定已下載的 Phi-4 模型。我們以 WSL 為例來執行該指令。模型成功下載後，你就可以直接在終端機上互動。

![執行](../../../../../translated_images/hk/ollama_run.e9755172b162b381.png)

## **2. 從 Ollama 呼叫 phi-4 API**

如果你想呼叫 Ollama 產生的 Phi-4 API，可以在終端機使用此指令來啟動 Ollama 伺服器。

```bash

ollama serve

```

> [!NOTE]
> 如果在 MacOS 或 Linux 上執行，請注意你可能會遇到以下錯誤 **"Error: listen tcp 127.0.0.1:11434: bind: address already in use"**。當你執行該指令時可能會看到此錯誤。你可以忽略該錯誤，因為通常代表伺服器已經在執行，或是你可以停止並重新啟動 Ollama：

**macOS**

```bash

brew services restart ollama

```

**Linux**

```bash

sudo systemctl stop ollama

```

Ollama 支援兩種 API：generate 和 chat。你可以依需求向在 11434 埠上運行的本地服務發送請求來呼叫 Ollama 所提供的模型 API。

**Chat**

```bash

curl http://127.0.0.1:11434/api/chat -d '{
  "model": "phi3",
  "messages": [
    {
      "role": "system",
      "content": "Your are a python developer."
    },
    {
      "role": "user",
      "content": "Help me generate a bubble algorithm"
    }
  ],
  "stream": false
  
}'
```

## 其他資源

可在 [其資料庫](https://ollama.com/library) 查看 Ollama 的可用模型列表。

使用此指令從 Ollama 伺服器拉取你的模型

```bash
ollama pull phi4
```

使用此指令運行模型

```bash
ollama run phi4
```

***注意：*** 請造訪此連結 [https://github.com/ollama/ollama/blob/main/docs/api.md](https://github.com/ollama/ollama/blob/main/docs/api.md) 以了解更多

## 從 Python 呼叫 Ollama

你可以使用 `requests` 或 `urllib3` 向上面使用的本地伺服器端點發送請求。不過，在 Python 中使用 Ollama 的常見作法是透過 [openai](https://pypi.org/project/openai/) SDK，因為 Ollama 也提供與 OpenAI 相容的伺服器端點。

以下是 phi3-mini 的範例：

```python
import openai

client = openai.OpenAI(
    base_url="http://localhost:11434/v1",
    api_key="nokeyneeded",
)

response = client.chat.completions.create(
    model="phi4",
    temperature=0.7,
    n=1,
    messages=[
        {"role": "system", "content": "You are a helpful assistant."},
        {"role": "user", "content": "Write a haiku about a hungry cat"},
    ],
)

print("Response:")
print(response.choices[0].message.content)
```

## 從 JavaScript 呼叫 Ollama 

```javascript
// 使用 Phi-4 對檔案進行摘要的範例
script({
    model: "ollama:phi4",
    title: "Summarize with Phi-4",
    system: ["system"],
})

// 摘要的範例
const file = def("FILE", env.files)
$`Summarize ${file} in a single paragraph.`
```

## 從 C# 呼叫 Ollama

建立一個新的 C# Console 應用程式，並加入下列 NuGet 套件：

```bash
dotnet add package Microsoft.SemanticKernel --version 1.34.0
```

然後在 `Program.cs` 檔案中取代這段程式碼

```csharp
using Microsoft.SemanticKernel;
using Microsoft.SemanticKernel.ChatCompletion;

// add chat completion service using the local ollama server endpoint
#pragma warning disable SKEXP0001, SKEXP0003, SKEXP0010, SKEXP0011, SKEXP0050, SKEXP0052
builder.AddOpenAIChatCompletion(
    modelId: "phi4",
    endpoint: new Uri("http://localhost:11434/"),
    apiKey: "non required");

// invoke a simple prompt to the chat service
string prompt = "Write a joke about kittens";
var response = await kernel.InvokePromptAsync(prompt);
Console.WriteLine(response.GetValue<string>());
```

使用下列指令執行此應用程式：

```bash
dotnet run
```
