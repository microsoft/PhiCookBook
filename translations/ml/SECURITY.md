## സുരക്ഷ

Microsoft നമ്മുടെ സോഫ്റ്റ്‌വെയർ ഉൽപ്പന്നങ്ങളുടെയും സേവനങ്ങളുടെയും സുരക്ഷ ഗൗരവമായി കാണുന്നു, ഇതിൽ [Microsoft](https://github.com/Microsoft), [Azure](https://github.com/Azure), [DotNet](https://github.com/dotnet), [AspNet](https://github.com/aspnet) and [Xamarin](https://github.com/xamarin) എന്നിവയിലൂടെ മാനേജുചെയ്യപ്പെടുന്ന എല്ലാ സോഴ്സ് കോഡ് റിപോസിറ്ററികളും ഉൾപ്പെടുന്നു.

നിങ്ങൾ ഏതെങ്കിലും Microsoft-ഉടമയായ റിപോസിറ്ററിയിൽ Microsoft-ന്റെ സുരക്ഷാ ദുർബലതയുടെ നിർവചനത്തിന് അനുയോജ്യമായ ഒരു സുരക്ഷാ ദുർബലത കണ്ടെത്തിയതായി നിങ്ങൾ വിശ്വസിക്കുന്നുവെങ്കിൽ, ദയവായി താഴെ വിവരണമനുസരിച്ച് അത് ഞങ്ങളോട് റിപ്പോർട്ട് ചെയ്യുക.

## സുരക്ഷാ പ്രശ്നങ്ങൾ റിപ്പോർട്ട് ചെയ്യൽ

**സുരക്ഷാ ദുർബലതകൾ പൊതുജന GitHub ഇഷ്യൂകളിലൂടെ റിപ്പോർട്ട് ചെയ്യരുത്.**

പകരം, ദയവായി അവ Microsoft Security Response Center (MSRC)-ൽ [https://msrc.microsoft.com/create-report](https://aka.ms/security.md/msrc/create-report) വഴി റിപ്പോർട്ട് ചെയ്യുക.

If you prefer to submit without logging in, send email to [secure@microsoft.com](mailto:secure@microsoft.com).  If possible, encrypt your message with our PGP key; please download it from the [Microsoft Security Response Center PGP Key page](https://aka.ms/security.md/msrc/pgp).

You should receive a response within 24 hours. If for some reason you do not, please follow up via email to ensure we received your original message. Additional information can be found at [microsoft.com/msrc](https://www.microsoft.com/msrc). 

Please include the requested information listed below (as much as you can provide) to help us better understand the nature and scope of the possible issue:

  * പ്രശ്നത്തിന്റെ തരം (ഉദാ: buffer overflow, SQL injection, cross-site scripting, etc.)
  * പ്രശ്നത്തിന്റെ പ്രകടനവുമായി ബന്ധപ്പെട്ട സോഴ്‌സ് ഫയൽ(കൾ)യുടെ പൂർണ്ണ പാതകൾ
  * പ്രഭാവിതമായ സോഴ്സ് കോഡിന്റെ സ്ഥാനം (tag/branch/commit or direct URL)
  * പ്രശ്നം പുനരുത്പാദിപ്പിക്കാൻ ആവശ്യമായ ഏതെങ്കിലും പ്രത്യേക ക്രമീകരണം
  * പ്രശ്നം പുനരുത്പാദിപ്പിക്കാൻ ഘട്ടം-ഘട്ടമായ നിർദ്ദേശങ്ങൾ
  * പ്രൂഫ്-ഓഫ്-കൺസെപ്റ്റ್ അല്ലെങ്കിൽ എക്സ്പ്ലോയിറ്റ് കോഡ് (സാദ്ധ്യമെങ്കിൽ)
  * പ്രശ്നത്തിന്റെ ബാധകൾ, ഒരു ആക്രമകൻ അത് എങ്ങനെ ഉപയോഗപ്പെടുത്താമെന്നതും ഉൾപ്പെടെ

ഈ വിവരങ്ങൾ നമുക്ക് നിങ്ങളുടെ റിപ്പോർട്ട് വേഗത്തിലുള്ളതായtriage ചെയ്യാൻ സഹായിക്കും.

If you are reporting for a bug bounty, more complete reports can contribute to a higher bounty award. Please visit our [Microsoft Bug Bounty Program](https://aka.ms/security.md/msrc/bounty) page for more details about our active programs.

## ഇഷ്ടഭാഷകൾ

ഞങ്ങൾക്ക് എല്ലാ ആശയവിനിമയങ്ങളും ഇംഗ്ലീഷിൽ ആകാൻ ഇഷ്ടമാണ്.
