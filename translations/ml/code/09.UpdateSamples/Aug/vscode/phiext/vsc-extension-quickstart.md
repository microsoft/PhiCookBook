# നിങ്ങളുടെ VS Code എക്സ്റ്റൻഷനിലേക്ക് സ്വാഗതം

## ഫോൾഡറിലെ ഉള്ളടക്കം

* ഈ ഫോൾഡർ നിങ്ങളുടെ എക്സ്റ്റൻഷനിനായുള്ള ആവശ്യമായ എല്ലാ ഫയലുകളും ഉൾക്കൊള്ളുന്നു.
* `package.json` - ഇത് നിങ്ങളുടെ എക്സ്റ്റൻഷനും കമാൻഡും പ്രഖ്യാപിക്കുന്ന മാനിഫസ്റ്റ് ഫയലാണ്.
  * സാമ്പിൾ പ്ലഗിൻ ഒരു കമാൻഡ് രജിസ്റ്റർ ചെയ്യുകയും അതിന്റെ ശീർഷകം এবং കമാൻഡിന്റെ പേര് നിർവചിക്കുകയും ചെയ്യുന്നു. ഈ വിവരത്തോടെ VS Code കമാൻഡ് പാളറ്റിൽ കമാൻഡ് കാണിക്കാം. ഇതിന് ഇപ്പോൾ പ്ലഗിൻ ലോഡ് ചെയ്യേണ്ടതില്ല.
* `src/extension.ts` - ഇത് നിങ്ങളുടെ കമാൻഡിന്റെ നടപ്പാക്കൽ നിങ്ങൾ നൽകുന്ന പ്രധാന ഫയലാണ്.
  * ഫയൽ ഒരു ഫങ്ക്ഷൻ `activate` എക്സ്പോർട്ടുചെയ്യുന്നു, നിങ്ങളുടെ എക്സ്റ്റൻഷൻ 처음 സജീവമാകുമ്പോൾ (ഈ കേസിൽ കമാൻഡ് എക്സിക്യൂട്ട് ചെയ്യുന്നതിലൂടെ) വിളിക്കപ്പെടുന്നതാണ്. `activate` ഫังก്ഷനിന്റെ ഉള്ളിൽ ഞങ്ങൾ `registerCommand` വിളിക്കുന്നു.
  * കമാൻഡിന്റെ നടപ്പാക്കലുള്ള ഫംഗ്ഷനെ രണ്ടാമത്തെ പാരാമീറ്ററായി `registerCommand`-ലേക്ക് നമുക്ക് കടത്തുന്നു.

## ക്രമീകരണം

* ശിപാർശ ചെയ്ത എക്സ്റ്റൻഷനുകൾ ഇൻസ്റ്റാൾ ചെയ്യുക (amodio.tsl-problem-matcher, ms-vscode.extension-test-runner, and dbaeumer.vscode-eslint)


## ഉടനെ പ്രവർത്തനം തുടങ്ങി തുടങ്ങി

* `F5` അമർത്തി നിങ്ങളുടെ എക്സ്റ്റൻഷൻ ലോഡ് ചെയ്ത ഒരു പുതിയ വിൻഡോ തുറക്കുക.
* കമാൻഡ് പാലറ്റിൽ നിന്ന് നിങ്ങളുടെ കമാൻഡ് പ്രവർത്തിപ്പിക്കാൻ (`Ctrl+Shift+P` or `Cmd+Shift+P` on Mac) അമർത്തി, തുടർന്ന് `Hello World` എന്ന് ടൈപ് ചെയ്യുക.
* നിങ്ങളുടെ കോഡിനുള്ളിൽ `src/extension.ts`-ൽ ബ്രേക്ക്പോയിന്റുകൾ സജ്ജമാക്കി നിങ്ങളുടെ എക്സ്റ്റൻഷൻ ഡീബഗ് ചെയ്യുക.
* ഡീബഗ് കൺസോളിൽ നിന്നാണ് നിങ്ങളുടെ എക്സ്റ്റൻഷനിൽ നിന്നുള്ള ഔട്ട്പുട്ട് കണ്ടെത്തുക.

## മാറ്റങ്ങൾ ചെയ്യുക

* `src/extension.ts`-ൽ കോഡ് മാറ്റിയ ശേഷം ഡീബഗ് ടൂള്ബാറിൽ നിന്ന് എക്സ്റ്റൻഷൻ വീണ്ടും ലോഞ്ച് ചെയ്യാവുന്നതാണ്.
* നിങ്ങളുടെ മാറ്റങ്ങൾ ലോഡ് ചെയ്യാൻ VS Code വിൻഡോയും റീലോഡ് ചെയ്യാവുന്നതാണ് (`Ctrl+R` or `Cmd+R` on Mac).


## API പരിശോധിക്കുക

* ഫയൽ `node_modules/@types/vscode/index.d.ts` തുറന്നാൽ നിങ്ങൾ നമ്മുടെ API-യുടെ മുഴുവൻ സെറ്റ് തുറക്കാം.

## ടെസ്റ്റുകൾ പ്രവർത്തിപ്പിക്കുക

* [എക്സ്റ്റൻഷൻ ടെസ്റ്റ് റണ്ണർ](https://marketplace.visualstudio.com/items?itemName=ms-vscode.extension-test-runner) ഇൻസ്റ്റാൾ ചെയ്യുക
* "watch" ടാസ്‌ക്ക് **Tasks: Run Task** കമാൻഡിലൂടെ പ്രവർത്തിപ്പിക്കുക. ഇത് പ്രവർത്തനത്തിലാണ് എന്ന് ഉറപ്പാക്കുക, അല്ലെങ്കിൽ ടെസ്റ്റുകൾ കണ്ടെത്താൻ കഴിയില്ല.
* പ്രവർത്തന ബാറിൽ നിന്നു Testing view തുറന്ന് Run Test" ബട്ടൺ ക്ലിക്ക് ചെയ്യുക, അല്ലെങ്കിൽ ഹോട്ട്‌കീ ഉപയോഗിക്കുക `Ctrl/Cmd + ; A`
* Test Results view-ൽ ടെസ്റ്റ് ഫലത്തിന്റെ ഔട്ട്പുട്ട് കാണുക.
* `src/test/extension.test.ts`-ൽ മാറ്റങ്ങൾ ചെയ്യുകയോ `test` ഫോൾഡിനകത്ത് പുതിയ ടെസ്റ്റ് ഫയലുകൾ സൃഷ്ടിക്കുകയോ ചെയ്യുക.
  * നൽകിയിട്ടുള്ള ടെസ്റ്റ് റണ്ണർ പേര് പാറ്റേണുമായി പൊരുത്തപ്പെടുന്ന മാത്രമേ ഫയലുകൾ പരിഗണിക്കൂ `**.test.ts`.
  * നിങ്ങൾക്ക് ഇents`test` ഫോൾഡിനകത്ത് ഫോളഡറുകൾ സൃഷ്ടിച്ച് ടെസ്റ്റുകൾ ഇഷ്ടാനുസരണം ഘടിപ്പിക്കാം.

## കൂടുതൽ മുന്നോട്ടു പോവുക

* [നിങ്ങളുടെ എക്സ്റ്റൻഷൻ ബണ്ടിൽ ചെയ്യൽ](https://code.visualstudio.com/api/working-with-extensions/bundling-extension) വഴി എക്സ്റ്റൻഷന്റെ വലിപ്പം കുറച്ചപ്പോലും സ്റ്റാർട്ടപ്പ് സമയവും മെച്ചപ്പെടുത്തുക.
* VS Code എക്സ്റ്റൻഷൻ മാർക്കറ്റ്‌പ്ളേസിൽ നിങ്ങളുടെ എക്സ്റ്റൻഷൻ [പ്രസിദ്ധീകരിക്കുക](https://code.visualstudio.com/api/working-with-extensions/publishing-extension).
* [നിരന്തര സംയോജനം](https://code.visualstudio.com/api/working-with-extensions/continuous-integration) സജ്ജീകരിച്ച് ബിൽഡുകൾ സ്വയംചാലിക്കുക.
