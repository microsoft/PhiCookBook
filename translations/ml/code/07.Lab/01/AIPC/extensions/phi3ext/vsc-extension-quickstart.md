# നിങ്ങളുടെ VS Code എക്സ്റ്റൻഷനിലേക്കു സ്വാഗതം

## ഫോൾഡറിൽ എന്താണ് ഉള്ളത്

* ഈ ഫോൾഡറിൽ നിങ്ങളുടെ എക്സ്റ്റൻഷനുവേണ്ടിയുള്ള എല്ലാ ആവശ്യമായ ഫയലുകളും ഉൾപ്പെടുന്നു.
* `package.json` - ഇത് മാനിഫെസ്റ്റ് ഫയൽ ആണ്, ഇവിടെ നിങ്ങൾ നിങ്ങളുടെ എക്സ്റ്റൻഷൻയും കമാൻഡും പ്രഖ്യാപിക്കുന്നു.
  * സാമ്പിൾ പ്ലഗിൻ ഒരു കമാൻഡ് രജിസ്റ്റർ ചെയ്ത് അതിന്റെ ശീർഷകവും കമാൻഡ് നാമവും നിർവചിക്കുന്നു. ഈ വിവരങ്ങളോടെ VS Code കമാൻഡ് പാലറ്റിൽ ആ കമാൻഡ് കാണിക്കാനാകും. ഇതുവരെ പ്ലഗിൻ ലോഡ് ചെയ്യേണ്ട ആവശ്യം ഇല്ല.
* `src/extension.ts` - ഇത് നിങ്ങളുടെ കമാൻഡിന്റെ 구현ം നൽകുന്ന പ്രധാന ഫയൽ ആണ്.
  * ഫയൽ ഒരു ഫംഗ്ഷൻ `activate` എക്സ്പോർട്ട് ചെയ്യുന്നു, ഇത് നിങ്ങളുടെ എക്സ്റ്റൻഷൻ ആദ്യമായി ആക്ടിവേറ്റ് ചെയ്യുമ്പോൾ വിളിക്കപ്പെടുന്നു (ഈ സംഭവത്തിൽ കമാൻഡ് 실행ചെയ്യുന്നതിലൂടെ). `activate` ഫംഗ്ഷനിന്റെ اندر ഞങ്ങൾ `registerCommand` വിളിക്കുന്നു.
  * ഞങ്ങൾ കമാൻഡിന്റെ നടപ്പിലാക്കലുള്ള ഫംഗ്ഷൻ `registerCommand`-ഇന് രണ്ടാമത്തെ പാരാമീറ്ററായി നൽകുന്നു.

## ക്രമീകരണം

* ശിപാർശചെയ്ത എക്സ്റ്റൻഷനുകൾ ഇൻസ്റ്റാൾ ചെയ്യുക (amodio.tsl-problem-matcher, ms-vscode.extension-test-runner, and dbaeumer.vscode-eslint)


## ഉടനെ ആരംഭിക്കുക

* പുതിയ മെതൃവിൻഡോയിൽ നിങ്ങളുടെ എക്സ്റ്റൻഷൻ ലോഡ് ചെയ്യപ്പെട്ടതോടെ തുറക്കാൻ `F5` അടിക്കുക.
* (`Ctrl+Shift+P` അല്ലെങ്കിൽ Mac-ൽ `Cmd+Shift+P` അമർത്തി) കമാൻഡ് പാലറ്റിൽ നിന്ന് നിങ്ങളുടെ കമാൻഡ് 실행ചെയ്യാൻ `Hello World` ടൈപ്പ് ചെയ്യുക.
* നിങ്ങളുടെ എക്സ്റ്റൻഷൻ ഡീബഗ് ചെയ്യാൻ `src/extension.ts` എന്നിലുള്ള നിങ്ങളുടെ കോഡിൽ ബ്രേക്ക്‌പോയിന്റുകൾ സജ്ജമാക്കുക.
* ഡീബഗ് കോൺസോളിൽ നിന്നു നിങ്ങളുടെ എക്സ്റ്റൻഷനിൽ നിന്നുളള ഔട്ട്പുട്ട് കണ്ടെത്തുക.

## മാറ്റങ്ങൾ ചെയ്യുക

* `src/extension.ts`-ൽ കോഡ് മാറ്റിയതിന് ശേഷം ഡീബഗ് ടൂള്ബാറിൽ നിന്ന് എക്സ്റ്റൻഷൻ വീണ്ടും ആരംഭിക്കാം.
* നിങ്ങളുടെ മാറ്റങ്ങൾ ലോഡ് ചെയ്യാൻ എക്സ്റ്റൻഷൻ ഉള്ള VS Code വിൻഡോ റീലോഡ് ചെയ്യാൻ (`Ctrl+R` അല്ലെങ്കിൽ Mac-ൽ `Cmd+R`) സമർഥമാണ്.


## API പരിശോധിക്കുക

* `node_modules/@types/vscode/index.d.ts` ഫയൽ തുറക്കുമ്പോൾ ഞങ്ങളുടെ API-യുടെ മുഴുവൻ സെറ്റ് നിങ്ങൾക്ക് തുറക്കാം.

## ടെസ്റ്റുകൾ നടത്തുക

* [Extension Test Runner](https://marketplace.visualstudio.com/items?itemName=ms-vscode.extension-test-runner) ഇൻസ്റ്റാൾ ചെയ്യുക
* **Tasks: Run Task** കമാൻഡ് വഴി "watch" ടാസ്ക് 실행ചെയ്യുക. ഇത് പ്രവർത്തിച്ചുകൊണ്ടിരിക്കുമെന്ന് ഉറപ്പാക്കുക, അല്ലെങ്കിൽ ടെസ്റ്റുകൾ കണ്ടെത്തപ്പെടാതിരിക്കാൻ സാധ്യതയുണ്ട്.
* activity bar-ൽ നിന്നുള്ള Testing കാഴ്ച തുറന്ന് Run Test" ബട്ടൺ ക്ലിക്ക് ചെയ്യുക, അല്ലെങ്കിൽ ഹോട്ട്‌കീ `Ctrl/Cmd + ; A` ഉപയോഗിക്കുക
* Test Results കാഴ്ചയിൽ ടെസ്റ്റ് ഫലത്തിന്റെ ഔട്ട്പുട്ട് കാണുക.
* `src/test/extension.test.ts`-ൽ മാറ്റങ്ങൾ ചെയ്യുകയോ `test` ഫോൾഡറിനുള്ളിൽ പുതിയ ടെസ്റ്റ് ഫയലുകൾ സൃഷ്ടിക്കുകയോ ചെയ്യുക.
  * നൽകിയിട്ടുള്ള ടെസ്റ്റ് റണ്ണർ പേര് പാലിക്കുന്ന ഫയലുകൾ മാത്രം പരിഗണിക്കും: `**.test.ts`.
  * നിങ്ങളുടെ ടെസ്റ്റുകൾ ക്രമീകരിക്കാൻ `test` ഫോൾഡറിനുള്ളിൽ ഫോൾഡറുകൾ സൃഷ്ടിക്കാം.

## തുടരുക

* നിങ്ങളുടെ എക്സ്റ്റൻഷന്റെ വലിപ്പം കുറച്ചു സ്റ്റാർട്ടപ്പ് സമയം മെച്ചപ്പെടുത്താൻ [bundling your extension](https://code.visualstudio.com/api/working-with-extensions/bundling-extension?WT.mc_id=aiml-137032-kinfeylo) ഉപയോഗിക്കുക.
* VS Code എക്സ്റ്റൻഷൻ മാർക്കറ്റ്പ്ലേസിൽ നിങ്ങളുടെ എക്സ്റ്റൻഷൻ [Publish your extension](https://code.visualstudio.com/api/working-with-extensions/publishing-extension?WT.mc_id=aiml-137032-kinfeylo) ചെയ്യുക.
* [Continuous Integration](https://code.visualstudio.com/api/working-with-extensions/continuous-integration?WT.mc_id=aiml-137032-kinfeylo) സെറ്റ് ചെയ്ത് ബിൽഡുകൾ аўടോമേറ്റ് ചെയ്യുക.
