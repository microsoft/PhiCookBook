# നിങ്ങളുടെ VS Code എക്സ്റ്റൻഷനിലേക്ക് സ്വാഗതം

## ഫോൾഡറിൽ എന്തുണ്ട്

* ഈ ഫോൾഡറിൽ നിങ്ങൾගේ എക്സ്റ്റൻഷനൊപ്പം ആവശ്യമായ എല്ലാ ഫയലുകളും അടങ്ങിയിട്ടുണ്ട്.
* `package.json` - നിങ്ങളുടെ എക്സ്റ്റൻഷനും കമാൻഡും പ്രഖ്യാപിക്കുന്ന മാനിഫസ്റ്റ് ഫയൽ.
  * സാമ്പിൾ പ്ലഗിൻ ഒരു കമാൻഡ് രജിസ്റ്റർ ചെയ്ത് അതിന്റെ തലക്കെട്ടും കമാൻഡ് നാമവും നിർവചിക്കുന്നു. ഈ വിവരത്തോടെ VS Code കമാൻഡ് പാലറ്റിൽ ആ കമാൻഡ് കാണിക്കാമെന്നു നിർവഹിക്കും. ഇപ്പോൾ പ്ലഗിൻ ലോഡ് ചെയ്യേണ്ടതില്ല.
* `src/extension.ts` - നിങ്ങളുടെ കമാൻഡിന്റെ നിർവഹണം ഇവിടെ നൽകിയിരിക്കും.
  * ഫയൽ ഒരു ഫംഗ്ഷൻ `activate` എക്സ്പോർട്ട് ചെയ്യുന്നു, ഇത് നിങ്ങളുടെ എക്സ്റ്റൻഷൻ ആദ്യമായി സജീവമാകുമ്പോളാണ് വിളിക്കപ്പെടുന്നത് (ഇപ്പോൾ ഉദാഹരണത്തിൽ കമാൻഡ് നിർവ്വഹിച്ചതിലൂടെ). `activate` ഫംഗ്ഷനിലുള്ളതിനകം ഞങ്ങൾ `registerCommand` വിളിക്കുന്നു.
  * `registerCommand`-ന് രണ്ടാം പാരാമീറ്ററായി കമാൻഡിന്റെ നിർവഹണം അടങ്ങിയ ഫംഗ്ഷൻ നാം പാസ്സ് ചെയ്യുന്നു.

## സജ്ജീകരണം

* ശിപാർശ ചെയ്ത എക്സ്റ്റൻഷനുകൾ ഇൻസ്റ്റാൾ ചെയ്യുക (amodio.tsl-problem-matcher, ms-vscode.extension-test-runner, and dbaeumer.vscode-eslint)


## ഉടൻ തന്നെ ആരംഭിക്കുക

* നിങ്ങളുടെ എക്സ്റ്റൻഷൻ ലോഡ് ചെയ്ത പുതിയ ഒരു വിൻഡോ തുറക്കാൻ `F5` അമർത്തുക.
* (`Ctrl+Shift+P` അല്ലെങ്കിൽ Mac-ൽ `Cmd+Shift+P`) അമർത്തി കമാൻഡ് പാലറ്റിൽ നിന്ന് `Hello World` ടൈപ്പ് ചെയ്ത് നിങ്ങളുടെ കമാൻഡ് പ്രവർത്തിപ്പിക്കുക.
* നിങ്ങളുടെ എക്സ്റ്റൻഷൻ ഡീബഗ് ചെയ്യാൻ `src/extension.ts` ഉള്ളതിൽ ബ്രേക്പോയിന്റുകൾ സജ്ജമാക്കുക.
* ഡീബഗ്ഗിംഗ് കോൺസോളിൽ നിന്ന് നിങ്ങളുടെ എക്സ്റ്റൻഷന്റെ ഔട്ട്പുട്ട് കണ്ടെത്തുക.

## മാറ്റങ്ങൾ ചെയ്യുക

* `src/extension.ts`-ൽ കോഡ് മാറ്റിയ ശേഷം ഡീബഗ് ടൂൾബാർവഴിയോ അതവസരം പുറത്ത് വീണ്ടും ലോഞ്ച് ചെയ്തുകൊണ്ട് എക്സ്റ്റൻഷൻ റീലോഞ്ച് ചെയ്യാം.
* നിങ്ങളുടെ മാറ്റങ്ങൾ ലോഡ് ചെയ്യാൻ എക്സ്റ്റൻഷൻ ഉള്ള സ്ഥിതിയിൽ VS Code വിംഡോ റീലോഡ് ചെയ്യാൻ (`Ctrl+R` അല്ലെങ്കിൽ Mac-ൽ `Cmd+R`) കഴിയും.


## API പഠിക്കുക

* ഞങ്ങളുടെ മുഴുവൻ API ഫയൽ `node_modules/@types/vscode/index.d.ts` തുറന്ന് കാണാം.

## ടെസ്റ്റുകൾ chạy ചെയ്യുക

* [Extension Test Runner](https://marketplace.visualstudio.com/items?itemName=ms-vscode.extension-test-runner) ഇൻസ്റ്റാൾ ചെയ്യുക
* **Tasks: Run Task** കമാൻഡ് വഴി "watch" ടാസ്‌ക് പ്രവർത്തിപ്പിക്കുക. ഇത് ഓടിക്കൊണ്ടിരിക്കണം, അല്ലെങ്കിൽ ടെസ്റ്റുകൾ കണ്ടുപിടിക്കപ്പെടാതിരിക്കാം.
* ആക്ടിവിറ്റി ബാറിൽ നിന്ന് Testing ദൃശ്യം തുറന്ന് Run Test" ബട്ടൺ ക്ലിക്ക് ചെയ്യുക, അല്ലെങ്കിൽ ഹോട്ട്‌കീ `Ctrl/Cmd + ; A` ഉപയോഗിക്കുക
* Test Results view-ൽ ടെസ്റ്റ് ഫലത്തിന്റെ ഔട്ട്പുട്ട് കാണുക.
* `src/test/extension.test.ts` മാറ്റങ്ങൾ ചെയ്യുക അല്ലെങ്കിൽ `test` ഫോൾഡറിനുള്ളിൽ പുതിയ ടെസ്റ്റ് ഫയലുകൾ സൃഷ്ടിക്കുക.
  * നൽകിയിട്ടുള്ള ടെസ്റ്റ് റണ്ണർ പേരിന്റെ പാറ്റേണിന് phùയാത്ത ഫയലുകൾ മാത്രമേ പരിഗണിക്കൂ `**.test.ts`.
  * നിങ്ങളുടെ ടെസ്റ്റുകൾ എങ്ങിനെ വേണമെങ്കിലുമെങ്കിൽ ഘടനപ്പെടുത്താൻ `test` ഫോൾഡറിനുള്ളിൽ ഫോൾഡറുകൾ സൃഷ്ടിക്കാം.

## കൂടുതൽ ചെയ്യാൻ

* നിങ്ങളുടെ എക്സ്റ്റൻഷന്റെ വലുപ്പം കുറയ്ക്കാനും സ്റ്റാർട്ടപ് സമയം മെച്ചപ്പെടുത്താനും [bundling your extension](https://code.visualstudio.com/api/working-with-extensions/bundling-extension) ഉപയോഗിക്കുക.
* VS Code എക്സ്റ്റൻഷൻ മാർക്കെറ്റ്പ്ലേസിൽ നിങ്ങളുടെ എക്സ്റ്റൻഷൻ [Publish your extension](https://code.visualstudio.com/api/working-with-extensions/publishing-extension) ചെയ്യുക.
* [Continuous Integration](https://code.visualstudio.com/api/working-with-extensions/continuous-integration) സജ്ജമാക്കി ബിൽഡുകൾ ഓട്ടോമേറ്റ് ചെയ്യുക.
