# സംഭാവനകൾ

ഈ പ്രോജക്ട് സംഭാവനകളും നിർദ്ദേശങ്ങളും സ്വാഗതം ചെയ്യുന്നു. മിക്ക സംഭാവനകൾക്കും നിങ്ങൾക്കുള്ള അവകാശം ഉള്ളതും അതിന്റെ അടിസ്ഥാനത്തിൽ ഞങ്ങൾക്ക് നിങ്ങളുടെ സംഭാവന ഉപയോഗപ്പെടുത്താനുള്ള അവകാശം നിങ്ങൾ അനുവദിക്കുന്നുണ്ടെന്ന് പ്രഖ്യാപിക്കുന്ന Contributor License Agreement (CLA) ഒപ്പിടാൻ നിങ്ങൾക്ക് സമ്മതിക്കേണ്ടതുണ്ട്. അധികവിവരങ്ങൾക്ക്, കാണുക [https://cla.opensource.microsoft.com](https://cla.opensource.microsoft.com)

നിങ്ങൾ ഒരു pull request സമർപ്പിക്കുമ്പോൾ, CLA ബോട്ട് സ്വയം നിശ്ചയിക്കുന്നു നിങ്ങൾക്ക് CLA നൽകേണ്ടതുണ്ടോ എന്നതും PR യഥാർത്ഥത്തിൽ ശരിയായ വിധത്തിൽ അടയാളപ്പെടുത്തപ്പെടുകയുമാണ് (ഉദാ., status check, comment). ബോട്ട് നൽകുന്ന നിർദ്ദേശങ്ങൾ പിന്തുടരുക. ഞങ്ങളുടെ CLA ഉപയോഗിക്കുന്ന എല്ലാ റിപോസിറ്ററികളിലും ഇത് ഒറ്റപ്പെട്ടൊരു തവണ മാത്രമേ ചെയ്യേണ്ടതുള്ളൂ.

## പെരുമാറ്റ നയം

ഈ പ്രോജക്ട് ([Microsoft Open Source Code of Conduct](https://opensource.microsoft.com/codeofconduct/)) സ്വീകരിച്ചിട്ടുണ്ട്. കൂടുതൽ വിവരങ്ങൾക്ക് [Code of Conduct FAQ](https://opensource.microsoft.com/codeofconduct/faq/) വായിക്കുക അല്ലെങ്കിൽ അധിക ചോദ്യങ്ങൾക്കോ അഭിപ്രായങ്ങൾക്കോ [opencode@microsoft.com](mailto:opencode@microsoft.com) ൽ ബന്ധപ്പെടുക.

## പ്രശ്നങ്ങൾ സൃഷ്ടിക്കുമ്പോൾ ജാഗ്രത

സാധാരണ സപ്പോർട്ട് ചോദ്യംകളിൽ GitHub issues തുറക്കരുത്, കാരണം GitHub ലിസ്റ്റ് ഫീച്ചർ അഭ്യർത്ഥനകൾക്കും ബഗ് റിപ്പോർട്ടുകൾക്കും ഉപയോഗിക്കേണ്ടതാണ്. ഇതിലൂടെ കോഡിൽ നിന്നുള്ള സത്യമാർന്ന പ്രശ്നങ്ങളെ അല്ലെങ്കിൽ ബഗുകൾ കണ്ടെത്താനും സാധിക്കും, കൂടിയുള്ള പൊതുചർച്ചയെ യഥാർത്ഥ കോഡിൽ നിന്ന് വേർതിരിച്ചുമാറ്റാൻ കഴിയും.

## സംഭാവന ചെയ്യാനുള്ള മാർഗങ്ങൾ

### പുൾ റിക്വസ്റ്റ് മാർഗ്ഗനിർദ്ദേശങ്ങൾ

Phi-3 CookBook റിപോസിറ്ററിയിലേക്ക് ഒരു പുൾ റിക്വസ്റ്റ് (PR) സമർപ്പിക്കുമ്പോൾ, ദയവായി താഴെപ്പറയുന്ന മാർഗ്ഗനിർദ്ദേശങ്ങൾ പാലിക്കുക:

- **റിപ്പോസിറ്ററി Fork ചെയ്യുക**: നിങ്ങളുടെ മാറ്റങ്ങൾ ചെയ്യുന്നതിന് മുമ്പ് എപ്പോഴും റിപോസിറ്ററി നിങ്ങളുടെ അക്കൗണ്ടിലേക്ക് fork ചെയ്യുക.

- **വിവിധ pull requests (PR)**:
  - ഓരോ മാറ്റ തരംതാനുള്ള മാറ്റവും പ്രത്യേകം ഒരു pull request ആയി സമർപ്പിക്കുക. ഉദാഹരണത്തിന്, ബഗ് തിരുത്തലുകളും ഡോക്യുമെന്റേഷൻ അപ്ഡേറ്റുകളും വേർതിരിച്ചുള്ള PRകളായി സമർപ്പിക്കേണ്ടതാണ്.
  - ടൈപ്പോ തകർക്കലുകളും ചെറിയ ഡോക്യുമെന്റേഷൻ അപ്ഡേറ്റുകളും ബാധകമായ പക്ഷം ഒരുเดียวുള്ള PRയിൽ സംയോജിപ്പിക്കാൻ സാധിക്കും.

- **merge conflicts കൈകാര്യം ചെയ്യുക**: നിങ്ങളുടെ pull request merge conflicts കാണിച്ചാൽ, മാറ്റങ്ങൾ ചെയ്യാൻ മുമ്പ് നിങ്ങളുടെ ലോക്കൽ `main` ബ്രാഞ്ച് പ്രധാന റിപോസിറ്ററിയുമായി സിംഗൺ ചെയ്യണം.

- **ഭാഷാനുഭാഗം സമർപ്പണങ്ങൾ**: ഒരു ഭാഷാനുലിപ്പിക്കൽ PR സമർപ്പിക്കുമ്പോൾ,_original_ ഫോൾഡറിലുള്ള എല്ലാ ഫയലുകൾക്കും വിവർത്തനങ്ങൾ ഉൾപ്പെടുന്നുവെന്ന് ഉറപ്പാക്കുക.

### എഴുതാനുള്ള മാർഗ്ഗനിർദ്ദേശങ്ങൾ

എല്ലാ പ്രമാണങ്ങളിലേയും ഏകക്ഷമത ഉറപ്പാക്കുന്നതിനായി, ദയവായി താഴെപ്പറയുന്ന മാർഗ്ഗനിർദ്ദേശങ്ങൾ ഉപയോഗിക്കുക:

- **URL രൂപീകരണം**: എല്ലാ URLകളും ചതുരക്കൽക്കൂട്ടിൽ ഇട്ടതിന്റെ പിന്നാലെ ബ്രാക്കറ്റുകൾ ഉപയോഗിച്ച് പൂർത്തിയാക്കുക, അവരുടെ ചുറ്റും അല്ലെങ്കിൽ ഉള്ളിൽ അധിക സ്പേസുകൾ കൂടാതെ. ഉദാഹരണത്തിന്: `[example](https://www.microsoft.com)`.

- **Relative links**: സുഷ്രൂഷാ ഡയറക്ടറിയിലുള്ള ഫയലുകൾക്കും ഫോൾഡറുകൾക്കും സാപേക്ഷ ലിങ്കുകൾക്കായി `./` ഉപയോഗിക്കുക, മുകളിലുള്ള ഡയറക്ടറിയിലുള്ളവയ്ക്ക് `../` ഉപയോഗിക്കുക. ഉദാഹരണത്തിന്: `[example](../../path/to/file)` അല്ലെങ്കിൽ `[example](../../../path/to/file)`.

- **Not Country-Specific locales**: നിങ്ങളുടെ ലിങ്കുകളിൽ രാജ്യ-നിർദിഷ്ട locale ഉൾപ്പെടാത്തതായി ഉറപ്പാക്കുക. ഉദാഹരണത്തിന്, `/en-us/` അല്ലെങ്കിൽ `/en/` ഒഴിവാക്കുക.

- **Image storage**: എല്ലാം ചിത്രങ്ങൾ `./imgs` ഫോൾഡറിൽ സൂക്ഷിക്കുക.

- **Descriptive image names**: ചിത്രങ്ങൾക്ക് ഇംഗ്ലീഷ് അക്ഷരങ്ങൾ, സംഖ്യകൾ, ഡാഷ് എന്നിവ ഉപയോഗിച്ച് വിവരണാത്മകങ്ങളായ പേര് നൽകുക. ഉദാഹരണത്തിന്: `example-image.jpg`.

## GitHub പ്രവാഹങ്ങൾ

നിങ്ങൾ ഒരു pull request സമർപ്പിക്കുമ്പോൾ, മാറ്റങ്ങൾ സ്ഥിരീകരിക്കാൻ താഴെയുള്ള workflows പ്രവർത്തിപ്പിക്കും. നിങ്ങളുടെ pull request workflow പരിശോധകൾ കടക്കുന്നതിന് താഴെപ്പറയുന്ന നിർദ്ദേശങ്ങൾ പാലിക്കുക:

- [തെറ്റായ സാപেক্ষ പാതകൾ പരിശോധിക്കുക](../..)
- [URL-കളിൽ Locale ഇല്ലെന്ന് പരിശോധിക്കുക](../..)

### Check Broken Relative Paths

ഈ workflow നിങ്ങളുടെ ഫയലുകളിലെ എല്ലാ സാപേക്ഷ പാതകളും ശരിയാണെന്ന് ഉറപ്പാക്കാൻ സഹായിക്കുന്നു.

1. To make sure your links are working properly, perform the following tasks using VS Code:
    - നിങ്ങളുടെ ഫയലുകളിൽ ഏതെങ്കിലും ഒരു ലിങ്കിന്റെ മീതേ hover ചെയ്യുക.
    - ലിങ്കിലേക്ക് നാവിഗേറ്റ് ചെയ്യാൻ **Ctrl + Click** ഞെക്കുക.
    - നിങ്ങൾ ഒരു ലിങ്കിൽ ക്ലിക്ക് ചെയ്ത് അത് ലൊക്കലായി പ്രവർത്തിക്കുകയില്ലെങ്കിൽ, അത് workflow ട്രിഗർ ചെയ്ത് GitHub-ലും പ്രവർത്തിക്കില്ല.

1. To fix this issue, perform the following tasks using the path suggestions provided by VS Code:
    - ടൈപ്പ് ചെയ്യുക `./` അല്ലെങ്കിൽ `../`.
    - നിങ്ങൾ ടൈപ്പ് ചെയ്തതിന്റെ അടിസ്ഥാനത്തിൽ ലഭ്യമായ ഓപ്ഷനുകളിൽ നിന്ന് തിരഞ്ഞെടുക്കാൻ VS Code നിങ്ങളെ പ്രോംപ്റ്റ് ചെയ്യും.
    - പാത ശരിയാണെന്ന് ഉറപ്പുവരുത്താൻ ആഗ്രഹിച്ച ഫയലോ ഫോൾഡറോ ക്ലിക്ക് ചെയ്ത് പാത പിന്തുടരുക.

Once you have added the correct relative path, save and push your changes.

### Check URLs Don't Have Locale

ഈ workflow ഏതെങ്കിലും വെബ്ബ് URL-ലും രാജ്യ-നിർദിഷ്ട locale ഉൾക്കൊള്ളുന്നുണ്ടോ എന്ന് പരിശോധിക്കുന്നു. ഈ റിപോസിറ്ററി ആഗോളമായി ആക്‌സസ് ചെയ്യാവുന്നതാണ്, അതിനാൽ URL-കളിൽ നിങ്ങളുടെ രാജ്യത്തെ locale ഉണ്ടാകരുതെന്ന് ഉറപ്പുവരുത്തുന്നത് പ്രധാനമാണ്.

1. To verify that your URLs don't have country locales, perform the following tasks:

    - URLs-ൽ `/en-us/`, `/en/` പോലുള്ള ടെക്സ്റ്റുകൾ അടങ്ങിയാണോയെന്ന് പരിശോധിക്കുക.
    - ഇവ നിങ്ങളുടെ URL-കളിൽ ഉണ്ടാകാത്തെങ്കിൽ, നിങ്ങൾ ഈ പരിശോധന ഓഫീസിൽ കടക്കും.

1. To fix this issue, perform the following tasks:
    - workflow ഹൈലൈറ്റ് ചെയ്ത ഫയൽ പാത്ത് തുറക്കുക.
    - URL-കളിൽ നിന്നു രാജ്യ locale നീക്കം ചെയ്യുക.

Once you remove the country locale, save and push your changes.

### Check Broken Urls

ഈ workflow നിങ്ങളുടെ ഫയലുകളിലെ ഏതെങ്കിലും വെബ്ബ് URL പ്രവർത്തിക്കുന്നുണ്ടോ എന്നുണർവുകയും 200 സ്റ്റാറ്റസ് കോഡ് തിരിച്ച് നൽകുന്നുണ്ടോ എന്നും ഉറപ്പാക്കുന്നു.

1. To verify that your URLs are working correctly, perform the following tasks:
    - നിങ്ങളുടെ ഫയലുകളിലുള്ള URL-കളുടെ സ്റ്റാറ്റസ് പരിശോധിക്കുക.

2. To fix any broken URLs, perform the following tasks:
    - തകരാറായ URL ഉൾക്കൊള്ളുന്ന ഫയൽ തുറക്കുക.
    - URL ശരിയായതാക്കിയിടുക.

Once you have fixed the URLs, save and push your changes.

> [!NOTE]
>
> URL പരിശോധന പരാജയപ്പെടുന്നതിനും ലിങ്ക് ആക്സസ്ബിളായിട്ടുണ്ടെങ്കിലും ചില സാഹചര്യങ്ങൾ 있을 수 있습니다. ഇതിന് കാരണങ്ങളായി ചിലത് ഉൾപ്പെടാം:
>
> - **Network restrictions:** GitHub actions സർവറുകൾക്ക് ചില URL-കൾക്ക് ആക്സസ് തടയുന്ന നെറ്റ്‌വർക്ക് നിയന്ത്രണങ്ങൾ ഉണ്ടായേക്കാം.
> - **Timeout issues:** പ്രതികരിക്കാൻ വളരെ സമയം എടുക്കുന്ന URL-കൾ workflow-യിൽ timeout പിഴവ് ട്രിഗർ ചെയ്യാം.
> - **Temporary server issues:** സമയികമായ സെർവർ ഔട്ട്‍റേജ് അല്ലെങ്കിൽ നിലനിൽപ്പുസംരക്ഷണ പ്രവർത്തനങ്ങൾ പരിശോധന സമയത്ത് URL താൽക്കാലികമായി ലഭ്യമല്ലാതാക്കാം.
