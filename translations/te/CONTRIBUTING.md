# సహకారం

ఈ ప్రాజెక్ట్ సహకారాలు మరియు సూచనలను స్వాగతిస్తుంది. చాలా సహకారాలను మీరు మీ సహకారాన్ని ఉపయోగించడానికి మాకు హక్కులను ఇచ్చే హక్కు మీకు ఉన్నదని, మరియు మీరు నిజంగా ఆ హక్కులను మాకు ఇస్తున్నారని ప్రకటించే Contributor License Agreement (CLA) కు అంగీకరించేటటుగా చేయాలని అవసరం ఉంటుంది. వివరాలకు, చూడండి [https://cla.opensource.microsoft.com](https://cla.opensource.microsoft.com)

మీరు ఒక pull request సమర్పించినప్పుడు, CLA బాట్ ఆటోమేటిక్‌గా మీరు CLA అందించవలసిన అవసరమున్నదో లేదో నిర్ణయించి PRని అనుకూలంగా సంకేతం చేస్తుంది (ఉదాహరణకు, స్థితి పరీక్ష, వ్యాఖ్య). బాట్ ఇచ్చే సూచనలను అనుసరించండి. మా CLAని ఉపయోగిస్తున్న అన్ని రిపోస్‌లలో చూడవలసిన పని ఇది ఒక్కసారి మాత్రమే ఉంటుంది.

## ఆచరణ నియమాలు

ఈ ప్రాజెక్ట్ [Microsoft Open Source Code of Conduct](https://opensource.microsoft.com/codeofconduct/)ను దత్తత చేసుకుంది. మరిన్ని వివరాల కోసం [Code of Conduct FAQ](https://opensource.microsoft.com/codeofconduct/faq/) ను చదవండి లేదా అదనపు ప్రశ్నలు లేదా వ్యాఖ్యల కోసం [opencode@microsoft.com](mailto:opencode@microsoft.com) ను సంప్రదించండి.

## ఇష్యూలు సృష్టించేటప్పుడు జాగ్రత్తలు

క్రొత్త ఫీచర్ అభ్యర్థనలు మరియు బగ్ నివేదికల కోసం GitHub ఇష్యూలు ఉపయోగించాలి, సాధారణ సపోర్ట్ ప్రశ్నల కోసం GitHub ఇష్యూలు తెరవద్దని దయచేసి గమనించండి. ఈ విధంగా కోడ్ నుండి జరిగిన వాస్తవ ఇష్యూలను లేదా బగ్‌లను మేము సులభంగా ట్రాక్ చేయగలము మరియు సాధారణ చర్చలను వాస్తవ కోడ్ నుండి వేరుగా ఉంచగలము.

## ఎలా సహకరించాలి

### పుల్ రిక్వెస్ట్‌ల మార్గదర్శకాలు

Phi-3 CookBook రిపోజిటరీకి pull request (PR) సమర్పించినప్పుడు, దయచేసి క్రింది మార్గదర్శకాలను ఉపయోగించండి:

- **Fork the Repository**: మీ మార్పులు చేయడానికి ముందు ఎప్పుడూ రిపోజిటరీని మీ స్వంత ఖాతాకు ఫోర్క్ చేయండి.

- **Separate pull requests (PR)**:
  - ప్రతి రకం మార్పును దాని స్వంత pull request లో సమర్పించండి. ఉదాహరణకు, బగ్ ఫిక్సులు మరియు డాక్యుమెంటేషన్ నవీకరణలు వేర్వేరు PRలలో సమర్పించాలి.
  - టైపోల నిర్వహణలు మరియు చిన్న డాక్యుమెంటేషన్ అప్డేట్లు అవసరమైతే ఒకే PRలో కలపవచ్చు.

- **Handle merge conflicts**: మీ pull requestలో మర్జ్ కంప్లిక్ట్స్ కనబడితే, మార్పులు చేయక ముందు మీ స్థానిక `main` బ్రాంచ్‌ను ప్రధాన రిపోజిటరీని ప్రతిబింబించేలా अद्यतन చేయండి.

- **Translation submissions**: అనువాద PR సమర్పించినప్పుడు, అనువాద ఫోల్డర్‌లో ఆ ఒరిజినల్ ఫోల్డర్‌లో ఉన్న అన్ని ఫైళ్లకు అనువాదాలు ఉన్నాయో లేదో నిర్ధారించండి.

### రచనా మార్గదర్శకాలు

అన్ని డాక్యుమెంట్లలో అనుసరించదగిన తటస్థత్వాన్ని నిర్ధారించేందుకు, దయచేసి క్రింది మార్గదర్శకాలను అనుసరించండి:

- **URL formatting**: అన్ని URLsను చదరపు కోట్స్‌లో వ్రాసి వెంటనే స్థూలకోనాలు వేయండి, వాటి చుట్టూ లేదా లోపల ఎలాంటి అదనపు ఖాళీలు లేకుండా. ఉదాహరణకు: `[example](https://www.microsoft.com)`.

- **Relative links**: మొత్తం డైరక్టరీలోని ఫైళ్లు లేదా ఫోల్డర్లకు సూచించే సాపేక్ష లింక్లకు `./` ను, పేరెంట్ డైరక్టరీలో ఉన్నవారికి `../` ను ఉపయోగించండి. ఉదాహరణకు: `[example](../../path/to/file)` లేదా `[example](../../../path/to/file)`.

- **Not Country-Specific locales**: మీ లింకుల్లో దేశ-నిర్దిష్ట లోకేల్‌లు ఉండవద్దని నిర్ధారించండి. ఉదాహరణకు, `/en-us/` లేదా `/en/` వాడరద్దు.

- **Image storage**: అన్ని చిత్రాలను `./imgs` ఫోల్డర్లో నిల్వ చేయండి.

- **Descriptive image names**: చిత్రాలను వివరణాత్మకంగా పేరు పెట్టండి, ఇంగ్లీష్ అక్షరాలు, సంఖ్యలు మరియు డాష్‌లను ఉపయోగించండి. ఉదాహరణకు: `example-image.jpg`.

## GitHub వర్క్‌ఫ్లోలు

మీరు ఒక pull request సమర్పించినప్పుడు, మార్పులను పరిశీలించడానికి క్రింది వర్క్‌ఫ్లోలు ట్రిగ్గర్ అవుతాయి. మీ pull request వర్క్‌ఫ్లో చెక్లను విజయవంతంగా గడిపేందుకు క్రింద ఇచ్చిన సూచనలను అనుసరించండి:

- [Check Broken Relative Paths](../..)
- [Check URLs Don't Have Locale](../..)

### Check Broken Relative Paths

ఈ వర్క్‌ఫ్లో మీ ఫైళ్లలోని అన్ని సాపేక్ష మార్గాలు సరికొత్తవా అనే విషయాన్ని నిర్ధారిస్తుంది.

1. మీ లింకులు సరిగా పని చేస్తున్నాయా అని చూసుకోవడానికి, VS Code ఉపయోగించి క్రింద పేర్కొన్న పనులు చేయండి:
    - మీ ఫైళ్లలో ఏదైనా లింక్‌పై హోవర్ చేయండి.
    - లింక్‌కి వెళ్లడానికి **Ctrl + Click** నొక్కండి.
    - మీరు లింక్‌పై క్లిక్ చేసి అది మీ స్థానికంగా పనిచేయకపోతే, అది వర్క్‌ఫ్లోను ట్రిగ్గర్ చేస్తుంది మరియు GitHub లో కూడా పని చేయదు.

1. ఈ సమస్యను పరిష్కరించడానికి, VS Code ఇచ్చే పాత్ సూచనలను ఉపయోగించి క్రింది పనులను చేయండి:
    - `./` లేదా `../` టైప్ చేయండి.
    - మీరు టైప్ చేసిన ఆధారంగా అందుబాటులో ఉన్న ఎంపికల నుంచి ఎంచుకునేందుకు VS Code ప్రాంప్ట్ చేస్తుంది.
    - మీ పాత్ సరైనదిగా ఉండాలని నిర్ధారించడానికి కావలసిన ఫైల్ లేదా ఫోల్డర్‌పై క్లిక్ చేసి మార్గాన్ని అనుసరించండి.

సరైన సాపేక్ష మార్గాన్ని జోడించిన తర్వాత, మీ మార్పులను సేవ్ చేసి పుష్ చేయండి.

### Check URLs Don't Have Locale

ఈ వర్క్‌ఫ్లో వెబ్ URLలో ఏ దేశ-నిర్దిష్ట లోకేల్ ఉన్నదో లేదో నిర్ధారిస్తుంది. ఈ రిపోజిటరీ అంతర్జాతీయంగా 접근ించదగినందున, URLలు మీ దేశం-నిర్దిష్టలోకేల్‌ను కలిగి ఉండకూడదని నిర్ధారించడం ముఖ్యము.

1. మీ URLల్లో దేశ-లోకేల్‌లు లేవో లేదో నిర్థారించడానికి క్రింది పనులను చేయండి:

    - URLలలో `/en-us/`, `/en/` లేదా ఇతర ఏవైనా భాషా లోకేల్‌లను వలె ఉన్న టెక్స్ట్ కోసం చెక్ చేయండి.
    - ఇవి మీ URLల్లో కనిపించకపోతే, మీరు ఈ చెకును పాస్ అవుతారు.

1. ఈ సమస్యను పరిష్కరించడానికి, క్రింద సూచించిన పనులు చేయండి:
    - వర్క్‌ఫ్లో హైలైట్ చేసిన ఫైల్ పాత్‌ను తెరవండి.
    - URLల నుండి దేశ లోకేల్‌ను తీసివేయండి.

దేశ లోకేల్ తీసివేసిన తర్వాత, మీ మార్పులను సేవ్ చేసి పుష్ చేయండి.

### Check Broken Urls

ఈ వర్క్‌ఫ్లో మీ ఫైళ్లలోని వెబ్ URL ఏవైనా పనిచేస్తున్నాయా మరియు 200 స్థితి కోడ్‌ను రిటర్న్ చేస్తున్నాయా అని నిర్ధారిస్తుంది.

1. మీ URLలు సరైన రీతిన పనిచేస్తున్నాయా అని తనిఖీ చేసుకోవడానికి క్రింది పనులను చేయండి:
    - మీ ఫైళ్లలోని URLల స్థితిని చెక్ చేయండి.

2. పనిచేయని ఏవైనా URLలను సరిచేయడానికి, క్రింది పనులను చేయండి:
    - పనిచేయని URL ఉన్న ఫైల్‌ను తెరవండి.
    - URLను సరైనదిగా అప్డేట్ చేయండి.

URLలను సరిచేసిన తర్వాత, మీ మార్పులను సేవ్ చేసి పుష్ చేయండి.

> [!NOTE]
>
> URL తనిఖీ విఫలమయ్యే సందర్భాలు ఉండొచ్చు అయినప్పటికీ లింక్ ప్రాప్యత ఉన్నట్లే కనిపించవచ్చు. ఇది అనేక కారణాల వల్ల జరుగుతుంది, వాటిలో కొన్ని:
>
> - **Network restrictions:** GitHub actions సర్వర్లు కొన్ని URLల యాక్సెస్‌ను నిరోధించే నెట్‌వర్క్ పరిమితుల వల్ల ప్రభావితం కావచ్చు.
> - **Timeout issues:** తప్పు సమయానికి స్పందన ఇచ్చే URLలు వర్క్‌ఫ్లోలో టైమౌట్ లోపాన్ని ట్రిగ్గర్ చేయవచ్చు.
> - **Temporary server issues:** కొన్నిసార్లు సర్వర్ డౌన్ టైమ్ లేదా నిర్వహణ కారణంగా వెరిఫికేషన్ సమయంలో URL తాత్కాలికంగా అందుబాటులో లేకపోవచ్చు.
