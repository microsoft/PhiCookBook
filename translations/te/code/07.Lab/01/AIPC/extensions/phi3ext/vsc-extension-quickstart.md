# మీ VS Code ఎక్స్‌టెన్షన్‌కు స్వాగతం

## ఫోల్డర్‌లో ఏముంది

* ఈ ఫోల్డర్‌లో మీ ఎక్స్‌టెన్షన్ కోసం అవసరమైన అన్ని ఫైళ్లు ఉన్నాయి.
* `package.json` - ఇది మెనిఫెస్ట్ ఫైల్, ఇందులో మీరు మీ ఎక్స్‌టెన్షన్ మరియు కమాండ్‌ను ప్రకటిస్తారు.
  * ఉదాహరణ ప్లగిన్ ఒక కమాండ్‌ను రిజిస్టర్ చేయడంతో పాటు దాని శీర్షిక మరియు కమాండ్ పేరును నిర్వచిస్తుంది. ఈ సమాచారంతో VS Code ఆ కమాండ్‌ను కమాండ్ ప్యాలెట్‌లో చూపించగలదు. ప్లగిన్‌ను ఇప్పుడే లోడ్ చేయాల్సిన అవసరం లేదు.
* `src/extension.ts` - ఇది ప్రధాన ఫైల్, ఇక్కడ మీరు మీ కమాండ్ యొక్క అమలును అందిస్తారు.
  * ఫైల్ ఒక ఫంక్షన్‌ను ఎక్స్‌పోర్ట్ చేస్తుంది, `activate`, ఇది మీ ఎక్స్‌టెన్షన్ మొదటిసారి యాక్టివేట్ అయినప్పుడు (ఈ సందర్భంలో కమాండ్‌ను ఎగ్జిక్యూట్ చేసినప్పుడు) పిలవబడుతుంది. `activate` ఫంక్షన్‌లో మేము `registerCommand` ను కాల్ చేస్తాము.
  * కమాండ్ యొక్క అమలును కలిగివున్న ఫంక్షన్‌ను `registerCommand` కు రెండవ పరామితిగా పంపిస్తాం.

## సెటప్

* సూచించిన ఎక్స్‌టెన్షన్లను ఇన్‌స్టాల్ చేయండి (amodio.tsl-problem-matcher, ms-vscode.extension-test-runner, మరియు dbaeumer.vscode-eslint)


## వెంటనే ప్రారంభించండి

* మీ ఎక్స్‌టెన్షన్ లోడ్ అయిన కొత్త విండోను తెరవడానికి `F5` నొక్కండి.
* కమాండ్ ప్యాలెట్ నుంచి మీ కమాండ్‌ను నడిపించడానికి (`Ctrl+Shift+P` లేదా Macలో `Cmd+Shift+P`) నొక్కి `Hello World` టైప్ చేయండి.
* మీ ఎక్స్‌టెన్షన్‌ను డీబగ్ చేయడానికి `src/extension.ts` లో మీ కోడ్‌లో బ్రేక్‌పాయింట్లు సెట్ చేయండి.
* డీబగ్ కన్సోల్‌లో మీ ఎక్స్‌టెన్షన్ నుండి అవుట్‌పుట్‌ను చూడండి.

## మార్పులు చేయండి

* `src/extension.ts`‌లో కోడ్ మార్చిన తర్వాత డీబగ్ టూల్‌బార్ నుండి ఎక్స్‌టెన్షన్‌ను మళ్ళీ ప్రారంభించవచ్చు.
* మార్పులను లోడ్ చేయడానికి మీ ఎక్స్‌టెన్షన్‌తో ఉన్న VS Code విండోను రీలోడ్ (`Ctrl+R` లేదా Macలో `Cmd+R`) కూడా చేయవచ్చు.


## APIని అన్వేషించండి

* ఫైల్ `node_modules/@types/vscode/index.d.ts` ను తెరిచి మా API యొక్క పూర్తి సెట్‌ను చూడవచ్చు.

## పరీక్షలు నడపండి

* [ఎక్స్‌టెన్షన్ టెస్ట్ రన్నర్](https://marketplace.visualstudio.com/items?itemName=ms-vscode.extension-test-runner) ను ఇన్‌స్టాల్ చేయండి
* **Tasks: Run Task** కమాండ్ ద్వారా "watch" టాస్క్‌ను నడపండి. ఇది నడుస్తుందని ఖచ్చితంగా చూసుకోండి, లేకపోతే టెస్టులు కనుగొనబడకపోవచ్చు.
* యాక్టివిటీ బార్ నుండి Testing view ని తెరుచుకుని Run Test" బటన్‌పై క్లిక్ చేయండి, లేదా హాట్‌కీ `Ctrl/Cmd + ; A` ఉపయోగించండి
* Test Results view లో టెస్ట్ ఫలితాల అవుట్పుట్‌ను చూడండి.
* `src/test/extension.test.ts` ను మార్చండి లేదా `test` ఫోల్డర్‌లో కొత్త టెస్ట్ ఫైళ్లను సృష్టించండి.
  * అందించిన టెస్ట్ రన్నర్ పేరు ప్యాటర్న్ `**.test.ts` కి సరిపోయే ఫైళ్లనే పరిగణలోకి తీసుకుంటుంది.
  * మీరు మీ టెస్టులను ఏ విధంగా కావాలనుకున్నా నిర్మాణం కొరకు `test` ఫోల్డర్‌లో ఫోల్డర్లను సృష్టించవచ్చు.

## మరింత ముందుకు

* విస్తరణ పరిమాణాన్ని తగ్గించి స్టార్ట్‌అప్ సమయాన్ని మెరుగుపరచడానికి [మీ ఎక్స్‌టెన్షన్‌ను బండిల్ చేయండి](https://code.visualstudio.com/api/working-with-extensions/bundling-extension?WT.mc_id=aiml-137032-kinfeylo).
* [మీ ఎక్స్‌టెన్షన్‌ను ప్రచురించండి](https://code.visualstudio.com/api/working-with-extensions/publishing-extension?WT.mc_id=aiml-137032-kinfeylo) VS Code ఎక్స్‌టెన్షన్ మార్కెట్ప్లేస్‌లో.
* బిల్డులను ఆటోమేటు చేయడానికి [నిరంతర ఇంటిగ్రేషన్](https://code.visualstudio.com/api/working-with-extensions/continuous-integration?WT.mc_id=aiml-137032-kinfeylo) సెటప్ చేయండి.
