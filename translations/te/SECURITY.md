## భద్రత

Microsoft మా సాఫ్ట్వేర్ ఉత్పత్తుల మరియు సేవల భద్రతను గంభీరంగా తీసుకుంటుంది, ఇందులో మన GitHub సంస్థల ద్వారా నిర్వహించబడే అన్ని సోర్స్ కోడ్ రిపాజిటరీలు కూడా ఉన్నాయి, ఇవిలో [Microsoft](https://github.com/Microsoft), [Azure](https://github.com/Azure), [DotNet](https://github.com/dotnet), [AspNet](https://github.com/aspnet) మరియు [Xamarin](https://github.com/xamarin) ఉన్నాయి.

మీరు ఏదైనా Microsoft యాజమాన్యంలోని రిపాజిటరీలో [Microsoft's definition of a security vulnerability](https://aka.ms/security.md/definition) కు తగిన భద్రతా లోపాన్ని కనుగొన్నట్లయితే, దయచేసి క్రింద వివరించినట్లుగా మాకు నివేదించండి.

## భద్రతా సమస్యలు నివేదించడం

**దయచేసి భద్రతా లోపాలను పబ్లిక్ GitHub ఇష్యూల ద్వారా నివేదించవద్దు.**

దీని బదులుగా, దయచేసి వాటిని Microsoft Security Response Center (MSRC) వద్ద [https://msrc.microsoft.com/create-report](https://aka.ms/security.md/msrc/create-report) కి నివేదించండి.

లాగిన్ చేయకుండా సమర్పించాలనుకుంటే, [secure@microsoft.com](mailto:secure@microsoft.com) కి ఇమెయిల్ పంపండి.  సాధ్యమైతే, మీ సందేశాన్ని మా PGP కీతో సంక్షిప్తపరచండి; దయచేసి దాన్ని [Microsoft Security Response Center PGP Key page](https://aka.ms/security.md/msrc/pgp) నుండి డౌన్లోడ్ చేయండి.

మీకు 24 గంటల లోపల స్పందన అందవచ్చు. ఏ కారణంగా మీకు స్పందన అందకపోతే, మేము మీ అసలు సందేశాన్ని స్వీకరించామనే నిర్ధారణ కోసం దయచేసి ఇమెయిల్ ద్వారా అనుసరించండి. అదనపు సమాచారం [microsoft.com/msrc](https://www.microsoft.com/msrc) వద్ద అందుబాటులో ఉంటుంది। 

దయచేసి క్రింది అభ్యర్థిత సమాచారాన్ని (మీకు అందినంతగా) చేర్చండి, తద్వారా సమస్య యొక్క స్వభావం మరియు పరిధిని బాగా అర్థం చేసుకోవడంలో మాకు సహాయపడుతుంది:

  * సమస్య యొక్క రకం (ఉదా: buffer overflow, SQL injection, cross-site scripting, మొదలైనవి)
  * సమస్య ప్రదర్శనకు సంబంధించిన సోర్స్ ఫైల్(లు) యొక్క పూర్తి పాథ్లు
  * ప్రభావిత సోర్స్ కోడ్ యొక్క స్థానం (tag/branch/commit లేదా direct URL)
  * సమస్యను పునరుత్పత్తి చేయడానికి అవసరమయ్యే ప్రత్యేక కాన్ఫిగరేషన్
  * సమస్యను పునరుత్పత్తి చేయడానికి దశల వారీ సూచనలు
  * Proof-of-concept లేదా exploit కోడ్ (సాధ్యమైతే)
  * సమస్య యొక్క ప్రభావం, మరియు దాడికి అతివాది అది ఎలా దుర్వినియోగం చేయగలడో వివరాలు

ఈ సమాచారం మా నివేదికను తక్షణం_triage చేయడంలో సహాయపడుతుంది.

మీరు బగ్ బౌంటీకి నివేదిస్తున్నట్లయితే, పూర్తి నివేదికలు ఎక్కువ బౌంటీ అవార్డుకు దోహదపడ్డాయి. మా క్రియాశీల ప్రోగ్రామ్‌ల గురించి మరిన్ని వివరాలకు దయచేసి మా [Microsoft Bug Bounty Program](https://aka.ms/security.md/msrc/bounty) పేజీని సందర్శించండి.

## ఇష్టమైన భాషలు

మా విధానం అన్ని కమ్యూనికేషన్లు ఇంగ్లీష్‌లో ఉండాలని సూచిస్తుంది.

## విధానం

Microsoft [Coordinated Vulnerability Disclosure](https://aka.ms/security.md/cvd) సిద్ధాంతాన్ని అనుసరిస్తుంది।
