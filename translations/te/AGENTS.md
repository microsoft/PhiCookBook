# AGENTS.md

## ప్రాజెక్ట్ అవలోకనం

PhiCookBook అనేది Microsoft యొక్క Phi కుటుంబానికి చెందిన Small Language Models (SLMs) పై పని చేయడానికి హ్యాండ్స్-ఆన్ ఉదాహరణలు, ట్యూటోరియల్స్, మరియు డాక్యుమెంటేషన్ కలిగిన సంపూర్ణ కుక్‌బుక్ రిపాజిటరీ. ఈ రిపోజిటరీ అనేక ఉపయోగాల‌ను చూపిస్తుంది, ఉదాహరణకు ఇన్ఫరెన్సు, ఫైన్-ట్యూనింగ్, క్వాంటైజేషన్, RAG అమల్లు, మరియు బహు-మోడల్ అప్లికేషన్లు వివిధ ప్లాట్‌ఫారమ్‌లు మరియు ఫ్రేమ్‌వర్క్‌లలో.

**ప్రధాన సాంకేతికతలు:**
- **భాషలు:** Python, C#/.NET, JavaScript/Node.js
- **ఫ్రేమ్‌వర్క్‌లు:** ONNX Runtime, PyTorch, Transformers, MLX, OpenVINO, Semantic Kernel
- **ప్లాట్‌ఫామ్‌లు:** Microsoft Foundry, GitHub Models, Hugging Face, Ollama
- **మోడల్ రకాల‌:** Phi-3, Phi-3.5, Phi-4 (text, vision, multimodal, reasoning variants)

**రిపాజిటరీ నిర్మాణం:**
- `/code/` - పనికి వేయగల కోడ్ ఉదాహరణలు మరియు సెంపుల్ అమల్లు
- `/md/` - విస్తృత డాక్యుమెంటేషన్, ట్యూటోరియల్స్, మరియు హౌ-టు గైడ్‌లు  
- `/translations/` - బహు-భాషా అనువాదాలు (50+ భాషలు ఆటోమెటెడ్ వర్క్‌ఫ్లో ద్వారా)
- `/.devcontainer/` - Dev container కాన్ఫిగరేషన్ (Python 3.12 తో Ollama)

### లోకల్ సెటప్

**అవశ్యమైనవి:**
- Python 3.12 లేదా అంతకంటే కొత్త
- .NET 8.0 SDK (C# ఉదాహరణలకు)
- Node.js 18+ మరియు npm (JavaScript ఉదాహరణలకు)
- కనీసం 16GB RAM సిఫార్సు చేయబడింది

**ఇన్స్టాలేషన్:**
```bash
git clone https://github.com/microsoft/PhiCookBook.git
cd PhiCookBook
```

**Python ఉదాహరణల కోసం:**
నిర్దిష్ట ఉదాహరణ డైరెక్టరీలకి వెళ్లి డిపెండెన్సీలను ఇన్స్టాల్ చేయండి:
```bash
cd code/<example-directory>
pip install -r requirements.txt  # requirements.txt ఉన్నట్లయితే
```

**.NET ఉదాహరణల కోసం:**
```bash
cd md/04.HOL/dotnet/src
dotnet restore LabsPhi.sln
dotnet build LabsPhi.sln
```

**JavaScript/Web ఉదాహరణల కోసం:**
```bash
cd code/08.RAG/rag_webgpu_chat
npm install
npm run dev  # డెవలప్‌మెంట్ సర్వర్ ప్రారంభించండి
npm run build  # ప్రొడక్షన్ కోసం నిర్మించండి
```

## రిపాజిటరీ భాగాల వర్గీకరణ

- **03.Finetuning/** మరియు **04.Finetuning/** - వివిధ పద్ధతుల ద్వారా ఫైన్-ట్యూనింగ్ ఉదాహరణలు
- **03.Inference/** - వివిధ హార్డ్వేర్ (AIPC, MLX) పై ఇన్ఫరెన్సు ఉదాహరణలు
- **06.E2E/** - end-to-end అప్లికేషన్ సెంపుల్స్
- **07.Lab/** - ప్రయోగాత్మక/ప్రాథమిక అమల్లు
- **08.RAG/** - Retrieval-Augmented Generation నమూనాలు
- **09.UpdateSamples/** - తాజా అప్డేట్ చేయబడ్డ నమూనాలు

### డాక్యుమెంటేషన్ (`/md/`)

- **01.Introduction/** - పరిచయ గైడ్‌లు, ఎన్విరాన్‌మెంట్ సెటప్, ప్లాట్‌ఫామ్ గైడ్‌లు
- **02.Application/** - రకం ఆధారంగా సజావుగా విభజించిన అప్లికేషన్ సెంపుల్స్ (Text, Code, Vision, Audio, మొదలైనవి)
- **02.QuickStart/** - Microsoft Foundry మరియు GitHub Models కోసం తక్షణ-ప్రారంభ గైడ్‌లు
- **03.FineTuning/** - ఫైన్-ట్యూనింగ్ డాక్యుమెంటేషన్ మరియు ట్యూటోరియల్స్
- **04.HOL/** - హ్యాండ్స్-ఆన్ ల్యాబ్స్ (దీనిలో .NET ఉదాహరణలు కూడా ఉన్నాయి)

### ఫైల్ ఫార్మాట్లు

- **Jupyter Notebooks (`.ipynb`)** - ఇంటరాక్టివ్ Python ట్యుటోరియల్స్, READMEలో 📓 తో గుర్తించబడ్డవి
- **Python Scripts (`.py`)** - స్టాండ్అలోన్ Python ఉదాహరణలు

- **JavaScript (`.js`, `package.json`)** - వెబ్-ఆధారిత మరియు Node.js ఉదాహరణలు

## పరీక్షలు

ఈ రిపాజిటరీ సాధారణంగా యూనిట్ టెస్టులు కలిగిన సంప్రదాయ సాఫ్ట్‌వేర్ ప్రాజెక్ట్ కాకపోయి ఉదాహరణ కోడ్ మరియు ట్యూటోరియల్స్ కలిగి ఉంటుంది. చెల్లింపు సాధారణంగా ఈ విధంగా జరుగుతుంది:

1. **ఉదాహరణలు నడపడం** - ప్రతి ఉదాహరణ తప్పులుంటే లేకుండా అమలు కావాలి
2. **ఔట్‌పుట్‌లను ధృవీకరించడం** - మోడల్ ప్రతిస్పందనలు సరైనవని తనిఖీ చేయండి
3. **ట్యూటోరియల్స్ అనుసరించడం** - స్టెప్-బై-స్టెప్ గైడ్‌లు డాక్యుమెంటేషన్ ప్రకారం పనిచేయాలి

**సామાન્ય ధృవీకరణ విధానం:**
- లక్ష్య ఎన్విరాన్‌మెంట్‌లో ఉదాహరణ అమలు పరీక్షించండి
- డిపెండెన్సీలు సరిగ్గా ఇన్స్టాల్ అయ్యాయా అని చెక్ చేయండి
- మోడల్ డౌన్లోడ్స్/లోడ్స్ విజయవంతంగా జరిగాయా అని తనిఖీ చేయండి
- సమాచార ప్రవర్తన డాక్యుమెంటేషన్‌కు సరిపోతుందో నిర్ధారించండి

## పుల్ రిక్వెస్ట్ మార్గదర్శకాలు

### సమర్పణకు ముందుగా

1. రిపాజిటరీని మీ ఖాతాకు ఫోర్క్ చేయండి
2. PRలను రకం ప్రకారం వేరు చేయండి:
   - ఒక PRలో బగ్ ఫిక్సులు
   - మరో PRలో డాక్యుమెంటేషన్ అప్డేట్స్
   - కొత్త ఉదాహరణల కోసం వేరు PRలు
   - టైపో ఫిక్సులు కలిపి ఉండవచ్చు

3. మర్జ్ సంకర్షణలు నిర్వహించండి:
   - మార్పులు ముందు మీ స్థానిక `main` బ్రాంచ్‌ను అప్డేట్ చేయండి
   - అప్‌స్ట్రీమ్‌తో తరచూ సింక్ చేయండి

4. అనువాద PRలు:
   - ఫోల్డర్లోని అన్ని ఫైళ్లకు అనువాదాలను కలిగి ఉండాలి
   - ఆరిజినల్ భాషా నిర్మాణంతో నిరంతరత ఉంచండి

   - `/en-us/`, `/en/` లేదా ఇతర భాష కోడ్స్ తీసివేయండి
   - సాధారణ అంతర్జాతీయ URLలు ఉపయోగించండి

3. **భ్రష్టుచేసిన URL చెక్** - అన్ని URLలు 200 స్థితిని తిరిగి ఇవ్వాలి
   - సమర్పించే ముందు లింకులు యాక్సెస్ చేయగలిగేలా నిర్ధారించండి
   - గమనిక: కొన్ని విఫలమయ్యే అంశాలు నెట్‌వర్క్ పరిమితుల కారణంగా ఉండవచ్చు
