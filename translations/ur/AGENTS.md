# AGENTS.md

## منصوبے کا جائزہ

PhiCookBook ایک جامع کوک بک ریپوزٹری ہے جس میں Microsoft کے Phi خاندان کے چھوٹے زبان ماڈلز (SLMs) کے ساتھ کام کرنے کے لیے عملی مثالیں، ٹیوٹوریلز، اور دستاویزات شامل ہیں۔ یہ ریپوزٹری مختلف استعمال کے کیسز جیسے انفرنس، فائن-ٹیوننگ، کوانٹائزیشن، RAG نفاذات، اور مختلف پلیٹ فارمز اور فریم ورکس پر ملٹی موڈل ایپلی کیشنز کو ظاہر کرتی ہے۔

**اہم ٹیکنالوجیز:**
- **زبانیں:** Python، C#/.NET، JavaScript/Node.js
- **فریم ورکس:** ONNX Runtime، PyTorch، Transformers، MLX، OpenVINO، Semantic Kernel
- **پلیٹ فارمز:** Microsoft Foundry، GitHub Models، Hugging Face، Ollama
- **ماڈل کی اقسام:** Phi-3، Phi-3.5، Phi-4 (متن، وژن، ملٹی موڈل، استدلال کے مختلف ورژن)

**ریپوزٹری کی ساخت:**
- `/code/` - کام کرنے والے کوڈ مثالیں اور نمونہ نفاذات
- `/md/` - تفصیلی دستاویزات، ٹیوٹوریلز، اور ہاؤ-ٹو گائیڈز  
- `/translations/` - کثیر الزبانی تراجم (50+ زبانیں خودکار ورک فلو کے ذریعے)
- `/.devcontainer/` - ڈویپ کنٹینر کنفیگریشن (Python 3.12 کے ساتھ Ollama)

## ترقیاتی ماحول کی ترتیب

### GitHub Codespaces یا Dev Containers کا استعمال (تجویز کردہ)

1. GitHub Codespaces میں کھولیں (تیز ترین):
   - README میں "Open in GitHub Codespaces" بیج پر کلک کریں
   - کنٹینر Python 3.12 اور Ollama کے ساتھ Phi-3 کے لیے خودکار ترتیب پائے گا

2. VS Code Dev Containers میں کھولیں:
   - README سے "Open in Dev Containers" بیج استعمال کریں
   - کنٹینر کے لیے کم از کم 16GB میزبان میموری درکار ہے

### مقامی ترتیب

**ضروریات:**
- Python 3.12 یا اس کے بعد کا ورژن
- .NET 8.0 SDK (C# مثالوں کے لیے)
- Node.js 18+ اور npm (JavaScript مثالوں کے لیے)
- کم از کم 16GB RAM تجویز کی جاتی ہے

**انسٹالیشن:**
```bash
git clone https://github.com/microsoft/PhiCookBook.git
cd PhiCookBook
```

**Python مثالوں کے لیے:**
مخصوص مثالوں کے ڈائریکٹریز میں جائیں اور dependencies انسٹال کریں:
```bash
cd code/<example-directory>
pip install -r requirements.txt  # اگر requirements.txt موجود ہو
```

**.NET مثالوں کے لیے:**
```bash
cd md/04.HOL/dotnet/src
dotnet restore LabsPhi.sln
dotnet build LabsPhi.sln
```

**JavaScript/Web مثالوں کے لیے:**
```bash
cd code/08.RAG/rag_webgpu_chat
npm install
npm run dev  # سرور کی ترقی شروع کریں
npm run build  # پیداوار کے لیے بنائیں
```

## ٹیسٹنگ

یہ ریپوزٹری ایک روایتی سوفٹ ویئر پراجیکٹ نہیں بلکہ مثال کوڈ اور ٹیوٹوریلز پر مشتمل ہے جن کے یونٹ ٹیسٹ نہیں ہوتے۔ ویلیڈیشن عام طور پر یوں ہوتی ہے:

1. **مثالیں چلانا** - ہر مثال بغیر کسی غلطی کے چلنی چاہیے
2. **نتائج کی توثیق** - چیک کریں کہ ماڈل کے جوابات مناسب ہیں
3. **ٹیوٹوریلز کی پیروی** - مرحلہ وار گائیڈز دستاویز کے مطابق کام کریں

**عام ویلیڈیشن کا طریقہ کار:**
- ٹارگٹ ماحول میں مثالوں کو چلانے کا ٹیسٹ کریں
- dependencies کی درست تنصیب کی تصدیق کریں
- ماڈل کی ڈاؤن لوڈ یا لوڈنگ کی تصدیق کریں
- دستاویزات کے مطابق متوقع رویہ کی تصدیق کریں

## کوڈ کا انداز اور اصول

### عمومی رہنما اصول

- مثالیں واضح، اچھی طرح کمنٹ کی گئی، اور تعلیمی ہونی چاہئیں
- زبان کے مخصوص اصولوں کی پیروی کریں (Python کے لیے PEP 8، .NET کے لیے C# معیارات)
- مثالوں کو خاص Phi ماڈل کی صلاحیتوں کو ظاہر کرنے پر مرکوز رکھیں
- اہم تصورات اور ماڈل مخصوص پیرامیٹرز کی وضاحت کے لیے کمنٹس شامل کریں

### دستاویزات کے معیارات

**URL فارمیٹنگ:**
- `[text](../../url)` فارمیٹ بغیر اضافی اسپیس کے استعمال کریں
- رشتہ دار لنکس کے لیے: موجودہ ڈائریکٹری کے لیے `./`، اوپر والی ڈائریکٹری کے لیے `../` استعمال کریں
- URLs میں ملک خاص لوکیل نہ ڈالیں (جیسے `/en-us/`، `/en/` سے پرہیز کریں)

**تصاویر:**
- تمام تصاویر `/imgs/` فولڈر میں اسٹور کریں
- ناموں میں انگریزی حروف، اعداد، اور ڈیش استعمال کریں
- مثال: `phi-3-architecture.png`

**Markdown فائلیں:**
- حقیقی کام کرنے والی مثالوں کا حوالہ دیں جو `/code/` میں ہوں
- دستاویزات کو کوڈ کی تبدیلیوں کے ساتھ ہم آہنگ رکھیں
- README میں Jupyter نوٹ بکس کے لنکس کو 📓 ایموجی کے ساتھ نشان زد کریں

### فائل کی تنظیم

- کوڈ مثالیں `/code/` میں موضوع یا خصوصیت کے حساب سے منظم ہوں

## پل ریکوئسٹ کے رہنما اصول

### جمع کرانے سے پہلے

1. **ریپوزٹری کو فورک کریں** اپنے اکاؤنٹ میں
2. **PR کو اقسام کے حساب سے جدا کریں:**
   - بگ فکسز ایک PR میں
   - دستاویزات کی اپ ڈیٹس دوسروں میں
   - نئی مثالیں الگ PRs میں
   - ٹائپوز کی اصلاحات کو ملا کر بھیج سکتے ہیں

3. **مرج تضادات کو سنبھالیں:**
   - تبدیلیاں کرنے سے پہلے اپنے لوکل `main` برانچ کو اپڈیٹ کریں
   - اپ اسٹریم کے ساتھ باقاعدگی سے ہم آہنگ رہیں

4. **ترجمہ کے PRs:**
   - فولڈر کی تمام فائلوں کے تراجم شامل ہوں
   - اصل زبان کے ساخت کے ساتھ مطابقت رکھیں

### ضروری چیکز

PR خودکار طریقے سے GitHub ورک فلو چلاتا ہے ویلیڈیشن کے لیے:

1. **رشتہ دار راستے کی تصدیق** - تمام اندرونی لنکس کام کریں  
   - VS Code میں Ctrl+Click سے لنکس کو مقامی طور پر ٹیسٹ کریں  
   - VS Code کی طرف سے راستے کی تجاویز استعمال کریں (`./` یا `../`)

2. **URL لوکیل چیک** - ویب URLs میں ملک کی لوکیل نہ ہو  
   - `/en-us/`، `/en/` یا دیگر زبان کوڈز ہٹائیں  
   - بین الاقوامی عام URLs استعمال کریں

3. **ٹوٹے ہوئے URL چیک** - تمام URLs کو 200 اسٹیٹس واپس کرنا چاہیے  
   - جمع کرانے سے پہلے لنکس کی دستیابی کی تصدیق کریں  
   - نوٹ: کچھ ناکامیاں نیٹ ورک پابندیوں کی وجہ سے ہو سکتی ہیں

### PR عنوان کا فارمیٹ

```
[component] Brief description
```

مثالیں:  
- `[docs] Phi-4 انفرنس ٹیوٹوریل شامل کریں`  
- `[code] ONNX Runtime انٹیگریشن کی مثال درست کریں`  
- `[translation] تعارفی گائیڈز کے لیے جاپانی ترجمہ شامل کریں`

**میموری کے مسائل:**
- Phi ماڈلز کو خاص طور پر وژن/ملٹی موڈل ورژنز کے لیے کافی RAM چاہیے

- انفرادی `requirements.txt` فائلز چیک کریں

**ماڈل ڈاؤن لوڈ میں ناکامی:**
- بڑے ماڈلز آہستہ کنکشنوں پر ٹائم آؤٹ ہو سکتے ہیں
