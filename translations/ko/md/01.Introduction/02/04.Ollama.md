## Ollama의 Phi 패밀리


[Ollama](https://ollama.com) 은 더 많은 사람들이 간단한 스크립트를 통해 오픈 소스 LLM 또는 SLM을 직접 배포할 수 있도록 하며, 로컬 Copilot 애플리케이션 시나리오를 돕는 API도 구축할 수 있습니다.

## **1. 설치**

Ollama는 Windows, macOS 및 Linux에서 실행을 지원합니다. 다음 링크([https://ollama.com/download](https://ollama.com/download))를 통해 Ollama를 설치할 수 있습니다. 설치가 완료되면 터미널 창에서 Ollama 스크립트를 사용하여 Phi-3를 바로 호출할 수 있습니다. 모든 [Ollama에서 사용 가능한 라이브러리](https://ollama.com/library)를 확인할 수 있습니다. 이 저장소를 Codespace에서 열면 이미 Ollama가 설치되어 있습니다.

```bash

ollama run phi4

```

> [!NOTE]
> 모델은 처음 실행할 때 먼저 다운로드됩니다. 물론 이미 다운로드한 Phi-4 모델을 직접 지정할 수도 있습니다. 명령 실행 예로 WSL을 사용합니다. 모델이 정상적으로 다운로드되면 터미널에서 바로 상호작용할 수 있습니다.

![실행](../../../../../translated_images/ko/ollama_run.e9755172b162b381.png)

## **2. Ollama에서 phi-4 API 호출하기**

ollama가 생성한 Phi-4 API를 호출하려면, 터미널에서 이 명령을 사용하여 Ollama 서버를 시작할 수 있습니다.

```bash

ollama serve

```

> [!NOTE]
> MacOS 또는 Linux에서 실행하는 경우 다음 오류 **"Error: listen tcp 127.0.0.1:11434: bind: address already in use"** 가 발생할 수 있습니다. 명령을 실행할 때 이 오류가 발생할 수 있습니다. 일반적으로 서버가 이미 실행 중임을 나타내므로 해당 오류를 무시해도 되거나, Ollama를 중지했다가 다시 시작할 수도 있습니다:

**macOS**

```bash

brew services restart ollama

```

**Linux**

```bash

sudo systemctl stop ollama

```

Ollama는 두 가지 API를 지원합니다: generate 및 chat. 포트 11434에서 실행 중인 로컬 서비스로 요청을 전송하여 필요에 따라 Ollama가 제공하는 모델 API를 호출할 수 있습니다.

**Chat**

```bash

curl http://127.0.0.1:11434/api/chat -d '{
  "model": "phi3",
  "messages": [
    {
      "role": "system",
      "content": "Your are a python developer."
    },
    {
      "role": "user",
      "content": "Help me generate a bubble algorithm"
    }
  ],
  "stream": false
  
}'
```

## 추가 자료

Ollama에서 사용 가능한 모델 목록은 [라이브러리](https://ollama.com/library)에서 확인하세요.

다음 명령으로 Ollama 서버에서 모델을 가져오세요

```bash
ollama pull phi4
```

다음 명령으로 모델을 실행하세요

```bash
ollama run phi4
```

***참고:*** 이 링크 [https://github.com/ollama/ollama/blob/main/docs/api.md](https://github.com/ollama/ollama/blob/main/docs/api.md) 를 방문하세요

## Python에서 Ollama 호출하기

위에서 사용한 로컬 서버 엔드포인트에 요청을 보내기 위해 `requests` 또는 `urllib3`를 사용할 수 있습니다. 그러나 Ollama가 OpenAI 호환 서버 엔드포인트도 제공하므로, Python에서 Ollama를 사용하는 일반적인 방법은 [openai](https://pypi.org/project/openai/) SDK를 사용하는 것입니다.

다음은 phi3-mini 예제입니다:

```python
import openai

client = openai.OpenAI(
    base_url="http://localhost:11434/v1",
    api_key="nokeyneeded",
)

response = client.chat.completions.create(
    model="phi4",
    temperature=0.7,
    n=1,
    messages=[
        {"role": "system", "content": "You are a helpful assistant."},
        {"role": "user", "content": "Write a haiku about a hungry cat"},
    ],
)

print("Response:")
print(response.choices[0].message.content)
```

## JavaScript에서 Ollama 호출하기 

```javascript
// Phi-4로 파일을 요약하는 예시
script({
    model: "ollama:phi4",
    title: "Summarize with Phi-4",
    system: ["system"],
})

// 요약 예시
const file = def("FILE", env.files)
$`Summarize ${file} in a single paragraph.`
```

## C#에서 Ollama 호출하기

새로운 C# 콘솔 애플리케이션을 생성하고 다음 NuGet 패키지를 추가하세요:

```bash
dotnet add package Microsoft.SemanticKernel --version 1.34.0
```

그런 다음 `Program.cs` 파일의 코드를 다음으로 교체하세요

```csharp
using Microsoft.SemanticKernel;
using Microsoft.SemanticKernel.ChatCompletion;

// add chat completion service using the local ollama server endpoint
#pragma warning disable SKEXP0001, SKEXP0003, SKEXP0010, SKEXP0011, SKEXP0050, SKEXP0052
builder.AddOpenAIChatCompletion(
    modelId: "phi4",
    endpoint: new Uri("http://localhost:11434/"),
    apiKey: "non required");

// invoke a simple prompt to the chat service
string prompt = "Write a joke about kittens";
var response = await kernel.InvokePromptAsync(prompt);
Console.WriteLine(response.GetValue<string>());
```

다음 명령으로 앱을 실행하세요:

```bash
dotnet run
```
