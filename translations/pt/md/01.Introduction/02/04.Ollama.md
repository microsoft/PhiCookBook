## Família Phi no Ollama


[Ollama](https://ollama.com) permite que mais pessoas implantem diretamente LLMs ou SLMs de código aberto através de scripts simples, e também pode construir APIs para ajudar cenários de aplicação Copilot locais.

## **1. Instalação**

Ollama suporta execução no Windows, macOS e Linux. Pode instalar o Ollama através deste link ([https://ollama.com/download](https://ollama.com/download)). Após a instalação bem-sucedida, pode utilizar diretamente o script do Ollama para chamar o Phi-3 através de uma janela de terminal. Pode ver todas as [bibliotecas disponíveis no Ollama](https://ollama.com/library). Se abrir este repositório num Codespace, este já terá o Ollama instalado.

```bash

ollama run phi4

```

> [!NOTE]
> O modelo será transferido primeiro quando o executar pela primeira vez. Claro que também pode especificar diretamente o modelo Phi-4 já transferido. Tomamos o WSL como exemplo para executar o comando. Depois do modelo ser transferido com sucesso, pode interagir diretamente no terminal.

![execução](../../../../../translated_images/pt/ollama_run.e9755172b162b381.png)

## **2. Invocar a API phi-4 a partir do Ollama**

Se quiser chamar a API Phi-4 gerada pelo Ollama, pode usar este comando no terminal para iniciar o servidor Ollama.

```bash

ollama serve

```

> [!NOTE]
> Se estiver a executar no MacOS ou Linux, note que pode encontrar o seguinte erro **"Error: listen tcp 127.0.0.1:11434: bind: address already in use"**. Pode obter este erro ao executar o comando. Pode ignorar esse erro, uma vez que normalmente indica que o servidor já está em execução, ou pode parar e reiniciar o Ollama:

**macOS**

```bash

brew services restart ollama

```

**Linux**

```bash

sudo systemctl stop ollama

```

Ollama suporta duas APIs: generate e chat. Pode chamar a API do modelo fornecida pelo Ollama conforme as suas necessidades, enviando pedidos para o serviço local a correr na porta 11434.

**Chat**

```bash

curl http://127.0.0.1:11434/api/chat -d '{
  "model": "phi3",
  "messages": [
    {
      "role": "system",
      "content": "Your are a python developer."
    },
    {
      "role": "user",
      "content": "Help me generate a bubble algorithm"
    }
  ],
  "stream": false
  
}'
```

## Recursos Adicionais

Consulte a lista de modelos disponíveis no Ollama na [biblioteca deles](https://ollama.com/library).

Obtenha o seu modelo do servidor Ollama usando este comando

```bash
ollama pull phi4
```

Execute o modelo com este comando

```bash
ollama run phi4
```

***Nota:*** Visite este link [https://github.com/ollama/ollama/blob/main/docs/api.md](https://github.com/ollama/ollama/blob/main/docs/api.md) para saber mais

## Invocar o Ollama a partir do Python

Pode usar `requests` ou `urllib3` para efetuar pedidos aos endpoints do servidor local usados acima. No entanto, uma forma popular de usar o Ollama em Python é através do SDK [openai](https://pypi.org/project/openai/), uma vez que o Ollama também fornece endpoints de servidor compatíveis com a OpenAI.

Aqui está um exemplo para phi3-mini:

```python
import openai

client = openai.OpenAI(
    base_url="http://localhost:11434/v1",
    api_key="nokeyneeded",
)

response = client.chat.completions.create(
    model="phi4",
    temperature=0.7,
    n=1,
    messages=[
        {"role": "system", "content": "You are a helpful assistant."},
        {"role": "user", "content": "Write a haiku about a hungry cat"},
    ],
)

print("Response:")
print(response.choices[0].message.content)
```

## Invocar o Ollama a partir do JavaScript 

```javascript
// Exemplo de resumir um ficheiro com Phi-4
script({
    model: "ollama:phi4",
    title: "Summarize with Phi-4",
    system: ["system"],
})

// Exemplo de resumir
const file = def("FILE", env.files)
$`Summarize ${file} in a single paragraph.`
```

## Invocar o Ollama a partir do C#

Crie uma nova aplicação Console em C# e adicione o seguinte pacote NuGet:

```bash
dotnet add package Microsoft.SemanticKernel --version 1.34.0
```

Depois, substitua este código no ficheiro `Program.cs`

```csharp
using Microsoft.SemanticKernel;
using Microsoft.SemanticKernel.ChatCompletion;

// add chat completion service using the local ollama server endpoint
#pragma warning disable SKEXP0001, SKEXP0003, SKEXP0010, SKEXP0011, SKEXP0050, SKEXP0052
builder.AddOpenAIChatCompletion(
    modelId: "phi4",
    endpoint: new Uri("http://localhost:11434/"),
    apiKey: "non required");

// invoke a simple prompt to the chat service
string prompt = "Write a joke about kittens";
var response = await kernel.InvokePromptAsync(prompt);
Console.WriteLine(response.GetValue<string>());
```

Execute a aplicação com o comando:

```bash
dotnet run
```
