Phi היא סדרה של דגמי בינה מלאכותית בקוד פתוח שפותחה על ידי מיקרוסופט.

Phi כיום הוא הדגם הקטן ביותר בעל הביצועים הטובים ביותר והעלות-תועלת הגבוהה ביותר (SLM), עם מדדים טובים מאוד בשפות מרובות, היגיון, יצירת טקסט/שיחה, קידוד, תמונות, שמע ותסריטים נוספים.

ניתן לפרוס את Phi בענן או במכשירים קצה, וניתן לבנות יישומים מגנרטיביים של AI בקלות עם כוח חישוב מוגבל.

> **מעדיף לשכפל באופן מקומי?**
>
> מאגר זה כולל מעל 50 תרגומים לשפות שונות, מה שמגדיל משמעותית את גודל ההורדה. כדי לשכפל בלי תרגומים, השתמש ב-sparse checkout:
>
> **Bash / macOS / Linux:**
> ```bash
> git clone --filter=blob:none --sparse https://github.com/microsoft/PhiCookBook.git
> cd PhiCookBook
> git sparse-checkout set --no-cone '/*' '!translations' '!translated_images'
> ```
>
> **CMD (Windows):**
> ```cmd
> git clone --filter=blob:none --sparse https://github.com/microsoft/PhiCookBook.git
> cd PhiCookBook
> git sparse-checkout set --no-cone "/*" "!translations" "!translated_images"
> ```
>
> זה נותן לך כל מה שאתה צריך כדי להשלים את הקורס עם הורדה הרבה יותר מהירה.

## AI אחראי

מייקרוסופט מחויבת לסייע ללקוחותיה להשתמש במוצרי ה-AI שלנו באחריות, לשתף את התובנות שלנו ולבנות שותפויות מבוססות אמון באמצעות כלים כמו הערות שקיפות והערכות השפעה. משאבים רבים ניתן למצוא ב-[https://aka.ms/RAI](https://aka.ms/RAI).  
הגישה של מייקרוסופט ל-AI אחראי מבוססת על עקרונות ה-AI שלנו: הוגנות, אמינות ובטיחות, פרטיות ואבטחה, כלילתנות, שקיפות ואחריותיות.

מודלים נרחבים של שפה טבעית, תמונה ודיבור - כמו אלה שבדוגמה זו - עלולים לפעול באופן לא הוגן, לא אמין או פוגעני, ולגרום נזקים. נא להתייעץ עם [הערת השקיפות של שירות Azure OpenAI](https://learn.microsoft.com/legal/cognitive-services/openai/transparency-note?tabs=text) כדי להיות מודעים לסיכונים ומגבלות.

הגישה המומלצת למזעור סיכונים אלה היא לכלול מערכת בטיחות בארכיטקטורה שלכם שיכולה לזהות ולמנוע התנהגות מזיקה. [Azure AI Content Safety](https://learn.microsoft.com/azure/ai-services/content-safety/overview) מספק שכבת הגנה עצמאית, המסוגלת לזהות תכנים מזיקים שנוצרים על ידי משתמשים או ב-AI באפליקציות ושירותים. שירות Content Safety של Azure AI כולל APIs לטקסט ותמונה המאפשרים לזהות חומר מזיק. בתוך Microsoft Foundry, שירות Content Safety מאפשר לכם לצפות, לחקור ולנסות דוגמאות קוד לזיהוי תכנים מזיקים במודאליות שונות. התיעוד הבא של [הדרכה מהירה](https://learn.microsoft.com/azure/ai-services/content-safety/quickstart-text?tabs=visual-studio%2Clinux&pivots=programming-language-rest) ינחה אתכם כיצד לבצע בקשות לשירות.
היבט נוסף שיש לקחת בחשבון הוא ביצועי האפליקציה הכוללים. עם אפליקציות מולטי-מודל ומולטי-מודלים, אנו מתייחסים לביצועים כמשמעות שהמערכת מתפקדת כפי שאתה והמשתמשים שלך מצפים, כולל אי-יצירת פלטים מזיקים. חשוב להעריך את ביצועי האפליקציה הכוללת שלך באמצעות [Performance and Quality and Risk and Safety evaluators](https://learn.microsoft.com/azure/ai-studio/concepts/evaluation-metrics-built-in). יש לך גם את האפשרות ליצור ולהעריך עם [custom evaluators](https://learn.microsoft.com/azure/ai-studio/how-to/develop/evaluate-sdk#custom-evaluators).

אתה יכול להעריך את אפליקציית ה-AI שלך בסביבת הפיתוח שלך באמצעות [Azure AI Evaluation SDK](https://microsoft.github.io/promptflow/index.html). בהתחשב במאגר נתונים לבחינה או מטרה, יצירות האיי-איי הגנרטיביות שלך נמדדות כמותית עם מעריכי ביצועים מובנים או מעריכי ביצועים מותאמים לבחירתך. כדי להתחיל עם azure ai evaluation sdk להערכת המערכת שלך, אתה יכול לעקוב אחר [quickstart guide](https://learn.microsoft.com/azure/ai-studio/how-to/develop/flow-evaluate-sdk). לאחר שתבצע ריצת הערכה, תוכל [visualize the results in Microsoft Foundry](https://learn.microsoft.com/azure/ai-studio/how-to/evaluate-flow-results).
