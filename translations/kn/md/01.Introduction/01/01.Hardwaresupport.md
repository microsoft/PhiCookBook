# Phi ಹಾರ್ಡ್‌ವೇರ್ ಬೆಂಬಲ

Microsoft Phi ಅನ್ನು ONNX Runtime ಗಾಗಿ 최ಪಡಿಸಲಾಗಿದೆ ಮತ್ತು Windows DirectML ಅನ್ನು ಬೆಂಬಲಿಸುತ್ತದೆ. ಇದು ವಿವಿಧ ಹಾರ್ಡ್‌ವೇರ್ ವಿಧಗಳಲ್ಲಿ ಚೆನ್ನಾಗಿ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತದೆ, ಅದರಲ್ಲಿ GPUs, CPUs ಮತ್ತು ಮೊಬೈಲ್ ಸಾಧನಗಳೂ ಸೇರಿವೆ. 

## ಸಾಧನ ಹಾರ್ಡ್‌ವೇರ್ 
ನಿರ್ದಿಷ್ಟವಾಗಿ, ಬೆಂಬಲಿತ ಹಾರ್ಡ್‌ವೇರ್‌ಗಳಲ್ಲಿ ಕೆಳಗಿನವುಗಳನ್ನು ಒಳಗೊಂಡಿವೆ:

- GPU SKU: RTX 4090 (DirectML)
- GPU SKU: 1 A100 80GB (CUDA)
- CPU SKU: Standard F64s v2 (64 vCPUs, 128 GiB memory)

## ಮೊಬೈಲ್ SKU

- Android - Samsung Galaxy S21
- Apple iPhone 14 ಅಥವಾ ಅದಕ್ಕಿಂತೆ ಮೇಲೆ A16/A17 Processor

## Phi ಹಾರ್ಡ್‌ವೇರ್ ನಿರ್ದಿಷ್ಟತೆ

- ಕನಿಷ್ಠ ಸಂರಚನೆ ಅಗತ್ಯ.
- Windows: DirectX 12-ಸಾಮರ್ಥ್ಯವುಳ್ಳ GPU ಮತ್ತು ಕನಿಷ್ಠ 4GB ಸಂಯುಕ್ತ RAM

CUDA: Compute Capability >= 7.02 ಹೊಂದಿರುವ NVIDIA GPU

![ಹಾರ್ಡ್‌ವೇರ್ ಬೆಂಬಲ](../../../../../translated_images/kn/01.phihardware.5d51b2377cba18af.png)

## onnxruntime ಅನ್ನು ಬಹು-GPUಗಳ ಮೇಲೆ ಕಾರ್ಯಗತಗೊಳಿಸುವುದು

ಪ್ರಸ್ತುತ ಲಭ್ಯವಿರುವ Phi ONNX ಮಾದರಿಗಳು ಕೇವಲ 1 GPU ಕ್ಕೆ ಮಾತ್ರವಾಗಿವೆ. Phi ಮಾದರಿಗಳಿಗೆ ಬಹು-GPU ಬೆಂಬಲವನ್ನು ಒದಗಿಸುವುದಕ್ಕೆ ಸಾಧ್ಯತೆ ಇದ್ದರೂ, 2 GPU ಗಳೊಂದಿಗೆ ORT ಬಳಸಿದರೆ ಅದು ort ನ 2 ಇನ್ಸ್ಟಾನ್ಸ್‌ಗಳಿಗಿಂತ ಹೆಚ್ಚಿನ throughput ನೀಡಲಿದೆ ಎಂದು ಖಚಿತಪಡಿಸಲಾಗುವುದಿಲ್ಲ. ಇತ್ತೀಚಿನ ಅಪ್ಡೇಟ್ಗಳಿಗಾಗಿ ದಯವಿಟ್ಟು [ONNX Runtime](https://onnxruntime.ai/) ನೋಡಿ.

[Build 2024 GenAI ONNX ತಂಡ](https://youtu.be/WLW4SE8M9i8?si=EtG04UwDvcjunyfC) ನಲ್ಲಿ ಅವರು Phi ಮಾದರಿಗಳಿಗಾಗಿ multi-gpu ಬದಲು multi-instance ಅನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಿದ್ದರೆಂದು ಘೋಷಿಸಿದರು. 

ಈಗಿಗೆ ಇದರಿಂದ ನೀವು ಕೆಳಗಿನಂತೆ CUDA_VISIBLE_DEVICES environment variable ಬಳಸಿ ಒಂದು onnxruntime ಅಥವಾ onnxruntime-genai ಇನ್ಸ್ಟಾನ್ಸ್ ಅನ್ನು ಚಲಾಯಿಸಬಹುದು.

```Python
CUDA_VISIBLE_DEVICES=0 python infer.py
CUDA_VISIBLE_DEVICES=1 python infer.py
```

Phi ಅನ್ನು ಇನ್ನಷ್ಟು ಅನ್ವೇಷಿಸಲು ಸ್ವತಂತ್ರವಾಗಿ [Azure AI Foundry](https://ai.azure.com) ಅನ್ನು ಅನ್ವೇಷಿಸಿ
