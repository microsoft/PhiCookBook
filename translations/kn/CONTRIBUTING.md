# ಸಹಭಾಗಿತ್ವ

ಈ ಪ್ರಾಜೆಕ್ಟ್‌ಗೆ ಕೊಡುಗೆಗಳು ಮತ್ತು ಸೂಚನೆಗಳನ್ನು ಸ್ವಾಗತಿಸಲಾಗಿದೆ. ಹೆಚ್ಚಿನ ಕೊಡುಗೆಗಳಿಗೆ ನೀವು ನಿಮ್ಮ ಕೊಡುಗೆಯನ್ನು ಬಳಸಲು ನಮ್ಮಿಗೆ ಹಕ್ಕುಗಳನ್ನು ನೀಡಲು ನಿಮಗೆ ಹಕ್ಕುವಿದೆ ಮತ್ತು ನಿಜವಾಗಿ ನೀವು ನೀಡುತ್ತಿದ್ದೀರಿ ಎಂದು ಘೋಷಿಸುವ ಕೆಲವು Contributor License Agreement (CLA) ಒಪ್ಪಿಕೊಳ್ಳಬೇಕು. ವಿವರಗಳಿಗೆ, ಭೇಟಿ ಮಾಡಿ [https://cla.opensource.microsoft.com](https://cla.opensource.microsoft.com)

ನೀವು ಪಲ್ಲ್ ರಿಕ್ವೆಸ್ಟ್ ಸಲ್ಲಿಸಿದಾಗ, CLA ಬಾಟ್ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ನೀವು CLA ಒದಗಿಸಬೇಕೇ ಎಂಬುದನ್ನು ನಿರ್ಧರಿಸಿ PR ಅನ್ನು ಸೂಕ್ತವಾಗಿ ಅಲಂಕರಿಸುತ್ತದೆ (ಉದಾ., ಸ್ಥಿತಿಚೌಕಟ್ಟು, ಕಾಮೆಂಟ್). ಬಾಟ್ ನೀಡುವ ಸೂಚನೆಗಳನ್ನು ಅನುಸರಿಸಿ. ನಮ್ಮ CLA ಬಳಕೆಮಾಡುತ್ತಿರುವ ಎಲ್ಲಾ ರೆಪ್‌ಗಳಲ್ಲಿಯೂ ನಿಮಗೆ ಇದನ್ನು ಒಂದೇ ಬಾರಿ ಮಾತ್ರ ಮಾಡಬೇಕಾಗುತ್ತದೆ.

## ಆಚರಣಾ ನಿಯಮಾವಳಿ

ಈ ಪ್ರాజೆಕ್ಟ್ [Microsoft Open Source Code of Conduct](https://opensource.microsoft.com/codeofconduct/) ಅನ್ನು ಅಂಗೀಕರಿಸಿದೆ. ಹೆಚ್ಚಿನ ಮಾಹಿತಿಗಾಗಿ [Code of Conduct FAQ](https://opensource.microsoft.com/codeofconduct/faq/) ಓದಿ ಅಥವಾ ಯಾವುದೇ ಹೆಚ್ಚುವರಿ ಪ್ರಶ್ನೆಗಳಿದ್ದರೆ ಅಥವಾ ಕಾಮೆಂಟ್ಗಳಿಗಾಗಿ [opencode@microsoft.com](mailto:opencode@microsoft.com) ಸಂಪರ್ಕಿಸಿರಿ.

## Issues ರಚಿಸುವಾಗ ಎಚ್ಚರಿಕೆ

ಸಾಮಾನ್ಯ ಸಪೋರ್ಟ್ ಪ್ರಶ್ನೆಗಳಿಗಾಗಿ GitHub Issues ತೆರೆಯದಿರಿ, ಏಕೆಂದರೆ GitHub ಪಟ್ಟಿಯನ್ನು ವೈಶಿಷ್ಟ್ಯ ವಿನಂತಿಗಳು ಮತ್ತು ದೋಷ ವರದಿಗಳುಗಾಗಿ ಬಳಸಲಾಗಬೇಕು. ಈ ರೀತಿಯಲ್ಲಿ ನಾವು ಕodesಿಂದ ಬಂದ ನಿಜವಾದ ಸಮಸ್ಯೆಗಳು ಅಥವಾ ದೋಷಗಳನ್ನು ಸುಲಭವಾಗಿ ಟ್ರ್ಯಾಕ್ ಮಾಡಿ ಸಾಮಾನ್ಯ ಚರ್ಚೆಯನ್ನು ನಿಜವಾದ ಕೋದಿಂದ ಬೇರ್ಪಡಿಸಬಹುದು.

## ಯೋಗದಾನ ಹೇಗೆ

### ಪುಲ್ ರಿಕ್ವೆಸ್ಟ್ ಮಾರ್ಗಸೂಚಿಗಳು

ನೀವು Phi-3 CookBook ರೆಪೊಸಿಟರಿಗೆ ಪುಲ್ ರಿಕ್ವೆಸ್ಟ್ (PR) ಸಲ್ಲಿಸುವಾಗ, ದಯವಿಟ್ಟು ಕೆಳಗಿನ ಮಾರ್ಗಸೂಚಿಗಳನ್ನು ಅನುಸರಿಸಿ:

- **ರೆಪೊಸಿಟರಿಯನ್ನು Fork ಮಾಡಿ**: ನಿಮ್ಮ ಬದಲಾವಣೆಗಳನ್ನು ಮಾಡುವ ಮೊದಲು 항상 ರೆಪೊಸಿಟರಿಯನ್ನು ನಿಮ್ಮ ಖಾತೆಗೆ Fork ಮಾಡಿ.

- **ವಿಭಿನ್ನ ಪುಲ್ ರಿಕ್ವೆಸ್ಟ್‌ಗಳು (PR)**:
  - ಪ್ರತಿಯೊಂದು ಬದಲಾವಣೆಯ ಪ್ರಕಾರವನ್ನು ಅದರದೇ ಪುಲ್ ರಿಕ್ವೆಸ್ಟ್‌ನಲ್ಲಿ ಸಲ್ಲಿಸಿ. ಉದಾಹರಣೆಗೆ, ದೋಷ ಪರಿಹಾರಗಳು ಮತ್ತು ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ತಿದ್ದುಪಡಿಗಳನ್ನು ವಿಭಿನ್ನ PR ಗಳಲ್ಲಿ ಸಲ್ಲಿಸಬೇಕು.
  - ಟೈಪೋ ತಿದ್ದೆಗಳು ಮತ್ತು ಸಣ್ಣ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ನವೀಕರಣಗಳನ್ನು ಸೂಕ್ತವಾದರೆ ಒಂದು PR ನಲ್ಲಿ ಸಂಯೋಜಿಸಬಹುದು.

- **ಮರ್ಜಿ ಸಂಘರ್ಷಗಳನ್ನು ಹ್ಯಾಂಡಲ್ ಮಾಡಿ**: ನಿಮ್ಮ ಪುಲ್ ರಿಕ್ವೆಸ್ಟ್‌ನಲ್ಲಿ ಮರ್ಜಿ ಸಂಘರ್ಷಗಳಿದ್ದರೆ, ನಿಮ್ಮ ಸ್ಥಳೀಯ `main` ಶಾಖೆಯನ್ನು ಮುಖ್ಯ ರೆಪೊಶಿಟರಿಯನ್ನು ಅನುಸರಣೆಯಂತೆ ಅಪ್ಡೇಟ್ ಮಾಡಿ ಮತ್ತು ಅದನ್ನು ಮಾಡಿದ ನಂತರ ನಿಮ್ಮ ಬದಲಾವಣೆಗಳನ್ನು ಮಾಡಿ.

- **ಅನುವಾದ ಸಲ್ಲಿಕೆಗಳು**: ಅನುವಾದ PR ಸಲ್ಲಿಸುವಾಗ, ಮೂಲ ಫೋಲ್ಡರಿನ ಎಲ್ಲಾ ಫೈಲ್‌ಗಳಿಗೂ ಅನುವಾದಗಳನ್ನು ಅನುವಾದ ಫೋಲ್ಡರಿನಲ್ಲಿ ಸೇರಿಸಿದ್ದೀರಿ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ.

### ಬರೆಯಲು ಮಾರ್ಗಸೂಚಿಗಳು

ಎಲ್ಲಾ ಡಾಕ್ಯುಮೆಂಟ್‌ಗಳಲ್ಲಿಯೂ ಸुसಂಗತತೆಯನ್ನು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಲು, ದಯವಿಟ್ಟು ಕೆಳಗಿನ ಮಾರ್ಗಸೂಚಿಗಳನ್ನು ಬಳಸಿ:

- **URL ಫಾರ್ಮ್ಯಾಟಿಂಗ್**: ಎಲ್ಲ URL ಗಳನ್ನು ಚೌಕಟ್ಟಿನ ಬ್ರಾಕೆಟ್‌ಗಳಲ್ಲಿ ಹಾಕಿ ನಂತರ ಕೌಲೀಸ್‌ಗಳೊಂದಿಗೆ ಸುತ್ತುವರಿಸಬೇಕು, ಈ ನಡುವೆ ಅಥವಾ ಒಳಗೆ ಯಾವುದೇ ಹೆಚ್ಚುವರಿ ಶೂನ್ಯಗಳು ಇರಬಾರದು. ಉದಾಹರಣೆಗೆ: `[example](https://www.microsoft.com)`.

- **ಸಂಬಂಧಿತ ಲಿಂಕ್‌ಗಳು**: ಪ್ರಸ್ತುತ ಡೈರೆಕ್ಟರಿಯಲ್ಲಿರುವ ಫೈಲ್‌ಗಳು ಅಥವಾ ಫೋಲ್ಡರ್ಗಳಿಗೆ ಸಂಬಂಧಿತ ಲಿಂಕ್‌ಗಳಿಗೆ `./` ಅನ್ನು ಮತ್ತು ಪೋಷಕ ಡೈರೆಕ್ಟರಿಯವಕ್ಕೆ `../` ಅನ್ನು ಬಳಸಿ. ಉದಾಹರಣೆಗೆ: `[example](../../path/to/file)` ಅಥವಾ `[example](../../../path/to/file)`.

- **ದೇಶ-ನಿರ್ದಿಷ್ಟ ಲೋಕೇಲ್‌ಗಳು ಬೇಡ**: ನಿಮ್ಮ ಲಿಂಕ್‌ಗಳಲ್ಲಿ ದೇಶ-ನಿರ್ದಿಷ್ಟ ಲೋಕೇಲ್‌ಗಳನ್ನು ಸೇರಿಸಬೇಡಿ. ಉದಾಹರಣೆಗೆ, `/en-us/` ಅಥವಾ `/en/` ಬಳಸಬೇಡಿ.

- **ಇಮೇಜ್ ಸಂಗ್ರಹಣೆ**: ಎಲ್ಲಾ ಚಿತ್ರಗಳನ್ನು `./imgs` ಫೋಲ್ಡರ್‌ನಲ್ಲಿ ಸಂಗ್ರಹಿಸಿ.

- **ವಿವರಣಾತ್ಮಕ ಇಮೇಜ್ ಹೆಸರುಗಳು**: ಇಮೇಜ್‌ಗಳ ಹೆಸರನ್ನು ಇಂಗ್ಲಿಷ್ ಅಕ್ಷರಗಳು, ಸಂಖ್ಯೆ ಮತ್ತು ಡ್ಯಾಶ್‌ಗಳನ್ನು ಬಳಸಿ ವಿವರಣಾತ್ಮಕವಾಗಿ ಹೆಸರಿಡಿ. ಉದಾಹರಣೆಗೆ: `example-image.jpg`.

### Check Broken Relative Paths

ಈ ಕಾರ್ಯಪ್ರವಾಹವು ನಿಮ್ಮ ಫೈಲ್‌ಗಳಲ್ಲಿನ ಎಲ್ಲಾ ಸಂಬಂಧಿತ ಮಾರ್ಗಗಳು ಸರಿಯಾಗಿವೆ ಎಂದು ಖಚಿತಪಡಿಸುತ್ತದೆ.

1. ನಿಮ್ಮ ಲಿಂಕ್‌ಗಳು ಸರಿಯಾಗಿ ಕೆಲಸ ಮಾಡುತ್ತಿವೆ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಲು, VS Code ಬಳಸಿ ಕೆಳಗಿನ ಕಾರ್ಯಗಳನ್ನು ಮಾಡಿ:
    - ಫೈಲ್‌ಗಳಲ್ಲಿನ ಯಾವುದೇ ಲಿಂಕ್‌ನ ಮೇಲೆ ಹವರ್ ಮಾಡಿ.
    - ಲಿಂಕ್‌ಗೆ ನಾವಿಗೇಟ್ ಮಾಡಲು **Ctrl + Click** ಒತ್ತಿ.
    - ನೀವು ಲಿಂಕ್ ಕ್ಲಿಕ್ ಮಾಡಿದಾಗ ಅದು ಸ್ಥಳೀಯವಾಗಿ ಕೆಲಸ ಮಾಡದಿದ್ದರೆ, ಅದು ಕಾರ್ಯಪ್ರವಾಹವನ್ನು ಪ್ರಚೋದಿಸಿ GitHub ನಲ್ಲಿ ಕೂಡ ಕೆಲಸ ಮಾಡುವುದಿಲ್ಲ.

1. ಈ ಸಮಸ್ಯೆಯನ್ನು ಸರಿಪಡಿಸಲು, VS Code ನೀಡುವ ಪಥ ಸೂಚನೆಗಳನ್ನು ಬಳಸಿಕೊಂಡು ಕೆಳಗಿನ ಕಾರ್ಯಗಳನ್ನು ಮಾಡಿ:
    - `./` ಅಥವಾ `../` ಟೈಪ್ ಮಾಡಿ.
    - ನೀವು ಟೈಪ್ ಮಾಡಿದ ಆಧಾರದ ಮೇಲೆ ಲಭ್ಯವಿರುವ ಆಯ್ಕೆಗಳ ಪಟ್ಟಿ VS Code ನಿಮಗೆ ಸೂಚಿಸುತ್ತದೆ.
    - ನಿಮ್ಮ ಸರಿಯಾದ ಮಾರ್ಗವನ್ನು ಖಚಿತಪಡಿಸಲು ಬೇಕಾದ ಫೈಲ್ ಅಥವಾ ಫೋಲ್ಡರ್ ಅನ್ನು ಕ್ಲಿಕ್ ಮಾಡಿ ಮತ್ತು ಆ ಮಾರ್ಗವನ್ನು ಅನುಸರಿಸಿ.

ಒಮ್ಮೆ ನೀವು ಸರಿಯಾದ ಸಂಬಂಧಿತ ಮಾರ್ಗವನ್ನು ಸೇರಿಸಿದ ನಂತರ, ನಿಮ್ಮ ಬದಲಾವಣೆಗಳನ್ನು ಉಳಿಸಿ ಮತ್ತು ಪುಶ್ ಮಾಡಿ.

### Check URLs Don't Have Locale

ಈ ಕಾರ್ಯಪ್ರವಾಹವು ಯಾವುದೇ ವೆಬ್ URL ನಲ್ಲಿ ದೇಶ-ನಿರ್ದಿಷ್ಟ ಲೋಕೇಲ್ ಇರದಿರುವುದನ್ನು ಖಚಿತಪಡಿಸುತ್ತದೆ. ಈ ರೆಪೊಗಿರಲು ಜಾಗತಿಕವಾಗಿ ಪ್ರವೇಶಸಾಧ್ಯವಾಗಿರುವ ಕಾರಣ, URL ಗಳು ನಿಮ್ಮ ದೇಶದ ಲೋಕೇಲ್ ಅನ್ನು ಹೊಂದಿಲ್ಲವೆಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳುವುದು ಮಹತ್ವದ್ದಾಗಿದೆ.

1. ನಿಮ್ಮ URL ಗಳಲ್ಲಿ ದೇಶ-ಲೋಕೇಲ್ ಇಲ್ಲದಿರುವುದನ್ನು ಪರಿಶೀಲಿಸಲು ಕೆಳಗಿನ ಕಾರ್ಯಗಳನ್ನು ಮಾಡಿ:

    - URL ಗಳಲ್ಲಿ `/en-us/`, `/en/`, ಅಥವಾ ಯಾವುದೇ ಇತರೆ ಭಾಷಾ ಲೋಕೇಲ್ ಹೀಗೆಯೇ ಇದ್ದೇ ಇರೋದನ್ನು ಪರಿಶೀಲಿಸಿ.
    - ಈ ರೀತಿ ಯಾವುದೇ ಲೋಕೇಲ್ ಲಭ್ಯವಿಲ್ಲದಿದ್ದರೆ, ನೀವು ಈ ತಪಾಸಣೆಯಲ್ಲಿ ಪಾಸಾಗುತ್ತೀರಿ.

1. ಈ ಸಮಸ್ಯೆಯನ್ನು ಸರಿಪಡಿಸಲು, ಕೆಳಗಿನ ಕಾರ್ಯಗಳನ್ನು ಮಾಡಿ:
    - ಕಾರ್ಯಪ್ರವಾಹದಿಂದ ಹೈಲೈಟ್ ಮಾಡಿದ ಫೈಲ್ ಪಥವನ್ನು ತೆರೆಯಿರಿ.
    - URL ಗಳಿಂದ ದೇಶದ ಲೋಕೇಲ್ ಅನ್ನು ತೆಗೆದುಹಾಕಿ.

ನೀವು ದೇಶ-ಲೋಕೇಲ್ ಅನ್ನು ತೆಗೆದುಹಾಕಿದ ನಂತರ, ನಿಮ್ಮ ಬದಲಾವಣೆಗಳನ್ನು ಉಳಿಸಿ ಮತ್ತು ಪುಶ್ ಮಾಡಿ.

### Check Broken Urls

ಈ ಕಾರ್ಯಪ್ರವಾಹವು ನಿಮ್ಮ ಫೈಲ್‌ಗಳಲ್ಲಿ ಇರುವ ಯಾವುದೇ ವೆಬ್ URL ಕೆಲಸ ಮಾಡುತ್ತದೆಯೇ ಮತ್ತು 200 ಸ್ಥಿತಿ ಕೋಡ್ ಅನ್ನು ಹುಡುಕುತ್ತದೆ ಎಂದು ಖಚಿತಪಡಿಸುತ್ತದೆ.

1. ನಿಮ್ಮ URL ಗಳು ಸರಿಯಾಗಿ ಕೆಲಸ ಮಾಡುತ್ತಿವೆಯೇ ಎಂದು ಪರಿಶೀಲಿಸಲು ಕೆಳಗಿನ ಕಾರ್ಯಗಳನ್ನು ಮಾಡಿ:

    - URL ಅನ್ನು ಸರಿಯಾದದಾಗಿ ಅಪ್ಡೇಟ್ ಮಾಡಿ.
