## ಸುರಕ್ಷತೆ

Microsoft ನಮ್ಮ ಸಾಫ್ಟ್‌ವೇರ್ ಉತ್ಪನ್ನಗಳು ಮತ್ತು ಸೇವೆಗಳ ಸುರಕ್ಷತೆಯನ್ನು ಗಂಭೀರತೆಯಿಂದ ತೆಗೆದುಕೊಳ್ಳುತ್ತದೆ, ಇದರಲ್ಲಿ ನಮ್ಮ GitHub ಸಂಸ್ಥೆಗಳ ಮುಖಾಂತರ ನಿರ್ವಹಿಸಲಾದ ಎಲ್ಲಾ ಮೂಲ ಕೋಡ್ ರೆಪೊಸಿಟರಿಗಳು ಸೇರಿವೆ, ಅವುಗಳಲ್ಲಿ [Microsoft](https://github.com/Microsoft), [Azure](https://github.com/Azure), [DotNet](https://github.com/dotnet), [AspNet](https://github.com/aspnet) ಮತ್ತು [Xamarin](https://github.com/xamarin) ಒಳಗ आतीವೆ.

ನೀವು ಯಾವುದೇ Microsoft-ಸ್ವಂತ ರೆಪೊಸಿಟರಿಯಲ್ಲಿ [Microsoft's definition of a security vulnerability](https://aka.ms/security.md/definition) ಅನ್ನು ಪೂರೈಸುವ ಸುರಕ್ಷತಾ ದುರ್ಬಲತೆಯನ್ನು ಕಂಡುಹಿಡಿದಿದ್ದೀರಿ ಎಂದು ನೀವು ನಂಬಿದರೆ, ದಯವಿಟ್ಟು ಕೆಳಗಿನಂತೆ ನಮಗೆ ವರದಿ ಮಾಡಿ.

## ಸುರಕ್ಷತಾ ಸಮಸ್ಯೆಗಳ ವರದಿ

**ದಯವಿಟ್ಟು ಸಾರ್ವಜನಿಕ GitHub issues ಮೂಲಕ ಸುರಕ್ಷತಾ ದುರ್ಬಲತೆಗಳನ್ನು ವರದಿ ಮಾಡುವುದನ್ನು ತಪ್ಪಿಸಿ.**

ಬದಲಿಗೆ, ದಯವಿಟ್ಟು ಅವುಗಳನ್ನು Microsoft Security Response Center (MSRC) ಗೆ ವರದಿ ಮಾಡಿ: [https://msrc.microsoft.com/create-report](https://aka.ms/security.md/msrc/create-report).

ಲಾಗಿನ್ ಮಾಡದೆ ಸಲ್ಲಿಸಲು ಇಚ್ಛಿಸುವವರು, ದಯವಿಟ್ಟು [secure@microsoft.com](mailto:secure@microsoft.com) ಗೆ ಇಮೇಲ್ ಕಳುಹಿಸಿ. ಸಾಧ್ಯವಾದರೆ, ದಯವಿಟ್ಟು ನಮ್ಮ PGP ಕೀಯಿಂದ ನಿಮ್ಮ ಸಂದೇಶವನ್ನು ಎನ್‌ಕ್ರಿಪ್ಟ್ ಮಾಡಿ; ದಯವಿಟ್ಟು ಅದನ್ನು [Microsoft Security Response Center PGP Key page](https://aka.ms/security.md/msrc/pgp) ನಿಂದ ಡೌನ್‌ಲೋಡ್ ಮಾಡಿ.

ನೀವು 24 ಗಂಟೆಗಳ ಒಳಗೆ ಉತ್ತರವನ್ನು ಪಡೆಯಬೇಕು. ಯಾವುದೇ ಕಾರಣಕ್ಕಾಗಿ ನೀವು ಉತ್ತರವನ್ನು ಪಡೆಯದಿದ್ದರೆ, ದಯವಿಟ್ಟು ನಮ್ಮ ಮೂಲ संदेशವನ್ನು ಸ್ವೀಕರಿಸಿದ್ದೇವೆ ಎಂಬುದನ್ನು ಖಚಿತಪಡಿಸಲು ಇಮೇಲ್ ಮೂಲಕ ಫಾಲೋಅಪ್ ಮಾಡಿ. ಹೆಚ್ಚಿನ ಮಾಹಿತಿಯು [microsoft.com/msrc](https://www.microsoft.com/msrc) ನಲ್ಲಿ ಲಭ್ಯವಿದೆ.

ದಯವಿಟ್ಟು ಕೆಳಗಿನಂತೆ ವಿನಂತಿಸಲಾದ ಮಾಹಿತಿಯನ್ನು (ನಿರೀಕ್ಷಿಸಬಹುದಾದಷ್ಟು) ನಮ್ಮೊಂದಿಗೆ ಸೇರಿಸಿ, ಇದರಿಂದ ಸಾಧ್ಯವಿರುವ ಸಮಸ್ಯೆಯ ಸ್ವಭಾವ ಮತ್ತು ವ್ಯಾಪ್ತಿಯನ್ನು ನಾವು ಉತ್ತಮವಾಗಿ ಅರ್ಥಮಾಡಿಕೊಳ್ಳಲು ಸಹಾಯವಾಗುತ್ತದೆ:

  * ಸಮಸ್ಯೆಯ ಪ್ರಕಾರ (ಉದಾಹರಣೆಗೆ buffer overflow, SQL injection, cross-site scripting ಇತ್ಯಾದಿ)
  * ಸಮಸ್ಯೆಯ ಪರಿಗಣನೆಯೊಂದಿಗೆ ಸಂಬಂಧಿಸಿದ ಮೂಲ ಫೈಲ್(ಗಳ) ಸಂಪೂರ್ಣ ಪಾಥ್‌ಗಳು
  * ಪ್ರಭಾವಿತ ಮೂಲ ಕೋಡ್‌ನ ಸ್ಥಳ (tag/branch/commit ಅಥವಾ ನಿರ್ಧಿಷ್ಟ URL)
  * ಸಮಸ್ಯೆಯನ್ನು ಪುನರಾವರ್ತಿಸಲು ಯಾವುದೇ ವಿಶೇಷ ಸಂರಚನೆ ಬೇಕಾದರೆ ಅದರ ವಿವರ
  * ಸಮಸ್ಯೆಯನ್ನು ಪುನರಾವರ್ತಿಸಲು ಹಂತಕ್ಕೆ ಹಂತದ ಸೂಚನೆಗಳು
  * ಪ್ರೂಫ್-ಆಫ್-ಕಾನ್‌ಸೆಪ್ಟ್ ಅಥವಾ ಎಕ್ಸ್‌ಪ್ಲಾಯಿಟ್ ಕೋಡ್ (ಸಂಭವವಾಗಿದ್ದರೆ)
  * ಸಮಸ್ಯೆಯ ಪ್ರಭಾವ, ಅಟಾಕರ್ ಎಂತಹ ರೀತಿ ಈ ಸಮಸ್ಯೆಯನ್ನು ದುರ್ಬಳಕೆ ಮಾಡಬಹುದು ಎಂಬದನ್ನು ಸೇರಿಸಿ

ಈ ಮಾಹಿತಿ ನಮ್ಮಿಗೆ ನಿಮ್ಮ ವರದಿಯನ್ನು ವೇಗವಾಗಿ ತಿರಾಜಿಸುವುದಕ್ಕೆ ಸಹಾಯ ಮಾಡುತ್ತದೆ.

ನೀವು ಬಗ್ ಬೌಂಟಿಗಾಗಿ ವರದಿ ನೀಡುತ್ತಿದ್ದರೆ, ಪೂರ್ಣ ವರದಿಗಳು ಹೆಚ್ಚಿನ ಬೌಂಟಿ ಬಹುಮಾನಕ್ಕೆ ಕಾರಣವಾಗಬಹುದು. ದಯವಿಟ್ಟು ನಮ್ಮ ಕ್ರಿಯಾಶೀಲ ಕಾರ್ಯಕ್ರಮಗಳ ಕುರಿತು ಹೆಚ್ಚಿನ ವಿವರಗಳಿಗೆ [Microsoft Bug Bounty Program](https://aka.ms/security.md/msrc/bounty) ಪುಟವನ್ನು ಭೇಟಿ ಮಾಡಿ.

## ಮೆಚ್ಚುವ ಭಾಷೆಗಳು

ನಾವು ಎಲ್ಲಾ ಸಂವಹನಗಳನ್ನು ಇಂಗ್ಲಿಷ್‌ನಲ್ಲಿ ನಡೆಸಲು ಇಷ್ಟಪಡುತ್ತೇವೆ.

## ನೀತಿ

Microsoft [ಸಂಯೋಜಿತ ದುರ್ಬಲತೆ ಬಹಿರಂಗಪಡಿಸುವಿಕೆ](https://aka.ms/security.md/cvd) ತತ್ವವನ್ನು ಅನುಸರಿಸುತ್ತದೆ.
