# ನಿಮ್ಮ VS Code ವಿಸ್ತರಣೆಗೆ ಸ್ವಾಗತ

## ಫೋಲ್ಡರ್‌ನಲ್ಲಿ ಏನು ಇದೆ

* ಈ ಫೋಲ್ಡರ್‌ನಲ್ಲಿ ನಿಮ್ಮ ವಿಸ್ತರಣೆಗೆ ಅಗತ್ಯವಿರುವ ಎಲ್ಲಾ ಫೈಲ್‌ಗಳು ಸೇರಿವೆ.
* `package.json` - ಇದು ನಿಮ್ಮ ವಿಸ್ತರಣೆಯನ್ನು ಮತ್ತು ಕಮಾಂಡ್ ಅನ್ನು ಘೋಷಿಸುವ ಮೆನಿಫೆಸ್ಟ್ ಫೈಲ್.
  * ಮಾದರಿ ಪ್ಲಗಿನ್ ಒಂದು ಕಮಾಂಡ್ ಅನ್ನು ನೋಂದಣಿ ಮಾಡುತ್ತದೆ ಮತ್ತು ಅದರ ಶೀರ್ಷಿಕೆ ಮತ್ತು ಕಮಾಂಡ್ ಹೆಸರನ್ನು ನಿರ್ಧರಿಸುತ್ತದೆ. ಈ ಮಾಹಿತಿಯೊಂದಿಗೆ VS Code ಆ ಕಮಾಂಡ್ ಅನ್ನು ಕಮಾಂಡ್ ಪ್ಯಾಲೆಟ್‌ನಲ್ಲಿ ತೋರಿಸಬಹುದು. ಇದಕ್ಕೆ ತಕ್ಷಣವೇ ಪ್ಲಗಿನ್ ಅನ್ನು ಲೋಡ್ ಮಾಡಬೇಕಾಗುವುದಿಲ್ಲ.
* `src/extension.ts` - ಇದು ಮುಖ್ಯ ಫೈಲ್ ಆಗಿದ್ದು ನೀವು ನಿಮ್ಮ ಕಮಾಂಡ್‌ನ ಅನುಷ್ಠಾನವನ್ನು ಇಲ್ಲಿ ಒದಗಿಸುತ್ತೀರಿ.
  * ಫೈಲ್ ಒಂದು ಫಂಕ್ಶನ್ ಅನ್ನು ಎಕ್ಸ್ಪೋರ್ಟ್ ಮಾಡುತ್ತದೆ, `activate`, որը ನಿಮ್ಮ ವಿಸ್ತರಣೆ ಮೊದಲ ಬಾರಿಗೆ ಆಕ್ಟಿವೇಟ್ ಆಗಿದಾಗ ಕರೆಮಾಡಲಾಗುತ್ತದೆ (ಈ ಪ್ರಕರಣದಲ್ಲಿ ಕಮಾಂಡ್ ಅನ್ನು 실행ಿಸುವ ಮೂಲಕ). `activate` ಫಂಕ್ಶನ್ ಒಳಗರಲ್ಲಿ ನಾವು `registerCommand` ಅನ್ನು ಕರೆಮಾಡುತ್ತೇವೆ.
  * ನಾವು `registerCommand` ಗೆ ಕಮಾಂಡ್‌ನ ಅನುಷ್ಠಾನವನ್ನು ಹೊಂದಿರುವ ಫಂಕ್ಶನ್ ಅನ್ನು ಎರಡನೇ ಪ್ಯಾರಾಮೀಟರ್ ಆಗಿ ಪಾಸ್ ಮಾಡುತ್ತೇವೆ.

## ಸೆಟ್‌ಅಪ್

* ಶಿಫಾರಸು ಮಾಡಲ್ಪಟ್ಟ ವಿಸ್ತರಣೆಗಳನ್ನು ಸ್ಥಾಪಿಸಿ (amodio.tsl-problem-matcher, ms-vscode.extension-test-runner, ಮತ್ತು dbaeumer.vscode-eslint)


## ತಕ್ಷಣವೇ ಪ್ರಾರಂಭಿಸಿರಿ

* ನಿಮ್ಮ ವಿಸ್ತರಣೆ ಲೋಡ್ ಹೊಂದಿರುವ ಹೊಸ ವಿಂಡೋವನ್ನು ತೆರೆಯಲು `F5` ಒತ್ತಿ.
* (`Ctrl+Shift+P` or `Cmd+Shift+P` on Mac) ಒತ್ತಿ ಮತ್ತು `Hello World` ಟೈಪ್ ಮಾಡಿ, ಕಮಾಂಡ್ ಪ್ಯಾಲೆಟ್‌ನಿಂದ ನಿಮ್ಮ ಕಮಾಂಡ್ ಅನ್ನು 실행ಿಸಿ.
* ನಿಮ್ಮ ವಿಸ್ತರಣೆಯನ್ನು ಡಿಬಗ್ ಮಾಡುವಂತೆ `src/extension.ts` ಒಳಗಿನ ನಿಮ್ಮ ಕೋಡ್‌ನಲ್ಲಿ ಬ್ರೇಕ್‌ಪಾಯಿಂಟ್‌ಗಳನ್ನು ಸೆಟ್ ಮಾಡಿ.
* ಡಿಬಗ್ ಕನ್‌ಸೋಲ್‌ನಲ್ಲಿ ನಿಮ್ಮ ವಿಸ್ತರಣೆಯಿಂದ ಆಗುವ ಔಟ್‌ಪುಟ್ ಅನ್ನು ಕಂಡುಹಿಡಿಯಿರಿ.

## ಬದಲಾವಣೆಗಳನ್ನು ಮಾಡಿ

* `src/extension.ts` ನಲ್ಲಿ ಕೋಡ್ ಬದಲಾವಣೆ ಮಾಡಿದ ನಂತರ ಡಿಬಗ್ ಟೂಲ್‌ಬಾರ್‌ನಿಂದ ವಿಸ್ತರಣೆಯನ್ನು ಮರುಪ್ರಾರಂಭಿಸಬಹುದು.
* ನಿಮ್ಮ ಬದಲಾವಣೆಗಳನ್ನು ಲೋಡ್ ಮಾಡಲು VS Code ವಿಂಡೋವನ್ನು ನಿಮ್ಮ ವಿಸ್ತರಣೆಯೊಂದಿಗೆ ರೀಲೋಡ್ (`Ctrl+R` ಅಥವಾ `Cmd+R` ಮ್ಯಾಕ್‌ನಲ್ಲಿ) ಮಾಡಬಹುದು.


## API ಅನ್ನು ಅನ್ವೇಷಿಸಿ

* `node_modules/@types/vscode/index.d.ts` ಫೈಲ್ ಅನ್ನು ತೆರೆಯುವಾಗ ನೀವು ನಮ್ಮ API ನ ಪೂರ್ಣ ಸೆಟ್ ಅನ್ನು ತೆರೆಯಬಹುದು.

## ಪರೀಕ್ಷೆಗಳು ನಡೆಸಿ

* [Extension Test Runner](https://marketplace.visualstudio.com/items?itemName=ms-vscode.extension-test-runner) ಅನ್ನು ಸ್ಥಾಪಿಸಿ
* **Tasks: Run Task** ಕಮಾಂಡ್ ಮೂಲಕ "watch" ಟಾಸ್ಕ್ ಅನ್ನು ರನ್ ಮಾಡಿ. ಇದು ಚಲಿಸುತ್ತಿರುವಂತೆ ಖಚಿತಪಡಿಸಿಕೊಂಡಿರಲಿ, ಇಲ್ಲದಿದ್ದರೆ ಪರೀಕ್ಷೆಗಳು ಕಂಡುಹಿಡಿಯಲಾಗದು.
* ಚಟುವಟಿಕೆ ಬಾರ್‌ನಿಂದ Testing view ಅನ್ನು ತೆರೆಯಿರಿ ಮತ್ತು Run Test" ಬಟನ್ ಅನ್ನು ಕ್ಲಿಕ್ ಮಾಡಿ, ಅಥವಾ ಹಾಟ್‌ಕೀ `Ctrl/Cmd + ; A` ಅನ್ನು ಉಪಯೋಗಿಸಿ
* Test Results view ನಲ್ಲಿ ಟೆಸ್ಟ್ ಫಲಿತಾಂಶದ ಔಟ್‌ಪುಟ್ ಅನ್ನು ನೋಡಿ.
* `src/test/extension.test.ts` ನಲ್ಲಿ ಬದಲಾವಣೆಗಳನ್ನು ಮಾಡಿ ಅಥವಾ `test` ಫೋಲ್ಡರ್ ಒಳಗಿನಲ್ಲಿ ಹೊಸ ಟೆಸ್ಟ್ ಫೈಲ್‌ಗಳನ್ನು ರಚಿಸಿ.
  * ಒದಗಿಸಲಾದ ಟೆಸ್ಟ್ ರನ್ನರ್ ಕೇವಲ `**.test.ts` ಎಂಬ ಹೆಸರು ಮಾದರಿಯನ್ನು ಹೊಂದಿರುವ ಫೈಲ್‌ಗಳನ್ನು ಮಾತ್ರ ಪರಿಗಣಿಸುತ್ತದೆ.
  * ನೀವು ನಿಮ್ಮ ಟೆಸ್ಟ್‌ಗಳನ್ನು ಇಚ್ಛೆಯಂತೆ ರಚಿಸಲು `test` ಫೋಲ್ಡರ್ ಒಳಗಿನಲ್ಲಿ ಫೋಲ್ಡರ್‌ಗಳನ್ನು ರಚಿಸಬಹುದು.

## ಮುಂದೆ ಹೋಗಿ

* ನಿಮ್ಮ ವಿಸ್ತರಣೆಯ ಗಾತ್ರವನ್ನು ಕಡಿಮೆ ಮಾಡಿ ಮತ್ತು ಸ್ಟಾರ್ಟ್‌ಅಪ್ ಸಮಯವನ್ನು ಸುಧಾರಿಸಲು [ನಿಮ್ಮ ವಿಸ್ತರಣೆಯನ್ನು ಬ್ಯಂಡಲ್ ಮಾಡುವುದು](https://code.visualstudio.com/api/working-with-extensions/bundling-extension).
* VS Code ವಿಸ್ತರಣೆ ಮಾರುಕಟ್ಟೆಯಲ್ಲಿ [ನಿಮ್ಮ ವಿಸ್ತರಣೆಯನ್ನು ಪ್ರಕಟಿಸಿ](https://code.visualstudio.com/api/working-with-extensions/publishing-extension).
* [ನಿರಂತರ ಏಕೀಕರಣ](https://code.visualstudio.com/api/working-with-extensions/continuous-integration) ಅನ್ನು ಹೊಂದಿಸಿ ಬಿಲ್ಡ್‌ಗಳನ್ನು ಸ್ವಯಂಚಾಲಿತಗೊಳಿಸಿ.
