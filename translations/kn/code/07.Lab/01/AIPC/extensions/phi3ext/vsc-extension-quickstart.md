# ನಿಮ್ಮ VS Code ವಿಸ್ತರಣೆಗೆ ಸ್ವಾಗತ

## ಫೋಲ್ಡರ್‌ನಲ್ಲಿ ಏನು ಇದೆ

* ಈ ಫೋಲ್ಡರ್ ನಿಮ್ಮ ವಿಸ್ತರಣೆಗೆ ಅಗತ್ಯವಿರುವ ಎಲ್ಲಾ ಫೈಲ್‌ಗಳನ್ನು ಒಳಗೊಂಡಿದೆ.
* `package.json` - ಇದು ನಿಮ್ಮ ವಿಸ್ತರಣೆ ಮತ್ತು ಕಮಾಂಡ್ ಅನ್ನು ಘೋಷಿಸುವ ಮ್ಯಾನಿಫೆಸ್ಟ್ ಫೈಲ್.
  * ಉದಾಹರಣೆಯ ಪ್ಲಗಿನ್ ಒಂದು ಕಮಾಂಡ್ ಅನ್ನು ನೋಂದಣಿ ಮಾಡಿ ಅದರ ಶಿರೋನಾಮೆ ಮತ್ತು ಕಮಾಂಡ್ ಹೆಸರನ್ನು ನಿರ್ಧರಿಸುತ್ತದೆ. ಈ ಮಾಹಿತಿಯ ಮೂಲಕ VS Code ಆ ಕಮಾಂಡ್ ಅನ್ನು ಕಮಾಂಡ್ ಪ್ಯಾಲೆಟ್‌ನಲ್ಲಿ ತೋರಿಸಬಹುದು. ಈ ಸಂದರ್ಭದಲ್ಲಿ ಪ್ಲಗಿನ್ ಅನ್ನು ಲೋಡ್ ಮಾಡಬೇಕಾಗಿಲ್ಲ.
* `src/extension.ts` - ಇದು ಮೇನ್ ಫೈಲ್ ಆಗಿದ್ದು, ಇಲ್ಲಿ ನೀವು ನಿಮ್ಮ ಕಮಾಂಡ್‌ನ ಅನುಷ್ಠಾನವನ್ನು ಒದಗಿಸುತ್ತೀರಿ.
  * ಫೈಲ್ ಒಂದು ಫಂಕ್ಷನ್ ಅನ್ನು ಎಕ್ಸ್ಪೋರ್ಟ್ ಮಾಡುತ್ತದೆ, `activate`, ಇದು ನಿಮ್ಮ ವಿಸ್ತರಣೆ ಮೊದಲ ಬಾರಿ ಸಕ್ರಿಯಗೊಳ್ಳುವಾಗ (ಈ ಪ್ರಕರಣದಲ್ಲಿ ಕಮಾಂಡ್ ಅನ್ನು ನಿರ್ವಹಿಸುವ ಮೂಲಕ) ಕರೆಸಲಾಗುತ್ತದೆ. `activate` ಫಂಕ್ಷನ್ ಒಳಗೆ ನಾವು `registerCommand` ಅನ್ನು ಕರೆಸುತ್ತೇವೆ.
  * ನಾವು ಕಮಾಂಡ್‌ನ ಅನುಷ್ಠಾನವನ್ನು ಹೊಂದಿರುವ ಫಂಕ್ಷನ್ ಅನ್ನು `registerCommand` ಗೆ ಎರಡನೇ ಪ್ಯಾರಾಮೀಟರ್ ಆಗಿ ಪಾಸ್ ಮಾಡುತ್ತೇವೆ.

## ಸೆಟಪ್

* ಶಿಫಾರಸು ಮಾಡಲಾದ ವಿಸ್ತರಣೆಗಳನ್ನು ಸ್ಥಾಪಿಸಿ (amodio.tsl-problem-matcher, ms-vscode.extension-test-runner, and dbaeumer.vscode-eslint)


## ತಕ್ಷಣವೇ ಪ್ರಾರಂಭಿಸಿ

* ಹೊಸ ವಿಂಡೋವನ್ನು ನಿಮ್ಮ ವಿಸ್ತರಣೆ ಲೋಡ್ ಆಗಿರುವಂತೆ ತೆರೆಯಲು `F5` ಒತ್ತಿರಿ.
* (`Ctrl+Shift+P` or `Cmd+Shift+P` on Mac) ಒತ್ತಿ ಮತ್ತು `Hello World` ಎಂದು ಟೈಪ್ ಮಾಡಿ ಕಮಾಂಡ್ ಪ್ಯಾಲೆಟ್‌ನಿಂದ ನಿಮ್ಮ ಕಮಾಂಡ್ ಅನ್ನು ಚಾಲನೆ ಮಾಡಿ.
* ನಿಮ್ಮ ವಿಸ್ತರಣೆಯನ್ನು ಡಿಬಗ್ ಮಾಡಲು `src/extension.ts` ಒಳಗಿನ ನಿಮ್ಮ ಕೋಡ್‌ನಲ್ಲಿ ಬ್ರೇಕ್‌പಾಯಿಂಟ್‌ಗಳನ್ನು ಸೆಟ್ ಮಾಡಿ.
* ಡಿಬಾಗ್ ಕನ್‌ಸೋಲ್‌ನಲ್ಲಿ ನಿಮ್ಮ ವಿಸ್ತರಣೆಯಿಂದ ಉತ್ಸವವನ್ನು ಹುಡುಕಿ.

## ಬದಲಾವಣೆಗಳನ್ನು ಮಾಡಿ

* `src/extension.ts` ನಲ್ಲಿ ಕೋಡ್ ಬದಲಿಸಿದ ನಂತರ ಡಿಬಾಗ್ ಟೂಲ್ಬಾರ್‌ನಿಂದ ವಿಸ್ತರಣೆಯನ್ನು ಪುನಃಪ್ರಾರಂಭಿಸಬಹುದು.
* ನಿಮ್ಮ ಬದಲಾವಣೆಗಳನ್ನು ಲೋಡ್ ಮಾಡಲು VS Code ವಿಂಡೋವನ್ನು ಪುನಃಲೋಡ್ (`Ctrl+R` or `Cmd+R` on Mac) ಮಾಡಲು ಸಹ μπορείτε.

## API ಅನ್ನು ಅನ್ವೇಷಿಸಿ

* `node_modules/@types/vscode/index.d.ts` ಫೈಲ್ ಅನ್ನು ತೆರೆಯುವಾಗ ನೀವು ನಮ್ಮ API ಯ ಸಂಪೂರ್ಣ ಸೆಟ್ ಅನ್ನು ತೆರೆಯಬಹುದು.

## ಟೆಸ್ಟ್‌ಗಳನ್ನು ಚಾಲನೆ ಮಾಡಿ

* [ವಿಸ್ತರಣೆ ಪರೀಕ್ಷಾ ರನ್ನರ್](https://marketplace.visualstudio.com/items?itemName=ms-vscode.extension-test-runner) ಅನ್ನು ಸ್ಥಾಪಿಸಿ
* **Tasks: Run Task** ಕಮಾಂಡ್ ಮೂಲಕ "watch" ಟಾಸ್ಕ್ ಅನ್ನು ಚಾಲನೆ ಮಾಡಿ. ಇದು ಚಲಿಸಲಾಗುತ್ತಿರುವುದನ್ನು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ, ಇಲ್ಲದಿದ್ದರೆ ಟೆಸ್ಟ್‌ಗಳು ಕಂಡುಕೊಳ್ಳುವುದಿಲ್ಲ.
* activity bar ನಿಂದ Testing view ಅನ್ನು ತೆರೆಯಿರಿ ಮತ್ತು Run Test" ಬಟನ್ ಕ್ಲಿಕ್ ಮಾಡಿ, ಅಥವಾ ಹಾಟ್‌ಕೀ `Ctrl/Cmd + ; A` ಅನ್ನು ಬಳಸಿ
* Test Results view ನಲ್ಲಿ ಟೆಸ್ಟ್ ಫಲಿತಾಂಶದ ಔಟ್‌ಪುಟ್ ಅನ್ನು ನೋಡಿ.
* `src/test/extension.test.ts` ನಲ್ಲಿ ಬದಲಾವಣೆಗಳನ್ನು ಮಾಡಿರಿ ಅಥವಾ `test` ಫೋಲ್ಡರ್ ಒಳಗೆ ಹೊಸ ಟೆಸ್ಟ್ ಫೈಲ್‌ಗಳನ್ನು ನಿರ್ಮಿಸಿ.
  * ಒದಗಿಸಲಾದ ಟೆಸ್ಟ್ ರನ್ನರ್ ಕೇವಲ `**.test.ts` ಎಂಬ ಹೆಸರು ಮಾದರಿಯನ್ನು ಹೊಂದುವ ಫೈಲ್‌ಗಳನ್ನೇ ಪರಿಗಣಿಸುತ್ತದೆ.
  * ನೀವು ನಿಮ್ಮ ಟೆಸ್ಟ್‌ಗಳನ್ನು ಯಾವುದೇ ರೀತಿಯಾಗಿ ರಚಿಸಲು `test` ಫೋಲ್ಡರ್ ಒಳಗೆ ಫೋಲ್ಡರ್‌ಗಳನ್ನು ರಚಿಸಬಹುದು.

* [ನಿಮ್ಮ ವಿಸ್ತರಣೆಯನ್ನು ಪ್ರಕಟಿಸಿ](https://code.visualstudio.com/api/working-with-extensions/publishing-extension?WT.mc_id=aiml-137032-kinfeylo) VS Code ಎಕ್ಸ್ಟೆಂಶನ್ ಮಾರ್ಕೆಟ್‌ಪ್ಲೇಸ್‌ನಲ್ಲಿ.
