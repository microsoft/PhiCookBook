# ನಿಮ್ಮ VS Code ವಿಸ್ತರಣೆಗೆ ಸ್ವಾಗತ

## ಫೋಲ್ಡರ್‌ನಲ್ಲಿ ಏನು ಇದೆ

* ಈ ಫೋಲ್ಡರ್ ನಿಮ್ಮ ವಿಸ್ತರಣೆಗೆ ಅಗತ್ಯವಿರುವ ಎಲ್ಲಾ ಫೈಲ್‌ಗಳನ್ನು ಒಳಗೊಂಡಿದೆ.
* `package.json` - ಇದು ಮ್ಯಾನಿಫೆಸ್ಟ್ ಫೈಲ್ ಆಗಿದ್ದು, ನೀವು ನಿಮ್ಮ ವಿಸ್ತರಣೆ ಮತ್ತು ಕಮಾಂಡ್ ಅನ್ನು ಘೋಷಿಸುವ ಸ್ಥಳ.
  * ಉದಾಹರಣೆಯ ಪ್ಲಗಿನ್ ಒಂದು ಕಮಾಂಡ್ ಅನ್ನು ನೋಂದಾಯಿಸುತ್ತದೆ ಮತ್ತು ಅದರ ಶೀರ್ಷಿಕೆ ಮತ್ತು ಕಮಾಂಡ್ ಹೆಸರನ್ನು ನಿರ್ಧರಿಸುತ್ತದೆ. ಈ ಮಾಹಿತಿಯೊಂದಿಗೆ VS Code ಆ ಕಮಾಂಡ್ ಅನ್ನು ಕಮಾಂಡ್ ಪ್ಯಾಲೆಟ್‌ನಲ್ಲಿ ತೋರಿಸಬಹುದು. ಇದು ಇನ್ನೂ ಪ್ಲಗಿನ್ ಅನ್ನು ಲೋಡ್ ಮಾಡಲು ಅಗತ್ಯವಿಲ್ಲ.
* `src/extension.ts` - ಇದು ಮುಖ್ಯ ಫೈಲ್ ಆಗಿದ್ದು, ಇಲ್ಲಿ ನೀವು ನಿಮ್ಮ ಕಮಾಂಡ್‌ನ ಅನುಷ್ಠಾನವನ್ನು ಒದಗಿಸುತ್ತೀರಿ.
  * ಆ ಫೈಲ್ ಒಂದು ಫಂಕ್ಷನ್, `activate`, ಅನ್ನು ಎಕ್ಸ್‍ಪೋರ್ಟ್ ಮಾಡುತ್ತದೆ, ಇದು ನಿಮ್ಮ ವಿಸ್ತರಣೆ ಮೊದಲ ಬಾರಿಗೆ ಸಕ್ರಿಯಗೊಂಡಾಗ (ಈ ಪ್ರಕರಣದಲ್ಲಿ ಕಮಾಂಡ್ ಕಾರ್ಯಗತಗೊಳಿಸುವ ಮೂಲಕ) ಕರೆಯಲಾಗುತ್ತದೆ. `activate` ಫಂಕ್ಷನ್ ಒಳಗೆ ನಾವು `registerCommand` ಅನ್ನು ಕರೆ ಮಾಡುತ್ತೇವೆ.
  * ನಾವು ಕಮಾಂಡ್‌ನ ಅನುಷ್ಠಾನವನ್ನು ಒಳಗೊಂಡ ಫಂಕ್ಷನ್ ಅನ್ನು `registerCommand` ಗೆ ಎರಡನೇ ಪ್ಯಾರಾಮೀಟರ್ ಆಗಿ ಪಾಸ್ ಮಾಡುತ್ತೇವೆ.

## ಸೆಟಪ್

* ಶಿಫಾರಸು ಮಾಡಲಾದ ವಿಸ್ತರಣೆಗಳನ್ನು ಸ್ಥಾಪಿಸಿ (amodio.tsl-problem-matcher, ms-vscode.extension-test-runner, ಮತ್ತು dbaeumer.vscode-eslint)


## ಕೂಡಲೇ ಚಾಲನೆಗೊಳ್ಳಿ

* ನಿಮ್ಮ ವಿಸ್ತರಣೆಯೊಂದಿಗೆ ಹೊಸ ವಿಂಡೋ ತೆರೆಯಲು `F5` ಒತ್ತಿ.
* ಕಮಾಂಡ್ ಪ್ಯಾಲೆಟ್‌ನಿಂದ (`Ctrl+Shift+P` ಅಥವಾ ಮ್ಯಾಕ್‌ನಲ್ಲಿ `Cmd+Shift+P` ಒತ್ತಿ) ನಿಮ್ಮ ಕಮಾಂಡ್ ಅನ್ನು 실행 ಮಾಡಿ ಮತ್ತು `Hello World` ಅನ್ನು ಟೈಪ್ ಮಾಡಿ.
* ನಿಮ್ಮ ವಿಸ್ತರಣೆಯನ್ನು ಡಿಬಗ್ ಮಾಡಲು `src/extension.ts` ಒಳಗಿನ ನಿಮ್ಮ ಕೋಡ್‌ನಲ್ಲಿ ಬ್ರೇಕ್ ಪಾಯಿಂಟ್‌ಗಳನ್ನು ಸೆಟ್ ಮಾಡಿ.
* ಡಿಬಗ್ ಕಾನ್ಸೋಲ್‌ನಲ್ಲಿ ನಿಮ್ಮ ವಿಸ್ತರಣೆಯ ಔಟ್‌ಪುಟ್ ಅನ್ನು ಕಂಡುಹಿಡಿಯಿರಿ.

## ಬದಲಾವಣೆಗಳನ್ನು ಮಾಡಿ

* `src/extension.ts`ನಲ್ಲಿ ಕೋಡ್ ಬದಲಿಸಿದ ನಂತರ ಡಿಬಗ್ ಟೂಲ್‌ಬಾರ್‌ನಿಂದ ವಿಸ್ತರಣೆಯನ್ನು ಮರುಪ್ರಾರಂಭಿಸಬಹುದು.
* ನಿಮ್ಮ ಬದಲಾವಣೆಗಳನ್ನು ಲೋಡ್ ಮಾಡಲು ನಿಮ್ಮ ವಿಸ್ತರಣೆಯೊಂದಿಗೆ VS Code ವಿಂಡೋವನ್ನು ಮತ್ತೆ ಲೋಡ್ ಮಾಡಲು (`Ctrl+R` ಅಥವಾ ಮ್ಯಾಕ್‌ನಲ್ಲಿ `Cmd+R`) ಬಳಸಬಹುದು.


## API ಅನ್ನು ಅನ್ವೇಷಿಸಿ

* `node_modules/@types/vscode/index.d.ts` ಫೈಲ್ ತೆರೆಯುವ ಮೂಲಕ ನೀವು ನಮ್ಮ API ನ ಸಂಪೂರ್ಣ ಸೆಟ್ ಅನ್ನು ನೋಡಬಹುದು.

## ಟೆಸ್ಟ್‌ಗಳನ್ನು ನಡೆಸಿ

* [Extension Test Runner](https://marketplace.visualstudio.com/items?itemName=ms-vscode.extension-test-runner) ಅನ್ನು ಸ್ಥಾಪಿಸಿ
* **Tasks: Run Task** ಕಮಾಂಡ್ ಮುಖಾಂತರ "watch" ಟಾಸ್ಕ್ ಅನ್ನು ರನ್ ಮಾಡಿ. ಇದು ಚಾಲನೆಯಲ್ಲಿದೆಯೇ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ, ಇಲ್ಲದಿದ್ದರೆ ಟೆಸ್ಟ್‌ಗಳನ್ನು ಕಂಡುಹೊರಡಲಾಗದಿರಬಹುದು.
* ಚಟುವಟಿಕೆ ಬಾರ್‌ನಿಂದ Testing ವೀಕ್ಷಣೆಯನ್ನು ತೆರೆಯಿರಿ ಮತ್ತು Run Test" ಬಟನ್ ಅನ್ನು ಕ್ಲಿಕ್ ಮಾಡಿ, ಅಥವಾ ಹಾಟ್‌ಕಿ `Ctrl/Cmd + ; A` ಬಳಸಿ"
* Test Results ವೀಕ್ಷಣೆಯಲ್ಲಿ ಟೆಸ್ಟ್ ಫಲಿತಾಂಶದ ಔಟ್‌ಪುಟ್ ಅನ್ನು ನೋಡಿ.
* `src/test/extension.test.ts` ನಲ್ಲಿ ಬದಲಾವಣೆಗಳನ್ನು ಮಾಡಿ ಅಥವಾ `test` ಫೋಲ್ಡರ್ ಒಳಗೆ ಹೊಸ ಟೆಸ್ಟ್ ಫೈಲ್‌ಗಳನ್ನು ರಚಿಸಿ.
  * ನೀಡಲಾಗಿರುವ ಟೆಸ್ಟ್ ರನ್ನರ್ ಕೇವಲ ಹೆಸರಿನ ಪ್ಯಾಟರ್ನ್ `**.test.ts` ಗೆ ಹೊಂದಿಕೆಯಾಗುವ ಫೈಲ್‌ಗಳನ್ನು ಮಾತ್ರ ಪರಿಗಣಿಸುತ್ತದೆ.
  * ನೀವು ನಿಮ್ಮ ಟೆಸ್ಟ್‌ಗಳನ್ನು ಬೇಕಾದ ರೀತಿಯಲ್ಲಿ ಸಂರಚಿಸಲು `test` ಫೋಲ್ಡರ್ ಒಳಗೆ ಫೋಲ್ಡರ್‌ಗಳನ್ನು ರಚಿಸಬಹುದು.

## ಮುಂದೆ ಸಾಗಿರಿ

* [ನಿಮ್ಮ ವಿಸ್ತರಣೆಯನ್ನು ಬ್ಯಂಡಲ್ ಮಾಡುವ ಮೂಲಕ](https://code.visualstudio.com/api/working-with-extensions/bundling-extension) ವಿಸ್ತರಣೆಯ ಗಾತ್ರವನ್ನು ಕಡಿಮೆ ಮಾಡಿ ಮತ್ತು ಪ್ರಾರಂಭ ಸಮಯವನ್ನು ಸುಧಾರಿಸಿ.
* [ನಿಮ್ಮ ವಿಸ್ತರಣೆಯನ್ನು ಪ್ರಕಟಿಸಿ](https://code.visualstudio.com/api/working-with-extensions/publishing-extension) VS Code ವಿಸ್ತರಣೆ ಮಾರ್ಕೆಟ್ಪ್ಲೇಸ್‌ನಲ್ಲಿ.
* [ಸತತ ಏಕೀಕರಣ](https://code.visualstudio.com/api/working-with-extensions/continuous-integration) ಅನ್ನು ಹೊಂದಿಸುವ ಮೂಲಕ ಬಿಲ್ಡ್‌ಗಳನ್ನು ಸ್ವಯಂಚಾಲಿತಗೊಳಿಸಿ.
