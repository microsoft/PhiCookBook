# AGENTS.md

## Project Overview

PhiCookBook ಭಾರತದ Microsoft's Phi small Language Models (SLMs) ಜೊತೆ ಕೆಲಸ ಮಾಡಲು ಕೈಕೊಡುವ ಉದಾಹರಣೆಗಳು, ಟ್ಯುಟೋರಿಯಲ್ಸ್ ಮತ್ತು ಡಾಕ್ಯುಮೆಂಟೇಶನ್ನನ್ನು ಒಳಗೊಂಡ ಸಮಗ್ರ ಕುಕ್‌ಬುಕ್ ರಿಪೋಜಿಟರಿ. ಈ ರಿಪೋಜಿಟರಿ ಇನ್ಫರೆನ್ಸ್, ಫೈನ್-ಟ್ಯೂನಿಂಗ್, ಕ್ವಾಂಟೈಸೇಷನ್, RAG ಅನುಷ್ಠಾನಗಳು ಮತ್ತು ವಿವಿಧ ವೇದಿಕೆಗಳು ಮತ್ತು ಫ್ರೇಮ್‌ವರ್ಕ್‌ಗಳಲ್ಲಿ ಬಹುಮಾದರಿ ಅನ್ವಯಿಕೆಗಳನ್ನು ಸೇರಿದಂತೆ ವಿವಿಧ ಬಳಕೆಯ ಪ್ರಕರಣಗಳನ್ನು ತೋರಿಸುತ್ತದೆ.

**Key Technologies:**
- **ಭಾಷೆಗಳು:** Python, C#/.NET, JavaScript/Node.js
- **ಫ್ರೇಮ್‌ವರ್ಕ್‌ಗಳು:** ONNX Runtime, PyTorch, Transformers, MLX, OpenVINO, Semantic Kernel
- **ಪ್ಲಾಟ್‌ಫಾರ್ಮ್‌ಗಳು:** Microsoft Foundry, GitHub Models, Hugging Face, Ollama
- **ಮಾದರಿ ಪ್ರಕಾರಗಳು:** Phi-3, Phi-3.5, Phi-4 (text, vision, multimodal, reasoning variants)

**Repository Structure:**
- `/code/` - ಕೆಲಸ ಮಾಡುವ ಕೋಡ್ ಉದಾಹರಣೆಗಳು ಮತ್ತು ಸ್ಯಾಂಪಲ್ ಅನುಷ್ಠಾನಗಳು
- `/md/` - ವಿವರಣಾತ್ಮಕ ಡಾಕ್ಯುಮೆಂಟೇಶನ್, ಟ್ಯುಟೋರಿಯಲ್ಗಳು, ಮತ್ತು ಹೇಗೆ-ಮಾಡುವ ಮಾರ್ಗದರ್ಶಿಗಳು  
- `/translations/` - ಬಹುಭಾಷಾ ಅನುವಾದಗಳು (ಸ್ವಯಂಚಾಲಿತ ವರ್ಕ್‌ಫ್ಲೋ ಮೂಲಕ 50+ ಭಾಷೆಗಳು)
- `/.devcontainer/` - ಡೆವ್ ಕಂಟೇನರ್ ಕಾನ್ಫಿಗರೇಷನ್ (Python 3.12 உடன் Ollama)

## Development Environment Setup

### Using GitHub Codespaces or Dev Containers (Recommended)

1. GitHub Codespaces ನಲ್ಲಿ ತೆರೆಯಿರಿ (ಅತಿ ವೇಗದ):
   - README ನಲ್ಲಿ "Open in GitHub Codespaces" ಬ್ಯಾಡ್ಜ್ ಕ್ಲಿಕ್ ಮಾಡಿ
   - ಕಂಟೇನರ್ ಸ್ವಯಂಚಾಲಿತವಾಗಿ Python 3.12 ಮತ್ತು Phi-3 ಇರುವ Ollama ಜೊತೆಗೆ ಕಾನ್‌ಫಿಗರ್ ಆಗುತ್ತದೆ

2. VS Code Dev Containers ನಲ್ಲಿ ತೆರೆಯಿರಿ:
   - README ನಿಂದ "Open in Dev Containers" ಬ್ಯಾಡ್ಜ್ ಬಳಸಿ
   - ಕಂಟೇನರ್‌ಗೆ ಕನಿಷ್ಠ 16GB ಹೋಸ್ಟ್ ಮೆಮೊರಿ ಅಗತ್ಯವಿದೆ

### Local Setup

**Prerequisites:**
- Python 3.12 ಅಥವಾ ನಂತರದ ಆವೃತ್ತಿ
- .NET 8.0 SDK (C# ಉದಾಹರಣೆಗಳಿಗಾಗಿ)
- Node.js 18+ ಮತ್ತು npm (JavaScript ಉದಾಹರಣೆಗಳಿಗಾಗಿ)
- ಕನಿಷ್ಠ 16GB RAM ಶಿಫಾರಸ್ಸಾಗಿದೆ

**Installation:**
```bash
git clone https://github.com/microsoft/PhiCookBook.git
cd PhiCookBook
```

**Python ಉದಾಹರಣೆಗಳಿಗೆ:**
ನಿರ್ದಿಷ್ಟ ಉದಾಹರಣೆ ಡೈರೆಕ್ಟರಿಗಳಿಗೆ ನ್ಯಾವಿಗೇಟ್ ಮಾಡಿ ಮತ್ತು ಅವಶ್ಯಕತೆಗಳನ್ನು ಇನ್‌ಸ್ಟಾಲ್ ಮಾಡಿ:
```bash
cd code/<example-directory>
pip install -r requirements.txt  # requirements.txt ಇದ್ದರೆ
```

**.NET ಉದಾಹರಣೆಗಳಿಗೆ:**
```bash
cd md/04.HOL/dotnet/src
dotnet restore LabsPhi.sln
dotnet build LabsPhi.sln
```

**JavaScript/Web ಉದಾಹರಣೆಗಳಿಗೆ:**
```bash
cd code/08.RAG/rag_webgpu_chat
npm install
npm run dev  # ವಿಕಾಸ ಸರ್ವರ್ ಪ್ರಾರಂಭಿಸಿ
npm run build  # ಉತ್ಪಾದನೆಗಾಗಿ ನಿರ್ಮಿಸಿ
```

## Testing

ಈ ರಿಪೋಜಿಟರಿ ಪರಂಪರাগত ಯೂನಿಟ್ ಟೆಸ್ಟ್‌ಗಳಿರುವ ಸಾಫ್ಟ್‌ವೇರ್ ಪ್ರಾಜೆಕ್ಟ್‌ವಲ್ಲದೆ ಉದಾಹರಣೆ ಕೋಡ್ ಮತ್ತು ಟ್ಯುಟೋರಿಯಲ್ಗಳನ್ನು ಒಳಗೊಂಡಿದೆ. ಪ್ರಮಾಣೀಕರಣವು ಸಾಮಾನ್ಯವಾಗಿ ಕೆಳಗಿನಂತೆ ಮಾಡಲ್ಪಡುವುದು:

1. **ಉದಾಹರಣೆಗಳನ್ನು ನಡಿಸುವುದು** - ಪ್ರತಿ ಉದಾಹರಣೆಯು ದೋಷವಿಲ್ಲದೆ ನಿರ್ವಹಿಸಬೇಕು
2. **ಔಟ್‌ಪುಟ್‌ಗಳ ಪರಿಶೀಲನೆ** - ಮಾದರಿ ಪ್ರತಿಕ್ರಿಯೆಗಳು ಸೂಕ್ತವಾಗಿವೆ ಎಂದು ಪರಿಶೀಲಿಸಿ
3. **ಟ್ಯುಟೋರಿಯಲ್ ಅನುಸರಣೆ** - ಹಂತ ಕ್ರಮದ ಮಾರ್ಗದರ್ಶಿಗಳು ದಾಖಲಾಗಿದ್ದಂತೆ ಕಾರ್ಯನಿರ್ವಹಿಸಬೇಕು

**ಸಾಮಾನ್ಯ ಪ್ರಮಾಣೀಕರಣ ವಿಧಾನ:**
- ಗಮ್ಯವಿರುವ ವಾತಾವರಣದಲ್ಲಿ ಉದಾಹರಣೆ ಕಾರ್ಯನಿರ್ವಹಣೆಯನ್ನು ಪರೀಕ್ಷಿಸಿ
- ಅವಶ್ಯಕತೆಗೆ ಹೊಂದಿರುವ ಡಿಪೆಂಡೆನ್ಸಿಗಳು ಸರಿಯಾಗಿ ಇನ್‌ಸ್ಟೋಲ್ ಆಗಿದೆಯೇ ಎಂದು ಪರಿಶೀಲಿಸಿ
- ಮಾದರಿ ಡೌನ್‌ಲೋಡ್/ಲೋಡ್ ಯಶಸ್ವಿಯಾಗಿದೆಯೇ ಎಂದು ಪರೀಕ್ಷಿಸಿ
- ನಿರೀಕ್ಷಿತ ವರ್ತನೆ ಡಾಕ್ಯುಮೆಂಟೇಶನಿಗೆ ಹೊಂದಿಕೊಳ್ಳುತ್ತಿದೆಯೇ ಎಂಬುದನ್ನು ಖಚಿತಪಡಿಸಿ

## Code Style and Conventions

### General Guidelines

- ಉದಾಹರಣೆಗಳು ಸ್ಪಷ್ಟ, ಚೆನ್ನಾಗಿ ಕಾಮೆಂಟ್ ಮಾಡಲಾದ ಮತ್ತು ಶೈಕ್ಷಣಿಕವಾಗಿರಬೇಕು
- ಭಾಷಾ-ನಿರ್ದಿಷ್ಟ ನಿಯಮಾವಳಿಗಳನ್ನು ಅನುಸರಿಸಿ (Python ಗೆ PEP 8, .NET ಗೆ C# ಸ್ಟ್ಯಾಂಡರ್ಡ್ಸ್)
- ಉದಾಹರಣೆಗಳನ್ನು ನಿರ್ದಿಷ್ಟ Phi ಮಾದರಿ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಪ್ರದರ್ಶಿಸುವದರಲ್ಲಿ ಕೇಂದ್ರೀಕರಿಸಿ
- ಮುಖ್ಯ ধারণೆಗಳು ಮತ್ತು ಮಾದರಿ-ನಿರ್ದಿಷ್ಟ ಪ್ಯಾರಾಮೀಟರ್‌ಗಳನ್ನು ವಿವರಣೆಗೊಳ್ಳುವ ಕಾಮೆಂಟ್‌ಗಳನ್ನು ಸೇರಿಸಿ

### Documentation Standards

**URL Formatting:**
- ಹೆಚ್ಚುವರಿ ಖಾಲಿ ಸ್ಥಳಗಳಿಲ್ಲದೆ `[text](../../url)` ಫಾರ್ಮ್ಯಾಟ್ ಬಳಸಿ
- ಸಂಬಂಧಿತ ಲಿಂಕ್‌ಗಳು: ಪ್ರಸ್ತುತ ಡೈರೆಕ್ಟರಿಗಾಗಿ `./`, ಪೇರಂಟ್‌ಗೆ `../` ಬಳಸಿ
- URLಗಳಲ್ಲಿ ದೇಶ-ನಿರ್ದಿಷ್ಟ ಲೋಕೇಲ್‌ಗಳನ್ನು ಬಳಸದಿರಿ (ಉದಾಹರಣೆಗೆ `/en-us/`, `/en/`)

**Images:**
- ಎಲ್ಲಾ ಚಿತ್ರಗಳನ್ನು `/imgs/` ಡೈರೆಕ್ಟರಿಯಲ್ಲಿ ಸಂಗ್ರಹಿಸಿ
- ಇಂಗ್ಲಿಷ್ ಅಕ್ಷರಗಳು, ಸಂಖ್ಯೆಗಳು ಮತ್ತು ಡ್ಯಾಶ್‌ಗಳನ್ನು ಬಳಸಿ ವಿವರಣಾತ್ಮಕ ಹೆಸರಿಗಳನ್ನು ಉಪಯೋಗಿಸಿ
- ಉದಾಹರಣೆ: `phi-3-architecture.png`

**Markdown Files:**
- `/code/` ಡೈರೆಕ್ಟರಿಯಲ್ಲಿ ಕಾರ್ಯನಿರ್ವಹಿಸುವ ನಿಜವಾದ ಉದಾಹರಣೆಗಳನ್ನು ಉಲ್ಲೇಖಿಸಿ
- ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಅನ್ನು ಕೋಡ್ ಬದಲಾವಣೆಗಳೊಂದಿಗೆ ಸಿಂಕ್ರೊನೈಸು ಮಾಡಿ
- README ನಲ್ಲಿ Jupyter ನೋಟ್ಬುಕ್ ಲಿಂಕ್‌ಗಳನ್ನು ಗುರುತಿಸಲು 📓 ಎಂಬ ಎಮೊಜಿಯನ್ನು ಬಳಸಿ

### File Organization

- `/code/` ನಲ್ಲಿ ವಿಷಯ/ವೈಶಿಷ್ಟ್ಯ ಆಧಾರದ ಮೇಲೆ ಕೋಡ್ ಉದಾಹರಣೆಗಳು ವ್ಯವಸ್ಥಿತಗೊಳಿಸಲಾಗಿದೆ
- `/md/` ನಲ್ಲಿ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಸಾಧ್ಯವಿದ್ದಲ್ಲಿ ಕೋಡ್ ರಚನೆಯೊಂದಿಗೆ ಸಮಾನವಾಗಿದೆ
- ಸಂಬಂಧಿತ ಫೈಲ್‌ಗಳು (ನೋಟ್ಬುಕ್‌ಗಳು, ಸ್ಕ್ರಿಪ್ಟ್‌ಗಳು, ಕಾನ್‌ಫಿಗ್‌ಗಳು) ಉಪಡೈರೆಕ್ಟರಿಗಳಲ್ಲಿ ಒಟ್ಟಿಗೆ ಇರಲಿ

## Pull Request Guidelines

### Before Submitting

1. **ರಿಪೋಜಿಟರಿಯನ್ನು ಫೋರ್ಕ್ ಮಾಡಿ** ನಿಮ್ಮ ಖಾತೆಗೆ
2. **ಪ್ರಕಾರದಂತೆ PRಗಳನ್ನು ಪ್ರತ್ಯೇಕಿಸಿ:**
   - ಒಂದು PR ನಲ್ಲಿ ಬಗ್ ಫಿಕ್ಸ್‌ಗಳು
   - ಮತ್ತೊಂದರಲ್ಲಿ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಅಪ್ಡೇಟ್‌ಗಳು
   - ಹೊಸ ಉದಾಹರಣೆಗಳಿಗೆ ಪ್ರತ್ಯೇಕ PRಗಳು
   - ಟೈಪೋ ನವೀಕರಣಗಳನ್ನು ಒಟ್ಟಿಗೆ ಸೇರಿಸಬಹುದು

3. **ಮರ್ಜ್ ಘರ್ಷಣೆಗಳನ್ನು ನಿರ್ವಹಿಸಿ:**
   - ಬದಲಾವಣೆಗಳನ್ನು ಮಾಡುವುದಕ್ಕೆ ಮುಂಚೆಯೇ ನಿಮ್ಮ ಸ್ಥಳೀಯ `main` ಶಾಖೆಯನ್ನು ಅಪ್‌ಡೇಟ್ ಮಾಡಿ
   - ಅಪ್‌ಸ್ಟ್ರೀಮ್ ಅನ್ನು συχνά ಸಿಂಕ್ ಮಾಡಿ

4. **ಅನುವಾದ PRಗಳು:**
   - ಫೋಲ್ಡರ್‌ನಲ್ಲಿನ ಎಲ್ಲಾ ಫೈಲ್‌ಗಳಿಗೆ ಅನುವಾದಗಳನ್ನು ಒಳಗೊಂಡಿರಬೇಕು
   - ಮೂಲ ಭಾಷೆಯೊಂದಿಗೆ ಸತತ ರಚನೆಯನ್ನು ಕಾಪಾಡಿಕೊಳ್ಳಿ

### Required Checks

PRಗಳು ಸ್ವಯಂಚಾಲಿತವಾಗಿ GitHub ವರ್ಕ್‌ಫ್ಲೋಗಳನ್ನು ಚಾಲನೆ ಮಾಡಿ ಮಾನ್ಯತೆ ನಡೆಸುತ್ತವೆ:

1. **ಸಂಬಂಧಿತ ಪಥ ಮಾಪನವೇಧಕತೆ** - ಎಲ್ಲಾ ಆಂತರಿಕ ಲಿಂಕ್‌ಗಳು ಕೆಲಸಮಾಡಬೇಕು
   - ಲಿಂಕ್‌ಗಳನ್ನು ಸ್ಥಳೀಯವಾಗಿ ಪರೀಕ್ಷಿಸಿ: VS Code ನಲ್ಲಿ Ctrl+Click ಬಳಸಿ
   - VS Code ನಿಂದ ಪಥ ಸಲಹೆಗಳನ್ನು ಬಳಸಿ (`./` ಅಥವಾ `../`)

2. **URL ಲೋಕೇಲ್ ಪರಿಶೀಲನೆ** - ವೆಬ್ URLಗಳಲ್ಲಿ ದೇಶ-ನಿರ್ದಿಷ್ಟ ಲೋಕೇಲ್‌ಗಳು ಇರಬಾರದು
   - `/en-us/`, `/en/` ಅಥವಾ ಇತರ ಭಾಷಾ ಕೋಡ್‌ಗಳನ್ನು ತೆಗೆದುಹಾಕಿ

3. **ತૂટು URL ಪರಿಶೀಲನೆ** - ಎಲ್ಲಾ URLಗಳು 200 ಸ್ಥಿತಿಯನ್ನು ಮಡಕಿಸಬೇಕು
   - ಸಲ್ಲಿಸುವ ಮೊದಲು ಲಿಂಕ್‌ಗಳ ಪ್ರವೇಶನೀಯತೆಯನ್ನು ಪರಿಶೀಲಿಸಿ
   - ಗಮನಿಸಿ: ಕೆಲವು ವಿಫಲತೆಗಳು ನೆಟ್‌ವರ್ಕ್ ನಿರ್ಬಂಧಗಳಿಂದ ಆಗಿರಬಹುದು

## Troubleshooting

### Common Issues

**Memory Issues:**
- Phi ಮಾದರಿಗಳಿಗೆ ಸಾಕಷ್ಟು RAM ಒದಗಬೇಕು (ಖಾಸಗಿ ದೃಷ್ಟಿ/ಬಹುಮಾದರಿ ಬೇರಿಯಂಟ್‌ಗಳಿಗೆ ಹೆಚ್ಚು)
- ಸಂಪನ್ಮೂಲ-ಸೀಮಿತ ವಾತಾವರಣಗಳಿಗಾಗಿ ಕ್ವಾಂಟೈಜ್ಡ್ ಮಾದರಿಗಳನ್ನು ಬಳಸಿ
- ನೋಡಿ `/md/01.Introduction/04/QuantifyingPhi.md`

**Dependency Conflicts:**
- Python ಉದಾಹರಣೆಗಳು ನಿರ್ದಿಷ್ಟ ಆವೃತ್ತಿ ಅವಶ್ಯಕತೆಗಳನ್ನು ಹೊಂದಿರಬಹುದು
- ಪ್ರತಿ ಉದಾಹರಣೆಗೆ ವರ್ಚುಯಲ್ ಎನ್ವಿರಾನ್‌ಮೆಂಟ್ ಬಳಸಿ
- ವೈಯಕ್ತಿಕ `requirements.txt` ಫೈಲ್‌ಗಳನ್ನು ಪರಿಶೀಲಿಸಿ

**Model Download Failures:**
- ದೊಡ್ಡ ಮಾದರಿಗಳು ನಿಧಾನ ಸಂಪರ್ಕದಲ್ಲಿ ಸಮಯ ಮೀರಿಸಬಹುದು
- Codespaces, Azure ಮುಂತಾದ ಕ್ಲೌಡ್ ವಾತಾವರಣಗಳನ್ನು ಬಳಸುವದನ್ನು ಪರಿಗಣಿಸಿ
- Hugging Face ಕ್ಯಾಶೆ ಪರಿಶೀಲಿಸಿ: `~/.cache/huggingface/`

**.NET Project Issues:**
- .NET 8.0 SDK ಸ್ಥಾಪಿತವಾಗಿರುವುದನ್ನು ಖಚಿತಪಡಿಸಿ
- ನಿರ್ಮಾಣಕ್ಕೂ ಮುಂಚೆ `dotnet restore` ಬಳಸಿ
- ಕೆಲವು ಪ್ರಾಜೆಕ್ಟ್‌ಗಳಿಗೆ CUDA-ನಿರ್ದಿಷ್ಟ ಕಾನ್ಫಿಗರ್‌ಗಳು ಇರುತ್ತವೆ (Debug_Cuda)

**JavaScript/Web Examples:**
- ಹೊಂದಾಣಿಕೆಗೆ Node.js 18+ ಬಳಸಿ
- ಸಮಸ್ಯೆಗಳು ಮುಂದುವರಿದರೆ `node_modules` ಅನ್ನು ತೆರವುಗೊಳಿಸಿ ಮತ್ತು ಮರುಅನುಸ್ಥಾಪಿಸಿ
- WebGPU ಹೊಂದಾಣಿಕೆಗೆ ಬ್ರೌಸರ್ ಕন್ಸೋಲ್ ಪರಿಶೀಲಿಸಿ
