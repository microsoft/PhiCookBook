# Συμβολή

Αυτό το έργο καλωσορίζει συνεισφορές και προτάσεις. Οι περισσότερες συνεισφορές απαιτούν να συμφωνήσετε με μια
Συμφωνία Άδειας Συμβολής (CLA) που δηλώνει ότι έχετε το δικαίωμα και πράγματι παραχωρείτε σε εμάς
τα δικαιώματα να χρησιμοποιήσουμε τη συνεισφορά σας. Για λεπτομέρειες, επισκεφθείτε [https://cla.opensource.microsoft.com](https://cla.opensource.microsoft.com)

Όταν υποβάλετε ένα pull request, ένα bot CLA θα καθορίσει αυτόματα αν χρειάζεται να παρέχετε
μια CLA και θα διακοσμήσει το PR ανάλογα (π.χ., έλεγχος κατάστασης, σχόλιο). Απλά ακολουθήστε τις οδηγίες
που παρέχει το bot. Θα χρειαστεί να το κάνετε μόνο μία φορά σε όλα τα αποθετήρια που χρησιμοποιούν τη CLA μας.

## Κώδικας Συμπεριφοράς

Αυτό το έργο έχει υιοθετήσει τον [Κώδικα Συμπεριφοράς Ανοιχτού Κώδικα της Microsoft](https://opensource.microsoft.com/codeofconduct/).
Για περισσότερες πληροφορίες διαβάστε τις [Συχνές Ερωτήσεις για τον Κώδικα Συμπεριφοράς](https://opensource.microsoft.com/codeofconduct/faq/) ή επικοινωνήστε με [opencode@microsoft.com](mailto:opencode@microsoft.com) για επιπλέον ερωτήσεις ή σχόλια.

## Προφυλάξεις για τη δημιουργία θεμάτων

Παρακαλούμε μην ανοίγετε θέματα στο GitHub για γενικές ερωτήσεις υποστήριξης, καθώς η λίστα του GitHub πρέπει να χρησιμοποιείται για αιτήματα χαρακτηριστικών και αναφορές σφαλμάτων. Με αυτόν τον τρόπο μπορούμε να παρακολουθούμε πιο εύκολα τα πραγματικά προβλήματα ή σφάλματα στον κώδικα και να διατηρούμε τη γενική συζήτηση ξεχωριστά από τον ίδιο τον κώδικα.

## Πώς να συμβάλλετε

### Οδηγίες για Pull Requests

Όταν υποβάλλετε ένα pull request (PR) στο αποθετήριο Phi-3 CookBook, παρακαλούμε να ακολουθείτε τις παρακάτω οδηγίες:

- **Κλωνοποίηση του Αποθετηρίου**: Πάντα να κλωνοποιείτε το αποθετήριο στον δικό σας λογαριασμό πριν κάνετε τις τροποποιήσεις σας.

- **Ξεχωριστά pull requests (PR)**:
  - Υποβάλετε κάθε τύπο αλλαγής σε ξεχωριστό pull request. Για παράδειγμα, διορθώσεις σφαλμάτων και ενημερώσεις τεκμηρίωσης πρέπει να υποβάλλονται σε ξεχωριστά PR.
  - Διορθώσεις τυπογραφικών λαθών και μικρές ενημερώσεις τεκμηρίωσης μπορούν να συνδυαστούν σε ένα PR όπου είναι κατάλληλο.

- **Διαχείριση συγκρούσεων συγχώνευσης**: Αν το pull request σας εμφανίζει συγκρούσεις συγχώνευσης, ενημερώστε το τοπικό σας `main` branch ώστε να αντικατοπτρίζει το κύριο αποθετήριο πριν κάνετε τις τροποποιήσεις σας.

- **Υποβολές μεταφράσεων**: Όταν υποβάλλετε PR μετάφρασης, βεβαιωθείτε ότι ο φάκελος μετάφρασης περιλαμβάνει μεταφράσεις για όλα τα αρχεία του αρχικού φακέλου.

### Οδηγίες Μετάφρασης

> [!IMPORTANT]
>
> Κατά τη μετάφραση κειμένου σε αυτό το αποθετήριο, μην χρησιμοποιείτε μηχανική μετάφραση. Αναλάβετε μετάφραση μόνο σε γλώσσες που γνωρίζετε καλά.

Αν γνωρίζετε καλά μια μη αγγλική γλώσσα, μπορείτε να βοηθήσετε στη μετάφραση του περιεχομένου. Ακολουθήστε αυτά τα βήματα για να διασφαλίσετε ότι οι συνεισφορές σας στη μετάφραση θα ενσωματωθούν σωστά, χρησιμοποιώντας τις παρακάτω οδηγίες:

- **Δημιουργία φακέλου μετάφρασης**: Μεταβείτε στον αντίστοιχο φάκελο ενότητας και δημιουργήστε έναν φάκελο μετάφρασης για τη γλώσσα στην οποία συνεισφέρετε. Για παράδειγμα:
  - Για την ενότητα εισαγωγής: `PhiCookBook/md/01.Introduce/translations/<language_code>/`
  - Για την ενότητα γρήγορης εκκίνησης: `PhiCookBook/md/02.QuickStart/translations/<language_code>/`
  - Συνεχίστε αυτό το μοτίβο και για άλλες ενότητες (03.Inference, 04.Finetuning, κ.λπ.)

- **Ενημέρωση σχετικών διαδρομών**: Κατά τη μετάφραση, προσαρμόστε τη δομή των φακέλων προσθέτοντας `../../` στην αρχή των σχετικών διαδρομών μέσα στα αρχεία markdown για να διασφαλίσετε ότι οι σύνδεσμοι λειτουργούν σωστά. Για παράδειγμα, αλλάξτε:
  - `(../../imgs/01/phi3aisafety.png)` σε `(../../../../imgs/01/phi3aisafety.png)`

- **Οργάνωση των μεταφράσεών σας**: Κάθε μεταφρασμένο αρχείο πρέπει να τοποθετείται στον αντίστοιχο φάκελο μετάφρασης της ενότητας. Για παράδειγμα, αν μεταφράζετε την ενότητα εισαγωγής στα ισπανικά, θα δημιουργήσετε:
  - `PhiCookBook/md/01.Introduce/translations/es/`

- **Υποβολή πλήρους PR**: Βεβαιωθείτε ότι όλα τα μεταφρασμένα αρχεία μιας ενότητας περιλαμβάνονται σε ένα PR. Δεν δεχόμαστε μερικές μεταφράσεις για μια ενότητα. Όταν υποβάλλετε PR μετάφρασης, βεβαιωθείτε ότι ο φάκελος μετάφρασης περιλαμβάνει μεταφράσεις για όλα τα αρχεία του αρχικού φακέλου.

### Οδηγίες Συγγραφής

Για να διασφαλιστεί η συνέπεια σε όλα τα έγγραφα, παρακαλούμε χρησιμοποιήστε τις παρακάτω οδηγίες:

- **Μορφοποίηση URL**: Περιβάλλετε όλα τα URLs με αγκύλες ακολουθούμενες από παρενθέσεις, χωρίς επιπλέον κενά γύρω ή μέσα σε αυτά. Για παράδειγμα: `[example](https://www.microsoft.com)`.

- **Σχετικοί σύνδεσμοι**: Χρησιμοποιήστε `./` για σχετικούς συνδέσμους που δείχνουν σε αρχεία ή φακέλους στον τρέχοντα κατάλογο, και `../` για αυτούς που δείχνουν σε γονικό κατάλογο. Για παράδειγμα: `[example](../../path/to/file)` ή `[example](../../../path/to/file)`.

- **Μη συγκεκριμένα σε χώρα τοπικά**: Βεβαιωθείτε ότι οι σύνδεσμοι σας δεν περιλαμβάνουν τοπικά που αφορούν συγκεκριμένες χώρες. Για παράδειγμα, αποφύγετε `/en-us/` ή `/en/`.

- **Αποθήκευση εικόνων**: Αποθηκεύστε όλες τις εικόνες στο φάκελο `./imgs`.

- **Περιγραφικά ονόματα εικόνων**: Ονομάστε τις εικόνες περιγραφικά, χρησιμοποιώντας αγγλικούς χαρακτήρες, αριθμούς και παύλες. Για παράδειγμα: `example-image.jpg`.

## GitHub Workflows

Όταν υποβάλλετε ένα pull request, τα παρακάτω workflows θα ενεργοποιηθούν για να επαληθεύσουν τις αλλαγές. Ακολουθήστε τις παρακάτω οδηγίες για να διασφαλίσετε ότι το pull request σας θα περάσει τους ελέγχους workflow:

- [Check Broken Relative Paths](../..)
- [Check URLs Don't Have Locale](../..)

### Check Broken Relative Paths

Αυτό το workflow διασφαλίζει ότι όλες οι σχετικές διαδρομές στα αρχεία σας είναι σωστές.

1. Για να βεβαιωθείτε ότι οι σύνδεσμοι σας λειτουργούν σωστά, εκτελέστε τις παρακάτω εργασίες χρησιμοποιώντας το VS Code:
    - Τοποθετήστε το ποντίκι πάνω από οποιονδήποτε σύνδεσμο στα αρχεία σας.
    - Πατήστε **Ctrl + Κλικ** για να μεταβείτε στον σύνδεσμο.
    - Αν κάνετε κλικ σε έναν σύνδεσμο και δεν λειτουργεί τοπικά, θα ενεργοποιήσει το workflow και δεν θα λειτουργεί στο GitHub.

1. Για να διορθώσετε αυτό το πρόβλημα, εκτελέστε τις παρακάτω εργασίες χρησιμοποιώντας τις προτάσεις διαδρομών που παρέχει το VS Code:
    - Πληκτρολογήστε `./` ή `../`.
    - Το VS Code θα σας προτείνει επιλογές με βάση αυτό που πληκτρολογήσατε.
    - Ακολουθήστε τη διαδρομή κάνοντας κλικ στο επιθυμητό αρχείο ή φάκελο για να βεβαιωθείτε ότι η διαδρομή σας είναι σωστή.

Μόλις προσθέσετε τη σωστή σχετική διαδρομή, αποθηκεύστε και στείλτε τις αλλαγές σας.

### Check URLs Don't Have Locale

Αυτό το workflow διασφαλίζει ότι κανένα URL δεν περιλαμβάνει τοπικό που σχετίζεται με συγκεκριμένη χώρα. Καθώς αυτό το αποθετήριο είναι προσβάσιμο παγκοσμίως, είναι σημαντικό να διασφαλίσουμε ότι τα URLs δεν περιέχουν τοπικά της χώρας σας.

1. Για να επαληθεύσετε ότι τα URLs σας δεν έχουν τοπικά χωρών, εκτελέστε τις παρακάτω εργασίες:

    - Ελέγξτε για κείμενο όπως `/en-us/`, `/en/` ή οποιοδήποτε άλλο τοπικό γλώσσας στα URLs.
    - Αν αυτά δεν υπάρχουν στα URLs σας, τότε περνάτε τον έλεγχο.

1. Για να διορθώσετε αυτό το πρόβλημα, εκτελέστε τις παρακάτω εργασίες:
    - Ανοίξτε το αρχείο που επισημάνθηκε από το workflow.
    - Αφαιρέστε το τοπικό χώρας από τα URLs.

Μόλις αφαιρέσετε το τοπικό χώρας, αποθηκεύστε και στείλτε τις αλλαγές σας.

### Check Broken Urls

Αυτό το workflow διασφαλίζει ότι οποιοδήποτε web URL στα αρχεία σας λειτουργεί και επιστρέφει κωδικό κατάστασης 200.

1. Για να επαληθεύσετε ότι τα URLs σας λειτουργούν σωστά, εκτελέστε τις παρακάτω εργασίες:
    - Ελέγξτε την κατάσταση των URLs στα αρχεία σας.

2. Για να διορθώσετε τυχόν σπασμένα URLs, εκτελέστε τις παρακάτω εργασίες:
    - Ανοίξτε το αρχείο που περιέχει το σπασμένο URL.
    - Ενημερώστε το URL στο σωστό.

Μόλις διορθώσετε τα URLs, αποθηκεύστε και στείλτε τις αλλαγές σας.

> [!NOTE]
>
> Μπορεί να υπάρχουν περιπτώσεις όπου ο έλεγχος URL αποτυγχάνει παρόλο που ο σύνδεσμος είναι προσβάσιμος. Αυτό μπορεί να συμβεί για διάφορους λόγους, όπως:
>
> - **Περιορισμοί δικτύου:** Οι servers των GitHub actions μπορεί να έχουν περιορισμούς δικτύου που εμποδίζουν την πρόσβαση σε ορισμένα URLs.
> - **Προβλήματα χρονικού ορίου:** URLs που χρειάζονται πολύ χρόνο για να απαντήσουν μπορεί να προκαλέσουν σφάλμα χρονικού ορίου στο workflow.
> - **Προσωρινά προβλήματα server:** Περιοδικές διακοπές λειτουργίας ή συντήρηση server μπορεί να κάνουν ένα URL προσωρινά μη διαθέσιμο κατά την επαλήθευση.
