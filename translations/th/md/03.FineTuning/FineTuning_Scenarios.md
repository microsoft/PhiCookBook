## กรณีการปรับจูนละเอียด (Fine Tuning Scenarios)

![FineTuning with MS Services](../../../../translated_images/th/FinetuningwithMS.3d0cec8ae693e094.webp)

ส่วนนี้นำเสนอภาพรวมของกรณีการปรับจูนละเอียดในสภาพแวดล้อม Microsoft Foundry และ Azure ซึ่งรวมถึงรูปแบบการปรับใช้ ชั้นโครงสร้างพื้นฐาน และเทคนิคการปรับแต่งที่ใช้บ่อย

**แพลตฟอร์ม**  
รวมถึงบริการที่มีการจัดการ เช่น Microsoft Foundry (เดิมชื่อ Azure AI Foundry) และ Azure Machine Learning ซึ่งให้บริการการจัดการโมเดล การประสานงาน การติดตามการทดลอง และเวิร์กโฟลว์การปรับใช้

**โครงสร้างพื้นฐาน**  
การปรับจูนละเอียดต้องการทรัพยากรการประมวลผลที่สามารถปรับขนาดได้ ในสภาพแวดล้อม Azure โดยทั่วไปจะรวมถึงเครื่องเสมือนที่ใช้ GPU และทรัพยากร CPU สำหรับงานเบา พร้อมกับที่เก็บข้อมูลที่ปรับขนาดได้สำหรับชุดข้อมูลและจุดตรวจสอบ

**เครื่องมือและเฟรมเวิร์ก**  
เวิร์กโฟลว์การปรับจูนละเอียดมักจะพึ่งพาเฟรมเวิร์กและไลบรารีการปรับแต่ง เช่น Hugging Face Transformers, DeepSpeed และ PEFT (Parameter-Efficient Fine-Tuning)

กระบวนการปรับจูนละเอียดด้วยเทคโนโลยีของ Microsoft ครอบคลุมบริการแพลตฟอร์ม โครงสร้างพื้นฐานการประมวลผล และเฟรมเวิร์กการฝึกอบรม โดยการเข้าใจว่าส่วนประกอบเหล่านี้ทำงานร่วมกันอย่างไร นักพัฒนาจะสามารถปรับโมเดลฐานให้เหมาะสมกับงานเฉพาะและกรณีใช้งานในสภาพแวดล้อมการผลิตได้อย่างมีประสิทธิภาพ

## โมเดลในฐานะบริการ (Model as Service)

ปรับจูนโมเดลโดยใช้บริการปรับจูนที่โฮสต์โดยไม่ต้องสร้างหรือจัดการทรัพยากรการประมวลผลเอง

![MaaS Fine Tuning](../../../../translated_images/th/MaaSfinetune.3eee4630607aff0d.webp)

ตอนนี้มีบริการปรับจูนแบบไม่มีเซิร์ฟเวอร์สำหรับตระกูลโมเดล Phi-3, Phi-3.5 และ Phi-4 ช่วยให้นักพัฒนาปรับแต่งโมเดลสำหรับกรณีใช้งานบนคลาวด์และ Edge ได้อย่างรวดเร็วและง่ายดายโดยไม่ต้องจัดเตรียมทรัพยากรการประมวลผล

## โมเดลในฐานะแพลตฟอร์ม (Model as a Platform)

ผู้ใช้จัดการทรัพยากรการประมวลผลของตนเองเพื่อปรับจูนโมเดล
