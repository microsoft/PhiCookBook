## Reporting Sekyuriti Issues

**Abeg no dey report security vulnerabilities through public GitHub issues.**

Instead, abeg report dem to the Microsoft Security Response Center (MSRC) at [https://msrc.microsoft.com/create-report](https://aka.ms/security.md/msrc/create-report).

If you prefer to submit without logging in, send email to [secure@microsoft.com](mailto:secure@microsoft.com).  If e possible, encrypt your message with our PGP key; abeg download am from the [Microsoft Security Response Center PGP Key page](https://aka.ms/security.md/msrc/pgp).

You suppose receive response within 24 hours. If for some reason you no receive, abeg follow up via email make sure we get your original message. Additional information dey for [microsoft.com/msrc](https://www.microsoft.com/msrc). 

Abeg include di requested information wey dey listed below (as much as you fit provide) to help us understand beta di nature and scope of di possible issue:

  * Wetin be di kind of issue (e.g. buffer overflow, SQL injection, cross-site scripting, etc.)
  * Full paths of source file(s) related to di way di issue show
  * Where di affected source code dey (tag/branch/commit or direct URL)
  * Any special configuration wey needed to reproduce di issue
  * Step-by-step instructions wey go reproduce di issue
  * Proof-of-concept or exploit code (if e possible)
  * Impact of di issue, including how attacker fit exploit di issue

Dis information go help us triage your report faster.

If you dey report for bug bounty, more complete reports fit result for higher bounty award. Abeg visit our [Microsoft Bug Bounty Program](https://aka.ms/security.md/msrc/bounty) page for more details about our active programs.
