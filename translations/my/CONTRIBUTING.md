# Contributing

ဒီပရောဂျက်မှာ အထောက်အပံ့နဲ့ အကြံပြုချက်တွေကို ကြိုဆိုပါတယ်။ အများဆုံး အထောက်အပံ့တွေမှာ Contributor License Agreement (CLA) ကို သဘောတူဖို့ လိုအပ်ပြီး သင့်ရဲ့ အထောက်အပံ့ကို အသုံးပြုခွင့်ရှိပြီး သုံးစွဲခွင့်ကို လက်မှတ်ထိုးပေးတာ ဖြစ်ကြောင်း သက်သေပြရပါမယ်။ အသေးစိတ်အချက်အလက်များအတွက် [https://cla.opensource.microsoft.com](https://cla.opensource.microsoft.com) ကို သွားကြည့်ပါ။

သင် pull request တင်တဲ့အခါ CLA bot က သင် CLA တင်ဖို့ လိုအပ်မလား၊ PR ကို ဘယ်လို အဆင်ပြေစွာ ပြင်ဆင်ပေးရမလဲ (ဥပမာ - status check, comment) စတာတွေကို အလိုအလျောက် သတ်မှတ်ပေးပါလိမ့်မယ်။ bot ရဲ့ ညွှန်ကြားချက်တွေကို လိုက်နာပေးရုံပါပဲ။ CLA ကို သုံးတဲ့ repo အားလုံးမှာ တစ်ကြိမ်တည်းသာ လုပ်ဆောင်ရပါမယ်။

## Code of Conduct

ဒီပရောဂျက်မှာ [Microsoft Open Source Code of Conduct](https://opensource.microsoft.com/codeofconduct/) ကို လက်ခံအသုံးပြုထားပါတယ်။ ပိုမိုသိရှိလိုပါက [Code of Conduct FAQ](https://opensource.microsoft.com/codeofconduct/faq/) ကို ဖတ်ရှုပါ၊ ဒါမှမဟုတ် မေးခွန်းများ သို့မဟုတ် မှတ်ချက်များရှိပါက [opencode@microsoft.com](mailto:opencode@microsoft.com) သို့ ဆက်သွယ်နိုင်ပါသည်။

## Cautions for creating issues

GitHub issues ကို အထွေထွေ အထောက်အပံ့မေးခွန်းများအတွက် မဖွင့်သင့်ပါ၊ GitHub issues ကို feature requests နဲ့ bug reports အတွက်သာ အသုံးပြုသင့်ပါတယ်။ ဒီလိုလုပ်ခြင်းဖြင့် အမှန်တကယ် ဖြစ်ပေါ်နေတဲ့ ပြဿနာများကို ပိုမိုလွယ်ကူစွာ စောင့်ကြည့်နိုင်ပြီး အထွေထွေ ဆွေးနွေးမှုများကို ကုဒ်နဲ့ သီးခြားထားနိုင်ပါသည်။

## How to Contribute

### Pull Requests Guidelines

Phi-3 CookBook repository သို့ pull request (PR) တင်တဲ့အခါ အောက်ပါ လမ်းညွှန်ချက်များကို အသုံးပြုပါ။

- **Repository ကို Fork လုပ်ပါ**: ပြင်ဆင်မှုများ မလုပ်မီ သင့်အကောင့်သို့ repository ကို အမြဲ Fork လုပ်ပါ။

- **Pull request (PR) များကို သီးခြားတင်ပါ**:
  - ပြောင်းလဲမှု အမျိုးအစား တစ်ခုစီကို သီးခြား PR တင်ပါ။ ဥပမာ - bug fixes နဲ့ documentation update များကို သီးခြား PR များအဖြစ် တင်ပါ။
  - စာလုံးပေါင်းမှားများနဲ့ သေးငယ်တဲ့ documentation update များကို တစ်ခုတည်း PR ထဲတွင် ပေါင်းစပ်တင်နိုင်ပါသည်။

- **Merge conflicts ကို ကိုင်တွယ်ပါ**: သင့် PR မှာ merge conflicts တွေရှိပါက ပြင်ဆင်မှု မလုပ်မီ သင့်ရဲ့ local `main` branch ကို main repository နဲ့ ကိုက်ညီအောင် update လုပ်ပါ။

- **ဘာသာပြန် PR များ**: ဘာသာပြန် PR တင်တဲ့အခါ ဘာသာပြန်ဖိုင် folder ထဲမှာ မူရင်း folder ထဲရှိ ဖိုင်အားလုံးအတွက် ဘာသာပြန်ထားမှု ရှိဖို့ သေချာစေပါ။

### Writing Guidelines

စာရွက်စာတမ်းအားလုံးမှာ တူညီမှုရှိစေရန် အောက်ပါ လမ်းညွှန်ချက်များကို အသုံးပြုပါ။

- **URL ဖော်မတ်**: URL များအားလုံးကို စတုရန်းကွက်တွေနဲ့ ဖုံးအုပ်ပြီး အနားမှာ အပိုအကွာအဝေး မရှိစေရန် သေချာပါ။ ဥပမာ - `[example](https://www.microsoft.com)`။

- **Relative links**: လက်ရှိ directory ထဲရှိ ဖိုင် သို့မဟုတ် folder များကို ရည်ညွှန်းတဲ့ relative link များအတွက် `./` ကို အသုံးပြုပါ၊ မိခင် directory ထဲရှိ ဖိုင် သို့မဟုတ် folder များအတွက် `../` ကို အသုံးပြုပါ။ ဥပမာ - `[example](../../path/to/file)` သို့မဟုတ် `[example](../../../path/to/file)`။

- **နိုင်ငံအလိုက် locale မပါရှိစေရန်**: သင့် link များတွင် နိုင်ငံအလိုက် locale မပါရှိစေရန် သေချာပါ။ ဥပမာ - `/en-us/` သို့မဟုတ် `/en/` မပါစေရန်။

- **ပုံများ သိမ်းဆည်းရာ**: ပုံအားလုံးကို `./imgs` folder ထဲ သိမ်းဆည်းပါ။

- **ပုံအမည်များ ဖော်ပြချက်ပြည့်စုံစေရန်**: ပုံအမည်များကို အင်္ဂလိပ်အက္ခရာ၊ နံပါတ်များနဲ့ dash (-) များကို အသုံးပြုပြီး ဖော်ပြချက်ပြည့်စုံစွာ အမည်ပေးပါ။ ဥပမာ - `example-image.jpg`။

## GitHub Workflows

သင် pull request တင်တဲ့အခါ အောက်ပါ workflows များက ပြောင်းလဲမှုများကို စစ်ဆေးပေးပါလိမ့်မယ်။ သင့် pull request က workflow စစ်ဆေးမှုများကို ဖြတ်သန်းနိုင်ဖို့ အောက်ပါ ညွှန်ကြားချက်များကို လိုက်နာပါ။

- [Check Broken Relative Paths](../..)
- [Check URLs Don't Have Locale](../..)

### Check Broken Relative Paths

ဒီ workflow က သင့်ဖိုင်ထဲရှိ relative path များမှန်ကန်မှုကို သေချာစေပါတယ်။

1. သင့် link များမှန်ကန်စွာ လည်ပတ်နေကြောင်း သေချာစေရန် VS Code ကို အသုံးပြုပြီး အောက်ပါ လုပ်ဆောင်ချက်များ ပြုလုပ်ပါ။
    - ဖိုင်ထဲရှိ link များကို mouse cursor နဲ့ hover လုပ်ပါ။
    - **Ctrl + Click** နှိပ်ပြီး link ကို သွားပါ။
    - သင် link ကို နှိပ်ပြီး ဒေသတွင်းမှာ မအောင်မြင်ပါက workflow က အလုပ်မလုပ်ပါဘူး၊ GitHub ပေါ်မှာလည်း မအောင်မြင်ပါ။

1. ဒီပြဿနာကို ဖြေရှင်းဖို့ VS Code က path အကြံပြုချက်များကို အသုံးပြုပြီး အောက်ပါ လုပ်ဆောင်ချက်များ ပြုလုပ်ပါ။
    - `./` သို့မဟုတ် `../` ကို ရိုက်ထည့်ပါ။
    - VS Code က ရိုက်ထည့်ထားသည့်အတိုင်း ရွေးချယ်စရာများကို ပြပါလိမ့်မယ်။
    - သင့်လိုချင်တဲ့ ဖိုင် သို့မဟုတ် folder ကို နှိပ်ပြီး path မှန်ကန်မှုကို သေချာစေပါ။

မှန်ကန်တဲ့ relative path ကို ထည့်ပြီးနောက် ပြင်ဆင်မှုများကို သိမ်းပြီး push လုပ်ပါ။

### Check URLs Don't Have Locale

ဒီ workflow က ဝက်ဘ် URL များမှာ နိုင်ငံအလိုက် locale မပါရှိကြောင်း သေချာစေပါတယ်။ ဒီ repository ကို ကမ္ဘာတစ်ဝှမ်း အသုံးပြုနိုင်တဲ့အတွက် URL များမှာ သင့်နိုင်ငံ locale မပါရှိစေရန် အရေးကြီးပါတယ်။

1. သင့် URL များမှာ နိုင်ငံ locale မပါရှိကြောင်း စစ်ဆေးဖို့ အောက်ပါ လုပ်ဆောင်ချက်များ ပြုလုပ်ပါ။

    - URL များထဲမှာ `/en-us/`, `/en/` သို့မဟုတ် အခြားဘာသာစကား locale များ ရှိမရှိ စစ်ဆေးပါ။
    - မရှိပါက ဒီစစ်ဆေးမှုကို ဖြတ်သန်းနိုင်ပါသည်။

1. ပြဿနာဖြေရှင်းဖို့ အောက်ပါ လုပ်ဆောင်ချက်များ ပြုလုပ်ပါ။
    - workflow က ဖော်ပြထားတဲ့ ဖိုင်လမ်းကြောင်းကို ဖွင့်ပါ။
    - URL များထဲက နိုင်ငံ locale ကို ဖယ်ရှားပါ။

နိုင်ငံ locale ကို ဖယ်ရှားပြီးနောက် သိမ်းပြီး push လုပ်ပါ။

### Check Broken Urls

ဒီ workflow က သင့်ဖိုင်ထဲရှိ ဝက်ဘ် URL များ အလုပ်လုပ်နေပြီး 200 status code ပြန်လာကြောင်း သေချာစေပါတယ်။

1. သင့် URL များ မှန်ကန်စွာ လည်ပတ်နေကြောင်း စစ်ဆေးဖို့ အောက်ပါ လုပ်ဆောင်ချက်များ ပြုလုပ်ပါ။
    - ဖိုင်ထဲရှိ URL များ၏ status ကို စစ်ဆေးပါ။

2. ချို့ယွင်းနေသော URL များကို ပြင်ဆင်ဖို့ အောက်ပါ လုပ်ဆောင်ချက်များ ပြုလုပ်ပါ။
    - ချို့ယွင်းနေသော URL ပါဝင်သော ဖိုင်ကို ဖွင့်ပါ။
    - URL ကို မှန်ကန်သော URL ဖြင့် ပြင်ဆင်ပါ။

URL များကို ပြင်ဆင်ပြီးနောက် သိမ်းပြီး push လုပ်ပါ။

> [!NOTE]
>
> URL စစ်ဆေးမှု မအောင်မြင်သော်လည်း link သည် ဝင်ရောက်ကြည့်ရှုနိုင်သေးသည့် အခြေအနေများ ရှိနိုင်ပါသည်။ ၎င်းအကြောင်းရင်းများမှာ -
>
> - **ကွန်ယက်ကန့်သတ်ချက်များ**: GitHub actions server များတွင် အချို့ URL များသို့ ဝင်ရောက်ခြင်းကို ကန့်သတ်ထားနိုင်သည်။
> - **အချိန်ကုန်ဆုံးမှု ပြဿနာများ**: တုံ့ပြန်ရန် အချိန်ကြာသော URL များ workflow တွင် timeout error ဖြစ်စေနိုင်သည်။
> - **ဆာဗာ အချိန်ပိုင်း ပြဿနာများ**: ဆာဗာ မရရှိနိုင်ခြင်း သို့မဟုတ် ပြုပြင်ထိန်းသိမ်းမှုကြောင့် URL မရရှိနိုင်ခြင်း ဖြစ်နိုင်သည်။

**အကြောင်းကြားချက်**  
ဤစာတမ်းကို AI ဘာသာပြန်ဝန်ဆောင်မှု [Co-op Translator](https://github.com/Azure/co-op-translator) ဖြင့် ဘာသာပြန်ထားပါသည်။ ကျွန်ုပ်တို့သည် တိကျမှန်ကန်မှုအတွက် ကြိုးစားသော်လည်း အလိုအလျောက် ဘာသာပြန်ခြင်းတွင် အမှားများ သို့မဟုတ် မှားယွင်းချက်များ ပါဝင်နိုင်ကြောင်း သတိပြုပါရန် မေတ္တာရပ်ခံအပ်ပါသည်။ မူရင်းစာတမ်းကို မိမိဘာသာစကားဖြင့်သာ တရားဝင်အချက်အလက်အဖြစ် ယူဆသင့်ပါသည်။ အရေးကြီးသော အချက်အလက်များအတွက် လူ့ပညာရှင်များ၏ ပရော်ဖက်ရှင်နယ် ဘာသာပြန်ခြင်းကို အကြံပြုပါသည်။ ဤဘာသာပြန်ချက်ကို အသုံးပြုရာမှ ဖြစ်ပေါ်လာနိုင်သည့် နားလည်မှုမှားယွင်းမှုများအတွက် ကျွန်ုပ်တို့သည် တာဝန်မခံပါ။
