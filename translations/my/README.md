Phi သည် Microsoft မှ ဖန်တီးထားသော အခမဲ့ မော်ဒယ်များစီးရီးဖြစ်သည်။

Phi သည် လက်ရှိတွင် အင်္ဂါရပ်အသေးစား သော စကားလုံး မော်ဒယ် (SLM) များအနက် အင်အားကြီးဆုံးနှင့် ကောင်းမွန်သော စျေးနှုန်းထိရောက်မှုရှိပြီး၊ ဘာသာစကားများစုံ၊ သဘောထားထည့်ခြင်း၊ စာသား/စကားပြော ဖန်တီးခြင်း၊ ကုဒ်ရေးခြင်း၊ ဓာတ်ပုံများ၊ အသံနှင့် အခြားအသုံးအဆောင်များတွင် အံ့သြစရာ ကောင်းမွန်သော စမ်းသပ်မှုများရှိသည်။

Phi ကို တိုက်ဆိုင်ရာ device များ သို့မဟုတ် cloud ပေါ်တွင် တပ်ဆင်နိုင်ပြီး၊ သင်သည် ကွန်ပျူတာစွမ်းဆောင်ရည်တန်ဖိုးနည်းငယ်ဖြင့် ပြင်ဆင် AI အပ်ပလီကေးရှင်းများကို လွယ်ကူစွာ တည်ဆောက်နိုင်ပါသည်။

ဤအရင်းအမြစ် များကို အသုံးပြုရန် စတင်ခြင်းအတွက် အောက်ပါအဆင့်များကို လိုက်နာပါ။

>
> ဤ repository တွင် ဘာသာစကား ၅၀ ကျော် ဘာသာပြန်ချက်များ ပါဝင်ထားပြီး ဒါက ဒေါင်းလုပ် အရွယ်အစားကို ကြီးထွားစေသည်။ ဘာသာပြန်ချက်များ မပါဘဲ Clone လုပ်ချင်ပါက sparse checkout ကို အသုံးပြုနိုင်သည်။
>
> **Bash / macOS / Linux:**
> ```bash
> git clone --filter=blob:none --sparse https://github.com/microsoft/PhiCookBook.git
> cd PhiCookBook
> git sparse-checkout set --no-cone '/*' '!translations' '!translated_images'
> ```
>
> **CMD (Windows):**
> ```cmd
> git clone --filter=blob:none --sparse https://github.com/microsoft/PhiCookBook.git
> cd PhiCookBook
> git sparse-checkout set --no-cone "/*" "!translations" "!translated_images"
> ```
>
> ဒါက သင်တို့အား သင်တန်းကို အလျင် မြန်စွာ ပြီးမြောက်စေရန်လိုအပ်သည့် အရာအားလုံးကို ပေးစွမ်းနိုင်သည်။

## တာဝန်ရှိသော AI  

Microsoft သည် ဦးစားပေး၍ ဖောက်သည်များကို AI ထုတ်ကုန်များကို တာဝန်ရှိစွာ အသုံးပြုနိုင်ရေးနှင့် မျှဝေမှုများ၊ ယုံကြည်မှု အခြေပြု ပူးပေါင်းဆောင်ရွက်မှုများကို Transparency Notes နှင့် Impact Assessments ကဲ့သို့သော ကိရိယာများဖြင့် ဆောင်ရွက်လျက်ရှိသည်။ များစွာသော အရင်းအမြစ်များကို [https://aka.ms/RAI](https://aka.ms/RAI) တွင် ရရှိနိုင်ပါသည်။  
Microsoft ၏ တာဝန်ရှိသော AI ရဲ့ ဝါဒအခြေခံသည် တရားမျှတမှု၊ ယုံကြည်စိတ်ချရမှုနှင့် ဘေးကင်းလုံခြုံမှု၊ ကိုယ်ရေးကာကွယ်မှုနှင့် လုံခြုံမှု၊ ပါဝင်ဆောင်ရွက်မှု၊ ထွက်လွှတ်မှု၊ တာဝန်ခံမှုတို့ ဖြစ်သည်။  

ဤနမူနာတွင် အသုံးပြုသော စွမ်းရည်ကြီး အပြင်ဘာသာစကား၊ ပုံရိပ်နှင့် အသံ မော်ဒယ်ကြီးများသည် တရားမျှတမှုမရှိခြင်း၊ ယုံကြည်စိတ်ချရမှုမရှိခြင်း သို့မဟုတ် မတရားသော အပြုအမူ ပြုခြင်းများဖြစ်စေနိုင်ပြီး ဤအနေအထားများမှ နိုင်ငံသားများအား ထိခိုက်မှု ရှိစေနိုင်သည်။ ထို့ကြောင့် [Azure OpenAI ဝန်ဆောင်မှု Transparency note](https://learn.microsoft.com/legal/cognitive-services/openai/transparency-note?tabs=text) ကို ကြည့်ရှုပြီး အန္တရာယ်များနှင့် ကန့်သတ်ချက်များအား အသိပညာ ရရှိစေရန် တိုက်တွန်းပါသည်။  

ဤအန္တရာယ်များကို ကာကွယ်ရန် အကြံပြုချက်မှာ သင့်၏ ပန်းတိုင်သုံးသပ်မူများတွင် အန္တရာယ်ပြု သူများနှင့် မလိုလားအပ်သော အပြုအမူများကို ဖန်တီးမှု ရှောင်ရှားနိုင်ရန် ဘေးကင်းလုံခြုံရေး စနစ်တစ်ခု ထည့်သွင်းဆောင်ရွက်ရန် ဖြစ်သည်။ [Azure AI Content Safety](https://learn.microsoft.com/azure/ai-services/content-safety/overview) သည် လွတ်လပ်စွာကာကွယ်ပေးသော အလွှာတစ်ခုဖြစ်ပြီး အသုံးပြုသူနှင့် AI တို့မှ ဖန်တီးသော အန္တရာယ်ရှိသော အကြောင်းအရာများကို ပြန်လည် စစ်ဆေးနိုင်သည်။ Azure AI Content Safety တွင် စာသားနှင့် ပုံရိပ် API များ ပါဝင်ပြီး အန္တရာယ်ရှိသော အကြောင်းအရာများကို ရှာဖွေ့စစ်ဆေးနိုင်သည်။ Azure AI Foundry များတွင် Content Safety ဝန်ဆောင်မှုက ကွဲပြားသော ပုံစံများအတွင်း အန္တရာယ်ရှိသော အကြောင်းအရာများကို ကြည့်ရှု၊ စူးစမ်း၊ နမူနာ ကုဒ်များ စမ်းသပ်နိုင်ရန် ပံ့ပိုးပေးသည်။ အောက်ပါ [quickstart စာတမ်း](https://learn.microsoft.com/azure/ai-services/content-safety/quickstart-text?tabs=visual-studio%2Clinux&pivots=programming-language-rest) သည် ဝန်ဆောင်မှုထံ တောင်းဆိုမှုများ ပြုလုပ်ခြင်းကို ဦးစွာ လမ်းပြထားသည်။
အခြားတစ်ချက်အနေဖြင့် စဉ်းစားရန်မှာ စုစုပေါင်း အပလီကေးရှင်း၏ လုပ်ဆောင်မှုစွမ်းဆောင်ရည်ဖြစ်သည်။ Multi-modal နှင့် multi-models အပလီကေးရှင်းများနှင့်အတူ၊ စနစ်သည် သင်နှင့် သင့်အသုံးပြုသူများ မျှော်လင့်သကဲ့သို့ လုပ်ဆောင်ပေးခြင်း၊ ထိခိုက်မှုရှိသော အထွက်များ မထုတ်ပေးခြင်းတို့ကို သဘောတူသည်ဟု ဆိုလိုသည်။ သင့်အပလီကေးရှင်း၏ စုစုပေါင်း လုပ်ဆောင်မှုစွမ်းဆောင်ရည်ကို [Performance and Quality and Risk and Safety evaluators](https://learn.microsoft.com/azure/ai-studio/concepts/evaluation-metrics-built-in) ကို အသုံးပြု၍ အကဲဖြတ်ခြင်း အရေးကြီးပါသည်။ ထို့အပြင် [custom evaluators](https://learn.microsoft.com/azure/ai-studio/how-to/develop/evaluate-sdk#custom-evaluators) အသုံးပြု၍ ဖန်တီးခြင်းနှင့် အကဲဖြတ်ခြင်းလုပ်နိုင်သည်။

သင်သည် [Azure AI Evaluation SDK](https://microsoft.github.io/promptflow/index.html) ကို အသုံးပြု၍ သင်၏ AI အပလီကေးရှင်းကို ဖွံ့ဖြိုးတိုးတက်မှု ပတ်ဝန်းကျင်တွင် အကဲဖြတ်နိုင်သည်။ စမ်းသပ် ဒေတာသိုက် တစ်ခု သို့မဟုတ် ရည်မှန်းချက် တစ်ခုရှိပါက၊ သင့် generative AI အပလီကေးရှင်း၏ ထုတ်လုပ်မှုများကို အလိုအလျောက် built-in evaluators သို့မဟုတ် သင်နှစ်သက်သည့် custom evaluators ဖြင့် အရေအတွက်အတွက် တိုင်းတာနိုင်သည်။ သင့်စနစ်ကို အကဲဖြတ်ရန် azure ai evaluation sdk ဖြင့် စတင်ရန်အတွက် [quickstart guide](https://learn.microsoft.com/azure/ai-studio/how-to/develop/flow-evaluate-sdk) ကို လိုက်နာနိုင်ပါသည်။ အကဲဖြတ်မှု လုပ်ငန်းစဉ်တစ်ခု ပြုလုပ်ပြီးလျှင် [Azure AI Foundry တွင် ရလဒ်များကို ကြည့်ရှု](https://learn.microsoft.com/azure/ai-studio/how-to/evaluate-flow-results) နိုင်ပါသည်။
