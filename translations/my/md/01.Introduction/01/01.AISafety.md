# Phi မော်ဒယ်များအတွက် AI လုံခြုံရေး  
Phi မော်ဒယ်များသည် [Microsoft Responsible AI Standard](https://query.prod.cms.rt.microsoft.com/cms/api/am/binary/RE5cmFl) အတိုင်း ဖွံ့ဖြိုးတည်ဆောက်ထားပြီး၊ ၎င်းသည် ကုမ္ပဏီအနှံ့ရှိ တာဝန်ယူမှု၊ ထင်ရှားမှု၊ တရားမျှတမှု၊ ယုံကြည်စိတ်ချရမှုနှင့် လုံခြုံမှု၊ ကိုယ်ရေးကိုယ်တာနှင့် လုံခြုံရေး၊ အပါအဝင် [Microsoft ၏ တာဝန်ယူသော AI 원칙များ](https://www.microsoft.com/ai/responsible-ai) အခြေခံထားသော ခြောက်ချက်အခြေခံသဘောတရားများဖြစ်သည်။  

ယခင် Phi မော်ဒယ်များကဲ့သို့ပင်၊ လုံခြုံရေးအကဲဖြတ်ခြင်းနှင့် လုံခြုံရေးလေ့ကျင့်မှုများကို မျိုးစုံအသုံးပြုခဲ့ပြီး၊ ဤထုတ်ဝေမှု၏ ဘာသာစကားစွမ်းရည်များအတွက် ထပ်မံကာကွယ်မှုများကိုလည်း ထည့်သွင်းစဉ်းစားထားသည်။ ဘာသာစကားစုံနှင့် အန္တရာယ်အမျိုးအစားများအပေါ် စမ်းသပ်ခြင်းအပါအဝင် လုံခြုံရေးလေ့ကျင့်မှုနှင့် အကဲဖြတ်မှုများကို [Phi Safety Post-Training Paper](https://arxiv.org/abs/2407.13833) တွင် ဖော်ပြထားသည်။ Phi မော်ဒယ်များသည် ဤနည်းလမ်းမှ အကျိုးခံစားနိုင်သော်လည်း၊ ဖွံ့ဖြိုးသူများသည် ၎င်းတို့၏ အသုံးပြုမှုအခြေအနေ၊ ယဉ်ကျေးမှုနှင့် ဘာသာစကားဆိုင်ရာ အခြေအနေများနှင့် သက်ဆိုင်သော အန္တရာယ်များကို မြေပုံဆွဲခြင်း၊ တိုင်းတာခြင်းနှင့် လျော့နည်းစေရေးအတွက် တာဝန်ယူသော AI အကောင်းဆုံးလေ့ကျင့်မှုများကို လိုက်နာရန် လိုအပ်သည်။  

## အကောင်းဆုံးလေ့ကျင့်မှုများ  

အခြားမော်ဒယ်များကဲ့သို့ Phi မော်ဒယ်များသည် မတရားမှု၊ ယုံကြည်စိတ်ချရမှုမရှိမှု သို့မဟုတ် မလေးစားဖွယ် အပြုအမူများ ပြုလုပ်နိုင်သည်။  

SLM နှင့် LLM မော်ဒယ်များ၏ ကန့်သတ်ချက်အချို့မှာ အောက်ပါအတိုင်း ဖြစ်သည်-  

- **ဝန်ဆောင်မှုအရည်အသွေး:** Phi မော်ဒယ်များကို အဓိကအားဖြင့် အင်္ဂလိပ်စာပေါ်တွင် လေ့ကျင့်ထားသည်။ အင်္ဂလိပ်မဟုတ်သော ဘာသာစကားများတွင် စွမ်းဆောင်ရည်နည်းပါးနိုင်ပြီး၊ လေ့ကျင့်မှုဒေတာတွင် ကိုယ်စားပြုမှုနည်းသော အင်္ဂလိပ်ဘာသာစကားမျိုးစုံသည် စံအမေရိကန်အင်္ဂလိပ်ထက် ပိုမိုနည်းပါးသော စွမ်းဆောင်ရည်ရှိနိုင်သည်။  
- **ထိခိုက်မှုများကို ကိုယ်စားပြုခြင်းနှင့် စတီရီယိုတိုက်များ ထပ်တိုးခြင်း:** ဤမော်ဒယ်များသည် လူအုပ်စုတချို့ကို များစွာ သို့မဟုတ် နည်းပါးစွာ ကိုယ်စားပြုနိုင်ပြီး၊ အချို့အုပ်စုများကို ဖျက်ဆီးပစ်နိုင်သလို၊ အနုတ်လက္ခဏာများ သို့မဟုတ် အနုတ်လက္ခဏာများပါသော စတီရီယိုတိုက်များကို ထပ်တိုးပေးနိုင်သည်။ လုံခြုံရေးလေ့ကျင့်ပြီးနောက်ဖြစ်ပေမယ့်၊ လူအုပ်စုများ၏ ကိုယ်စားပြုမှုအဆင့်ကွာခြားမှုများ သို့မဟုတ် လေ့ကျင့်မှုဒေတာတွင် ရှိသော အနုတ်လက္ခဏာများပါသော စတီရီယိုတိုက်များ၏ ပုံမှန်ဖြစ်ပေါ်မှုကြောင့် ဤကန့်သတ်ချက်များ မရှိမဖြစ် ရှိနေနိုင်သည်။  
- **မသင့်တော်သော သို့မဟုတ် မလေးစားဖွယ် အကြောင်းအရာများ:** ဤမော်ဒယ်များသည် မသင့်တော်သော သို့မဟုတ် မလေးစားဖွယ် အကြောင်းအရာများကို ထုတ်ပေးနိုင်ပြီး၊ အသုံးပြုမှုအခြေအနေအပေါ် မူတည်၍ ထပ်မံကာကွယ်မှုများမရှိပါက အထူးသဖြင့် အာရုံစိုက်ရမည့် နေရာများတွင် အသုံးပြုရန် မသင့်တော်နိုင်သည်။  
- **သတင်းအချက်အလက် ယုံကြည်စိတ်ချရမှု:** ဘာသာစကားမော်ဒယ်များသည် အဓိပ္ပါယ်မရှိသော အကြောင်းအရာများကို ဖန်တီးပေးနိုင်သလို၊ သင့်တော်သည့်အသံရှိသော်လည်း မှားယွင်းသော်လည်း သက်ဆိုင်မှုမရှိသည့် သတင်းအချက်အလက်များကို ဖန်တီးပေးနိုင်သည်။  
- **ကုဒ်ရေးသားမှုအတွက် ကန့်သတ်ချက်များ:** Phi-3 ၏ လေ့ကျင့်မှုဒေတာအများစုသည် Python ကို အခြေခံပြီး "typing, math, random, collections, datetime, itertools" ကဲ့သို့သော ပက်ကေ့ဂျ်များကို အသုံးပြုသည်။ မော်ဒယ်မှ အခြားပက်ကေ့ဂျ်များ သို့မဟုတ် အခြားဘာသာစကားများဖြင့် Python စာရင်းများ ထုတ်ပေးပါက အသုံးပြုသူများသည် API အသုံးပြုမှုအားလုံးကို လက်ဖြင့် စစ်ဆေးရန် အကြံပြုသည်။  

ဖွံ့ဖြိုးသူများသည် တာဝန်ယူသော AI အကောင်းဆုံးလေ့ကျင့်မှုများကို လိုက်နာရမည်ဖြစ်ပြီး၊ သက်ဆိုင်ရာ ဥပဒေများနှင့် စည်းမျဉ်းများ (ဥပမာ- ကိုယ်ရေးကိုယ်တာ၊ ကုန်သွယ်မှု စသည်) နှင့် ကိုက်ညီမှုရှိရန် တာဝန်ရှိသည်။  

## တာဝန်ယူသော AI စဉ်းစားချက်များ  

အခြားဘာသာစကားမော်ဒယ်များကဲ့သို့ Phi မော်ဒယ်များသည် မတရားမှု၊ ယုံကြည်စိတ်ချရမှုမရှိမှု သို့မဟုတ် မလေးစားဖွယ် အပြုအမူများ ပြုလုပ်နိုင်သည်။ သတိပြုရမည့် ကန့်သတ်ချက်အချို့မှာ-  

**ဝန်ဆောင်မှုအရည်အသွေး:** Phi မော်ဒယ်များကို အဓိကအားဖြင့် အင်္ဂလိပ်စာပေါ်တွင် လေ့ကျင့်ထားသည်။ အင်္ဂလိပ်မဟုတ်သော ဘာသာစကားများတွင် စွမ်းဆောင်ရည်နည်းပါးနိုင်သည်။ လေ့ကျင့်မှုဒေတာတွင် ကိုယ်စားပြုမှုနည်းသော အင်္ဂလိပ်ဘာသာစကားမျိုးစုံသည် စံအမေရိကန်အင်္ဂလိပ်ထက် ပိုမိုနည်းပါးသော စွမ်းဆောင်ရည်ရှိနိုင်သည်။  

**ထိခိုက်မှုများကို ကိုယ်စားပြုခြင်းနှင့် စတီရီယိုတိုက်များ ထပ်တိုးခြင်း:** ဤမော်ဒယ်များသည် လူအုပ်စုတချို့ကို များစွာ သို့မဟုတ် နည်းပါးစွာ ကိုယ်စားပြုနိုင်ပြီး၊ အချို့အုပ်စုများကို ဖျက်ဆီးပစ်နိုင်သလို၊ အနုတ်လက္ခဏာများ သို့မဟုတ် အနုတ်လက္ခဏာများပါသော စတီရီယိုတိုက်များကို ထပ်တိုးပေးနိုင်သည်။ လုံခြုံရေးလေ့ကျင့်ပြီးနောက်ဖြစ်ပေမယ့်၊ လူအုပ်စုများ၏ ကိုယ်စားပြုမှုအဆင့်ကွာခြားမှုများ သို့မဟုတ် လေ့ကျင့်မှုဒေတာတွင် ရှိသော အနုတ်လက္ခဏာများပါသော စတီရီယိုတိုက်များ၏ ပုံမှန်ဖြစ်ပေါ်မှုကြောင့် ဤကန့်သတ်ချက်များ မရှိမဖြစ် ရှိနေနိုင်သည်။  

**မသင့်တော်သော သို့မဟုတ် မလေးစားဖွယ် အကြောင်းအရာများ:** ဤမော်ဒယ်များသည် မသင့်တော်သော သို့မဟုတ် မလေးစားဖွယ် အကြောင်းအရာများကို ထုတ်ပေးနိုင်ပြီး၊ အသုံးပြုမှုအခြေအနေအပေါ် မူတည်၍ ထပ်မံကာကွယ်မှုများမရှိပါက အထူးသဖြင့် အာရုံစိုက်ရမည့် နေရာများတွင် အသုံးပြုရန် မသင့်တော်နိုင်သည်။  
သတင်းအချက်အလက် ယုံကြည်စိတ်ချရမှု: ဘာသာစကားမော်ဒယ်များသည် အဓိပ္ပါယ်မရှိသော အကြောင်းအရာများကို ဖန်တီးပေးနိုင်သလို၊ သင့်တော်သည့်အသံရှိသော်လည်း မှားယွင်းသော်လည်း သက်ဆိုင်မှုမရှိသည့် သတင်းအချက်အလက်များကို ဖန်တီးပေးနိုင်သည်။  

**ကုဒ်ရေးသားမှုအတွက် ကန့်သတ်ချက်များ:** Phi-3 ၏ လေ့ကျင့်မှုဒေတာအများစုသည် Python ကို အခြေခံပြီး "typing, math, random, collections, datetime, itertools" ကဲ့သို့သော ပက်ကေ့ဂျ်များကို အသုံးပြုသည်။ မော်ဒယ်မှ အခြားပက်ကေ့ဂျ်များ သို့မဟုတ် အခြားဘာသာစကားများဖြင့် Python စာရင်းများ ထုတ်ပေးပါက အသုံးပြုသူများသည် API အသုံးပြုမှုအားလုံးကို လက်ဖြင့် စစ်ဆေးရန် အကြံပြုသည်။  

ဖွံ့ဖြိုးသူများသည် တာဝန်ယူသော AI အကောင်းဆုံးလေ့ကျင့်မှုများကို လိုက်နာရမည်ဖြစ်ပြီး၊ သက်ဆိုင်ရာ ဥပဒေများနှင့် စည်းမျဉ်းများ (ဥပမာ- ကိုယ်ရေးကိုယ်တာ၊ ကုန်သွယ်မှု စသည်) နှင့် ကိုက်ညီမှုရှိရန် တာဝန်ရှိသည်။ စဉ်းစားရန် အရေးကြီးသော နေရာများမှာ-  

**အရင်းအမြစ်ခွဲဝေမှု:** ဤမော်ဒယ်များကို ဥပဒေရေးရာ အခြေအနေ သို့မဟုတ် အရင်းအမြစ်များ သို့မဟုတ် ဘဝအခွင့်အလမ်းများ ခွဲဝေမှုအပေါ် ထိခိုက်မှုရှိနိုင်သည့် အခြေအနေများတွင် အသုံးမပြုသင့်ပါ (ဥပမာ- အိမ်ရာ၊ အလုပ်အကိုင်၊ ခရက်ဒစ် စသည်)။ ထပ်မံသုံးသပ်ခြင်းနှင့် အနုတ်လက္ခဏာဖယ်ရှားခြင်းနည်းလမ်းများ လိုအပ်သည်။  

**အန္တရာယ်မြင့် အခြေအနေများ:** ဖွံ့ဖြိုးသူများသည် မတရားမှု၊ ယုံကြည်စိတ်ချရမှုမရှိမှု သို့မဟုတ် မလေးစားဖွယ် အဖြေများကြောင့် အလွန်ကြီးမားသော အန္တရာယ်ရှိနိုင်သည့် အခြေအနေများတွင် မော်ဒယ်အသုံးပြုမှု သင့်တော်မှုကို သုံးသပ်ရမည်။ ဤတွင် တိကျမှန်ကန်မှုနှင့် ယုံကြည်စိတ်ချရမှု အရေးကြီးသော အထူးသဖြင့် ဥပဒေရေးရာ သို့မဟုတ် ကျန်းမာရေးအကြံပေးမှုများ ပါဝင်သည်။ အသုံးပြုမှုအခြေအနေအရ အပလီကေးရှင်းအဆင့်တွင် ထပ်မံကာကွယ်မှုများ ထည့်သွင်းရမည်။  

**မှားယွင်းသတင်းအချက်အလက်:** မော်ဒယ်များသည် မှားယွင်းသတင်းအချက်အလက် ထုတ်ပေးနိုင်သည်။ ဖွံ့ဖြိုးသူများသည် ထင်ရှားမှုအကောင်းဆုံးလေ့ကျင့်မှုများကို လိုက်နာပြီး၊ အသုံးပြုသူများအား AI စနစ်နှင့် ဆက်သွယ်နေကြောင်း အသိပေးရမည်။ အပလီကေးရှင်းအဆင့်တွင် ဖွံ့ဖြိုးသူများသည် တုံ့ပြန်ချက်များကို အသုံးပြုမှုအခြေအနေနှင့် သက်ဆိုင်ရာ သတင်းအချက်အလက်များအပေါ် အခြေခံ၍ တည်ဆောက်နိုင်သော ပြန်လည်ရှာဖွေရေးနည်းလမ်း (Retrieval Augmented Generation - RAG) ကို အသုံးပြုနိုင်သည်။  

**ထိခိုက်မှုရှိသော အကြောင်းအရာ ထုတ်ပေးခြင်း:** ဖွံ့ဖြိုးသူများသည် ထုတ်လွှင့်ချက်များကို ၎င်းတို့၏ အသုံးပြုမှုအခြေအနေအရ သုံးသပ်ပြီး၊ လုံခြုံရေး စစ်ဆေးသူများ သို့မဟုတ် အသုံးပြုမှုအတွက် သင့်တော်သော စိတ်ကြိုက်ဖြေရှင်းနည်းများကို အသုံးပြုသင့်သည်။  

**မမှန်ကန်သော အသုံးပြုမှု:** လိမ်လည်မှု၊ စပမ်၊ မော်လ်ဝဲ ထုတ်လုပ်မှုကဲ့သို့ မမှန်ကန်သော အသုံးပြုမှုအမျိုးအစားများ ဖြစ်နိုင်ပြီး၊ ဖွံ့ဖြိုးသူများသည် ၎င်းတို့၏ အပလီကေးရှင်းများသည် သက်ဆိုင်ရာ ဥပဒေများနှင့် စည်းမျဉ်းများကို မဖောက်ဖျက်စေရန် သေချာစေရမည်။  

### ဖိုင်နက်တူနင်းနှင့် AI အကြောင်းအရာ လုံခြုံရေး  

မော်ဒယ်တစ်ခုကို ဖိုင်နက်တူနင်းပြီးနောက်၊ [Azure AI Content Safety](https://learn.microsoft.com/azure/ai-services/content-safety/overview) ကို အသုံးပြု၍ မော်ဒယ်များထုတ်ပေးသော အကြောင်းအရာများကို စောင့်ကြည့်၊ အန္တရာယ်များ၊ ခြိမ်းခြောက်မှုများနှင့် အရည်အသွေးပြဿနာများကို ရှာဖွေတားဆီးရန် အလွန်အကြံပြုသည်။
