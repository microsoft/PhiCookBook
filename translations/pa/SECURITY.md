## ਸੁਰੱਖਿਆ

Microsoft ਸਾਡੇ ਸਾਫਟਵੇਅਰ ਉਤਪਾਦਾਂ ਅਤੇ ਸੇਵਾਵਾਂ ਦੀ ਸੁਰੱਖਿਆ ਨੂੰ ਗੰਭੀਰਤਾ ਨਾਲ ਲੈਂਦਾ ਹੈ, ਜਿਸ ਵਿੱਚ ਸਾਡੇ GitHub ਸੰਗਠਨਾਂ ਰਾਹੀਂ ਪ੍ਰਬੰਧਿਤ ਸਾਰੇ ਸਰੋਤ ਕੋਡ ਰਿਪੋਜ਼ਿਟਰੀਜ਼ ਸ਼ਾਮਲ ਹਨ, ਜਿਨ੍ਹਾਂ ਵਿੱਚ [Microsoft](https://github.com/Microsoft), [Azure](https://github.com/Azure), [DotNet](https://github.com/dotnet), [AspNet](https://github.com/aspnet) ਅਤੇ [Xamarin](https://github.com/xamarin) ਸ਼ਾਮਲ ਹਨ।

ਜੇ ਤੁਸੀਂ ਕਿਸੇ Microsoft-ਮਲਕੀਅਤ ਰਿਪੋਜ਼ਿਟਰੀ ਵਿੱਚ ਕੋਈ ਸੁਰੱਖਿਆ ਦੀ ਖਾਮੀ ਲੱਭਦੇ ਹੋ ਜੋ [Microsoft ਦੀ ਸੁਰੱਖਿਆ ਖਾਮੀ ਦੀ ਪਰਿਭਾਸ਼ਾ](https://aka.ms/security.md/definition) ਨੂੰ ਪੂਰਾ ਕਰਦੀ ਹੈ, ਤਾਂ ਕਿਰਪਾ ਕਰਕੇ ਹੇਠਾਂ ਦਿੱਤੇ ਤਰੀਕੇ ਅਨੁਸਾਰ ਸਾਨੂੰ ਇਸ ਬਾਰੇ ਸੂਚਿਤ ਕਰੋ।

## ਸੁਰੱਖਿਆ ਸਮੱਸਿਆਵਾਂ ਦੀ ਰਿਪੋਰਟਿੰਗ

**ਕਿਰਪਾ ਕਰਕੇ ਸੁਰੱਖਿਆ ਖਾਮੀਆਂ ਨੂੰ ਪਬਲਿਕ GitHub issues ਰਾਹੀਂ ਰਿਪੋਰਟ ਨਾ ਕਰੋ।**

ਇਸ ਦੀ ਥਾਂ, ਕਿਰਪਾ ਕਰਕੇ Microsoft Security Response Center (MSRC) ਨੂੰ [https://msrc.microsoft.com/create-report](https://aka.ms/security.md/msrc/create-report) 'ਤੇ ਰਿਪੋਰਟ ਕਰੋ।

ਜੇ ਤੁਸੀਂ ਲੌਗਿਨ ਕੀਤੇ ਬਿਨਾਂ ਜਮ੍ਹਾਂ ਕਰਵਾਉਣਾ ਚਾਹੁੰਦੇ ਹੋ, ਤਾਂ ਈਮੇਲ ਭੇਜੋ [secure@microsoft.com](mailto:secure@microsoft.com) ਨੂੰ। ਜੇ ਸੰਭਵ ਹੋਵੇ, ਤਾਂ ਸਾਡੇ PGP ਕੁੰਜੀ ਨਾਲ ਆਪਣਾ ਸੁਨੇਹਾ ਇੰਕ੍ਰਿਪਟ ਕਰੋ; ਕਿਰਪਾ ਕਰਕੇ ਇਸਨੂੰ [Microsoft Security Response Center PGP Key page](https://aka.ms/security.md/msrc/pgp) ਤੋਂ ਡਾਊਨਲੋਡ ਕਰੋ।

ਤੁਹਾਨੂੰ 24 ਘੰਟਿਆਂ ਦੇ ਅੰਦਰ ਜਵਾਬ ਮਿਲਣਾ ਚਾਹੀਦਾ ਹੈ। ਜੇ ਕਿਸੇ ਕਾਰਨ ਤੁਸੀਂ ਜਵਾਬ ਨਾ ਮਿਲਣ, ਤਾਂ ਈਮੇਲ ਰਾਹੀਂ ਫਾਲੋਅੱਪ ਕਰੋ ਤਾਂ ਜੋ ਇਹ ਯਕੀਨੀ ਬਣਾਇਆ ਜਾ ਸਕੇ ਕਿ ਸਾਡੀ ਟੀਮ ਨੇ ਤੁਹਾਡਾ ਮੁਢਲਾ ਸੁਨੇਹਾ ਪ੍ਰਾਪਤ ਕਰ ਲਿਆ ਹੈ। ਵਾਧੂ ਜਾਣਕਾਰੀ ਲਈ [microsoft.com/msrc](https://www.microsoft.com/msrc) ਵੇਖੋ।

ਕਿਰਪਾ ਕਰਕੇ ਹੇਠਾਂ ਦਿੱਤੀ ਗਈ ਜਾਣਕਾਰੀ (ਜਿੰਨੀ ਤੁਸੀਂ ਦੇ ਸਕਦੇ ਹੋ) ਸ਼ਾਮਲ ਕਰੋ ਤਾਂ ਜੋ ਅਸੀਂ ਮਸਲੇ ਦੀ ਕਿਸਮ ਅਤੇ ਪੈਮਾਨੇ ਨੂੰ ਬਿਹਤਰ ਸਮਝ ਸਕੀਏ:

  * ਸਮੱਸਿਆ ਦੀ ਕਿਸਮ (ਜਿਵੇਂ ਕਿ ਬਫਰ ਓਵਰਫਲੋ, SQL ਇੰਜੈਕਸ਼ਨ, ਕ੍ਰਾਸ-ਸਾਈਟ ਸਕ੍ਰਿਪਟਿੰਗ, ਆਦਿ)
  * ਸਮੱਸਿਆ ਦੇ ਪ੍ਰਗਟਾਵੇ ਨਾਲ ਸਬੰਧਿਤ ਸਰੋਤ ਫਾਈਲ(ਆਂ) ਦੇ ਪੂਰੇ ਪਾਥ
  * ਪ੍ਰਭਾਵਿਤ ਸਰੋਤ ਕੋਡ ਦੀ ਸਥਿਤੀ (tag/branch/commit ਜਾਂ ਸਿੱਧਾ URL)
  * ਸਮੱਸਿਆ ਨੂੰ ਦੁਹਰਾਉਣ ਲਈ ਕੋਈ ਖਾਸ ਸੰਰਚਨਾ
  * ਸਮੱਸਿਆ ਨੂੰ ਦੁਹਰਾਉਣ ਲਈ ਕਦਮ-ਦਰ-ਕਦਮ ਨਿਰਦੇਸ਼
  * ਪ੍ਰਮਾਣ-ਆਧਾਰ ਜਾਂ exploit ਕੋਡ (ਜੇ ਸੰਭਵ ਹੋਵੇ)
  * ਸਮੱਸਿਆ ਦਾ ਪ੍ਰਭਾਵ, ਜਿਸ ਵਿੱਚ ਇਹ ਵੀ ਸ਼ਾਮਲ ਹੈ ਕਿ ਕਿਸ ਤਰ੍ਹਾਂ ਕੋਈ ਹਮਲਾਵਰ ਇਸਨੂੰ ਨੁਕਸਾਨ ਪਹੁੰਚਾ ਸਕਦਾ ਹੈ

ਇਹ ਜਾਣਕਾਰੀ ਸਾਡੀ ਟੀਮ ਨੂੰ ਤੁਹਾਡੀ ਰਿਪੋਰਟ ਨੂੰ ਜਲਦੀ ਤਰਤੀਬ ਦੇਣ ਵਿੱਚ ਮਦਦ ਕਰੇਗੀ।

ਜੇ ਤੁਸੀਂ bug bounty ਲਈ ਰਿਪੋਰਟ ਕਰ ਰਹੇ ਹੋ, ਤਾਂ ਵਧੇਰੇ ਪੂਰੀ ਰਿਪੋਰਟਾਂ ਨਾਲ ਵੱਡੀ ਇਨਾਮ ਰਕਮ ਮਿਲ ਸਕਦੀ ਹੈ। ਕਿਰਪਾ ਕਰਕੇ ਸਾਡੇ [Microsoft Bug Bounty Program](https://aka.ms/security.md/msrc/bounty) ਪੇਜ਼ 'ਤੇ ਜਾ ਕੇ ਸਾਡੇ ਸਰਗਰਮ ਪ੍ਰੋਗਰਾਮਾਂ ਬਾਰੇ ਹੋਰ ਜਾਣਕਾਰੀ ਲਵੋ।

## ਪਸੰਦੀਦਾ ਭਾਸ਼ਾਵਾਂ

ਸਾਨੂੰ ਸਾਰੀਆਂ ਸੰਚਾਰਾਂ ਲਈ ਅੰਗਰੇਜ਼ੀ ਭਾਸ਼ਾ ਪਸੰਦ ਹੈ।

## ਨੀਤੀ

Microsoft [Coordinated Vulnerability Disclosure](https://aka.ms/security.md/cvd) ਦੇ ਸਿਧਾਂਤ ਦੀ ਪਾਲਣਾ ਕਰਦਾ ਹੈ।

**ਅਸਵੀਕਾਰੋक्ति**:  
ਇਹ ਦਸਤਾਵੇਜ਼ AI ਅਨੁਵਾਦ ਸੇਵਾ [Co-op Translator](https://github.com/Azure/co-op-translator) ਦੀ ਵਰਤੋਂ ਕਰਕੇ ਅਨੁਵਾਦ ਕੀਤਾ ਗਿਆ ਹੈ। ਜਦੋਂ ਕਿ ਅਸੀਂ ਸਹੀਅਤ ਲਈ ਕੋਸ਼ਿਸ਼ ਕਰਦੇ ਹਾਂ, ਕਿਰਪਾ ਕਰਕੇ ਧਿਆਨ ਵਿੱਚ ਰੱਖੋ ਕਿ ਸਵੈਚਾਲਿਤ ਅਨੁਵਾਦਾਂ ਵਿੱਚ ਗਲਤੀਆਂ ਜਾਂ ਅਸਮਰਥਤਾਵਾਂ ਹੋ ਸਕਦੀਆਂ ਹਨ। ਮੂਲ ਦਸਤਾਵੇਜ਼ ਆਪਣੀ ਮੂਲ ਭਾਸ਼ਾ ਵਿੱਚ ਹੀ ਪ੍ਰਮਾਣਿਕ ਸਰੋਤ ਮੰਨਿਆ ਜਾਣਾ ਚਾਹੀਦਾ ਹੈ। ਜਰੂਰੀ ਜਾਣਕਾਰੀ ਲਈ, ਪੇਸ਼ੇਵਰ ਮਨੁੱਖੀ ਅਨੁਵਾਦ ਦੀ ਸਿਫਾਰਸ਼ ਕੀਤੀ ਜਾਂਦੀ ਹੈ। ਅਸੀਂ ਇਸ ਅਨੁਵਾਦ ਦੀ ਵਰਤੋਂ ਨਾਲ ਹੋਣ ਵਾਲੀਆਂ ਕਿਸੇ ਵੀ ਗਲਤਫਹਮੀਆਂ ਜਾਂ ਗਲਤ ਵਿਵਖਿਆਵਾਂ ਲਈ ਜ਼ਿੰਮੇਵਾਰ ਨਹੀਂ ਹਾਂ।
