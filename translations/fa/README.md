فی مجموعه‌ای از مدل‌های هوش مصنوعی متن‌باز است که توسط مایکروسافت توسعه یافته‌اند.

فی در حال حاضر قدرتمندترین و مقرون به صرفه‌ترین مدل زبان کوچک (SLM) است که معیارهای بسیار خوبی در زبان‌های چندگانه، استدلال، تولید متن/چت، کدنویسی، تصاویر، صدا و سایر سناریوها دارد.

شما می‌توانید فی را در فضای ابری یا دستگاه‌های لبه اجرا کنید و به‌راحتی برنامه‌های هوش مصنوعی مولد را با قدرت محاسباتی محدود بسازید.

برای شروع استفاده از این منابع، مراحل زیر را دنبال کنید:
1. **شاخۀ مخزن را فورک کنید**: کلیک کنید روی [![شاخۀ GitHub](https://img.shields.io/github/forks/microsoft/phicookbook.svg?style=social&label=Fork)](https://GitHub.com/microsoft/phicookbook/network/?WT.mc_id=aiml-137032-kinfeylo)
2. **مخزن را کلون کنید**: `git clone https://github.com/microsoft/PhiCookBook.git`
3. [**به انجمن دیسکورد هوش مصنوعی مایکروسافت بپیوندید و با کارشناسان و توسعه‌دهندگان دیگر ملاقات کنید**](https://discord.com/invite/ByRwuEEgH4?WT.mc_id=aiml-137032-kinfeylo)

> **ترجیح می‌دهید به صورت محلی کلون کنید؟**
>
> این مخزن بیش از ۵۰ ترجمه زبان دارد که اندازه دانلود را به طور قابل توجهی افزایش می‌دهد. برای کلون کردن بدون ترجمه‌ها، از sparse checkout استفاده کنید:
>
> **Bash / macOS / Linux:**
> ```bash
> git clone --filter=blob:none --sparse https://github.com/microsoft/PhiCookBook.git
> cd PhiCookBook
> git sparse-checkout set --no-cone '/*' '!translations' '!translated_images'
> ```
>
> **CMD (ویندوز):**
> ```cmd
> git clone --filter=blob:none --sparse https://github.com/microsoft/PhiCookBook.git
> cd PhiCookBook
> git sparse-checkout set --no-cone "/*" "!translations" "!translated_images"
> ```
>
> این به شما همه چیز لازم برای کامل کردن دوره را با دانلود بسیار سریع‌تر می‌دهد.

>
> **بش / مک‌او‌اس / لینوکس:**
> ```bash
> git clone --filter=blob:none --sparse https://github.com/microsoft/PhiCookBook.git
> cd PhiCookBook
> git sparse-checkout set --no-cone '/*' '!translations' '!translated_images'
> ```
>
> **CMD (ویندوز):**
> ```cmd
> git clone --filter=blob:none --sparse https://github.com/microsoft/PhiCookBook.git
> cd PhiCookBook
> git sparse-checkout set --no-cone "/*" "!translations" "!translated_images"
> ```
>

می‌توانید یاد بگیرید چگونه فی مایکروسافت را استفاده کنید و چگونه راه‌حل‌های E2E را روی دستگاه‌های سخت‌افزاری مختلف خود بسازید. برای تجربه فی خودتان، با بازی کردن با مدل و سفارشی‌سازی فی برای سناریوهای خود از طریق [کتابخانه مدل GitHub](https://github.com/marketplace/models?WT.mc_id=aiml-137032-kinfeylo) شروع کنید. می‌توانید بیشتر در مورد شروع کار با [کتابخانه مدل GitHub](/md/02.QuickStart/GitHubModel_QuickStart.md) بدانید.

**محیط آزمایشی**

## هوش مصنوعی مسئولانه

شرکت مایکروسافت متعهد به کمک به مشتریان خود برای استفاده مسئولانه از محصولات هوش مصنوعی ما، به اشتراک‌گذاری آموخته‌هایمان و ایجاد همکاری‌های مبتنی بر اعتماد از طریق ابزارهایی مانند یادداشت‌های شفافیت و ارزیابی‌های تأثیر است. بسیاری از این منابع را می‌توانید در [https://aka.ms/RAI](https://aka.ms/RAI) بیابید.
رویکرد مایکروسافت به هوش مصنوعی مسئولانه بر اصول هوش مصنوعی ما بر پایه عدالت، قابلیت اطمینان و ایمنی، حریم خصوصی و امنیت، فراگیری، شفافیت و پاسخگویی استوار است.

مدل‌های زبان طبیعی، تصویر و گفتار در مقیاس بزرگ - مانند مدل‌هایی که در این نمونه استفاده شده‌اند - ممکن است به گونه‌ای رفتار کنند که ناعادلانه، غیرقابل اعتماد یا توهین‌آمیز باشد، که به نوبه خود می‌تواند باعث آسیب شود. لطفاً برای اطلاع از ریسک‌ها و محدودیت‌ها به [یادداشت شفافیت سرویس Azure OpenAI](https://learn.microsoft.com/legal/cognitive-services/openai/transparency-note?tabs=text) مراجعه کنید.

رویکرد پیشنهادی برای کاهش این ریسک‌ها، افزودن یک سیستم ایمنی در معماری شما است که قادر به شناسایی و جلوگیری از رفتارهای مضر باشد. [ایمنی محتوای هوش مصنوعی Azure](https://learn.microsoft.com/azure/ai-services/content-safety/overview) یک لایه مستقل از حفاظت فراهم می‌کند که می‌تواند محتوای تولید شده توسط کاربر و تولید شده توسط هوش مصنوعی را در برنامه‌ها و خدمات تشخیص دهد. ایمنی محتوای هوش مصنوعی Azure شامل APIهای متن و تصویر است که امکان شناسایی محتوای مضر را فراهم می‌آورد. در Microsoft Foundry، سرویس ایمنی محتوا به شما امکان مشاهده، کاوش و آزمایش نمونه کدهایی برای شناسایی محتوای مضر در زمینه‌های مختلف را می‌دهد. مستندات شروع سریع زیر شما را در ارسال درخواست‌ها به این سرویس راهنمایی می‌کند: [quickstart documentation](https://learn.microsoft.com/azure/ai-services/content-safety/quickstart-text?tabs=visual-studio%2Clinux&pivots=programming-language-rest).

یکی دیگر از جنبه‌ها، عملکرد کلی برنامه است. در برنامه‌های چندمدلی و چندرسانه‌ای، با عملکرد به معنای این است که سیستم همانطور که شما و کاربران‌تان انتظار دارید عمل می‌کند، از جمله تولید نکردن خروجی‌های مضر. مهم است عملکرد کل برنامه خود را با استفاده از [ارزیاب‌های کیفیت، عملکرد، ریسک و ایمنی](https://learn.microsoft.com/azure/ai-studio/concepts/evaluation-metrics-built-in) ارزیابی کنید. همچنین توانایی ایجاد و ارزیابی با استفاده از [ارزیاب‌های سفارشی](https://learn.microsoft.com/azure/ai-studio/how-to/develop/evaluate-sdk#custom-evaluators) را دارید.

می‌توانید برنامه هوش مصنوعی خود را در محیط توسعه با استفاده از [SDK ارزیابی هوش مصنوعی Azure](https://microsoft.github.io/promptflow/index.html) ارزیابی کنید. با داشتن مجموعه داده آزمایشی یا هدف، خروجی‌های برنامه تولید محتوا هوش مصنوعی شما توسط ارزیاب‌های داخلی یا ارزیاب‌های سفارشی انتخابی شما به صورت کمی اندازه‌گیری می‌شود. برای شروع با azure ai evaluation sdk به منظور ارزیابی سیستم خود، می‌توانید از [راهنمای شروع سریع](https://learn.microsoft.com/azure/ai-studio/how-to/develop/flow-evaluate-sdk) پیروی کنید. پس از اجرای یک ارزیابی، می‌توانید [نتایج را در Microsoft Foundry مشاهده کنید](https://learn.microsoft.com/azure/ai-studio/how-to/evaluate-flow-results).

## علائم تجاری

این پروژه ممکن است شامل علائم تجاری یا لوگوهایی برای پروژه‌ها، محصولات، یا خدمات باشد. استفاده مجاز از علائم تجاری یا لوگوی مایکروسافت تابع و باید از [دستورالعمل‌های علائم تجاری و برند مایکروسافت](https://www.microsoft.com/legal/intellectualproperty/trademarks/usage/general) پیروی کند.
استفاده از علائم تجاری یا لوگوی مایکروسافت در نسخه‌های تغییر یافته این پروژه نباید باعث سردرگمی یا القای حمایت مایکروسافت شود. هرگونه استفاده از علائم تجاری یا لوگوی طرف‌های سوم نیز تابع سیاست‌های آن‌ها است.
