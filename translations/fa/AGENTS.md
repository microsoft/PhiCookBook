# AGENTS.md

## نمای کلی پروژه

PhiCookBook یک مخزن جامع کتاب آشپزی است که شامل نمونه‌های عملی، آموزش‌ها و مستندات برای کار با خانواده مدل‌های زبانی کوچک (SLM) مایکروسافت به نام Phi می‌باشد. این مخزن موارد استفاده متنوعی از جمله استنتاج، تنظیم دقیق، کمّی‌سازی، پیاده‌سازی‌های RAG و برنامه‌های چندرسانه‌ای در پلتفرم‌ها و چارچوب‌های مختلف را نشان می‌دهد.

**فناوری‌های کلیدی:**
- **زبان‌ها:** Python, C#/.NET, JavaScript/Node.js
- **چارچوب‌ها:** ONNX Runtime, PyTorch, Transformers, MLX, OpenVINO, Semantic Kernel
- **پلتفرم‌ها:** Microsoft Foundry, GitHub Models, Hugging Face, Ollama
- **نوع مدل‌ها:** Phi-3, Phi-3.5, Phi-4 (متن، بینایی، چندرسانه‌ای، نسخه‌های استدلالی)

**ساختار مخزن:**
- `/code/` - نمونه‌های کد کاری و پیاده‌سازی‌های نمونه
- `/md/` - مستندات دقیق، آموزش‌ها و راهنماهای چگونه انجام دادن  
- `/translations/` - ترجمه‌های چندزبانه (بیش از ۵۰ زبان از طریق روند خودکار)
- `/.devcontainer/` - پیکربندی کانتینر توسعه (Python 3.12 با Ollama)

### راه‌اندازی محلی

**پیش‌نیازها:**
- Python 3.12 یا بالاتر
- .NET 8.0 SDK (برای نمونه‌های C#)
- Node.js نسخه ۱۸ به بالا و npm (برای نمونه‌های JavaScript)
- حداقل ۱۶ گیگابایت رم توصیه می‌شود

**نصب:**
```bash
git clone https://github.com/microsoft/PhiCookBook.git
cd PhiCookBook
```

**برای نمونه‌های Python:**
به دایرکتوری‌های نمونه خاص بروید و وابستگی‌ها را نصب کنید:
```bash
cd code/<example-directory>
pip install -r requirements.txt  # اگر فایل requirements.txt وجود داشته باشد
```

**برای نمونه‌های .NET:**
```bash
cd md/04.HOL/dotnet/src
dotnet restore LabsPhi.sln
dotnet build LabsPhi.sln
```

**برای نمونه‌های JavaScript/وب:**
```bash
cd code/08.RAG/rag_webgpu_chat
npm install
npm run dev  # شروع سرور توسعه
npm run build  # ساخت برای تولید
```

## دستورالعمل‌های درخواست Pull

### قبل از ارسال

1. **مخزن را فورک کنید** به حساب خود
2. **PRها را بر اساس نوع جدا کنید:**
   - اصلاحات اشکال در یک PR
   - به‌روزرسانی مستندات در PR دیگر
   - نمونه‌های جدید در PRهای جداگانه
   - اصلاحات تایپی می‌توانند ترکیب شوند

3. **مدیریت تضادهای ادغام:**
   - قبل از تغییرات، شاخه `main` محلی خود را به‌روز کنید
   - به‌طور منظم با upstream سینک کنید

4. **PRهای ترجمه:**
   - باید شامل ترجمه تمام فایل‌های فولدر باشند
   - ساختار را با زبان اصلی هماهنگ نگه دارید

### بررسی‌های اجباری

PRها به‌طور خودکار توسط جریان‌های کاری GitHub اعتبارسنجی می‌شوند:

1. **اعتبارسنجی مسیرهای نسبی** - همه لینک‌های داخلی باید کار کنند
   - لینک‌ها را محلی در VS Code با Ctrl+کلیک تست کنید
   - از پیشنهادهای مسیر در VS Code استفاده کنید (`./` یا `../`)

2. **چک کردن زبان URL** - آدرس‌های وب نباید شامل زبان‌های محلی کشورها باشند
   - `/en-us/`، `/en/` یا کدهای زبان دیگر را حذف کنید
   - از URLهای بین‌المللی عمومی استفاده کنید

3. **بررسی URLهای شکسته** - همه URLها باید پاسخ ۲۰۰ بازگردانند
   - قبل از ارسال دسترسی لینک‌ها را تأیید کنید
   - توجه: برخی خطاها ممکن است به دلیل محدودیت‌های شبکه باشند
